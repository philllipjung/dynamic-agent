## 1. 테스트 개요

### 1.1 테스트 목표

**마이크로서비스와 스파크 잡 사이의 분산 추적(Distributed Tracing) 연결 검증**

```
사용자 HTTP 요청
    ↓
마이크로서비스 (OpenTelemetry Agent 적용)
    → Trace ID 생성
    → traceparent 생성
    ↓ spark-submit --conf
스파크 잡 (Spark Extension Agent 적용)
    → traceparent 읽기
    → 같은 Trace ID로 Span 생성
    ↓
Jaeger/OpenSearch (같은 Trace ID로 연결된 Spans 저장)
```

### 1.2 핵심 성취

- ✅ **Trace ID 일치 검증**: 마이크로서비스 Trace ID와 스파크 잡 Trace ID 완벽 일치
- ✅ **W3C Trace Context 표준 준수**: traceparent 형식으로 Trace Context 전파
- ✅ **Spark Configuration 활용**: `--conf spark.driver.extraJavaOptions=-Dtraceparent=...`
- ✅ **Jaeger 검증**: `microservice-spark-test` 서비스와 Trace 확인

### 2.2 핵심 컴포넌트

#### 2.2.1 마이크로서비스 (OpenTelemetry Java Agent)

**역할**: HTTP 요청을 받아 자동으로 Trace 생성 및 traceparent 전달

**주요 컴포넌트**:
- `SparkTestApplication`: Spring Boot 메인 클래스
- `SparkJobController`: REST API 컨트롤러
- `SparkJobService`: Spark Job 제출 서비스

**역할**: 스파크 Job의 main 메서드에 진입하여 traceparent를 파싱하고 같은 Trace ID로 Span 생성

**핵심 기술**: ByteBuddy 런타임 코드 생성

**주요 컴포넌트**:
- `SparkOtelAgent`: Java Agent 진입점 (premain)
- `SparkJobInstrumentation`: ByteBuddy Instrumentation 설치
- `SparkJobAdvice`: main 메서드 Advice (코드 삽입)
- `BatchInstrumentation`: 배치 처리 Instrumentation

## 2.3 ByteBuddy 자동 계측 상세 분석

### 2.3.1 Java Agent生命周期

```
1. JVM 시작
   ↓
2. -javaagent:spark-otel-extension.jar 로드
   ↓
3. MANIFEST.MF의 Premain-Class 읽기
   ↓
4. SparkOtelAgent.premain() 호출
   ↓
5. ByteBuddy Agent Builder로 클래스 변환
   ↓
6. 변환된 클래스가 클래스로더에 로드
   ↓
7. 애플리케이션 실행
```

### 2.3.2 ByteBuddy Agent Builder 동작 원리

**코드**:
```java
public static void install(Instrumentation instrumentation, String targetClassName) {
    new AgentBuilder.Default()
        .type(named(targetClassName))                    // 1. 대상 클래스 지정
        .transform(new AgentBuilder.Transformer() {        // 2. 변환기 정의
            @Override
            public DynamicType.Builder<?> transform(
                    DynamicType.Builder<?> builder,
                    TypeDescription typeDescription,
                    ClassLoader classLoader,
                    JavaModule module,
                    ProtectionDomain protectionDomain) {

                // 3. 메서드에 Advice 적용
                return builder.visit(
                    Advice.to(SparkJobAdvice.class)           // 4. Advice 클래스 지정
                    .on(named("main"))                      // 5. 대상 메서드: main()
                );
            }
        })
        .installOn(instrumentation);                      // 6. JVM에 설치
}
```

**동작 단계**:

| 단계 | 설명 | 코드 변환 |
|------|------|----------|
| 1. 클래스 로드 전 | JVM이 `WordCountJob` 클래스를 로드하려고 함 | 원본 코드 |
| 2. Agent Builder 감지 | ByteBuddy가 해당 클래스를 가로챔 | - |
| 3. 타입 매칭 | `named("com.javaagent.test.spark.job.WordCountJob")` | - |
| 4. 메서드 매칭 | `named("main")` → `public static void main(String[] args)` | - |
| 5. Advice 적용 | `SparkJobAdvice`의 로직을 main()에 삽입 | 변환된 코드 |
| 6. 바이트코드 생성 | 변환된 클래스의 바이트코드 생성 | 메모리에 로드 |
| 7. 클래스로더에 전달 | 변환된 클래스가 애플리케이션에 로드됨 | 실행됨 |

**ByteBuddy 변환 후 main() 메서드** (개념적 표현):
```java
package com.javaagent.test.spark.job;

public class WordCountJob {
    public static void main(String[] args) throws Exception {
        // ========== ByteBuddy가 삽입한 코드 시작 ==========

        // @Advice.OnMethodEnter
        String traceParent = System.getProperty("traceparent", "");
        Span span = null;
        Context parentContext = Context.root();
        String parsedTraceId = null;

        if (traceParent != null && !traceParent.isEmpty()) {
            String[] parts = traceParent.split("-");
            if (parts.length == 4 && parts[0].equals("00")) {
                String traceId = parts[1];
                String spanId = parts[2];
                parsedTraceId = traceId;

                SpanContext parentSpanContext = SpanContext.create(
                    traceId, spanId,
                    TraceFlags.getSampled(),
                    TraceState.getDefault()
                );

                Span parentSpan = Span.wrap(parentSpanContext);
                parentContext = Context.root().with(parentSpan);
            }
        }

        io.opentelemetry.context.Scope parentScope = parentContext.makeCurrent();

        try {
            getCustomOpenTelemetry();
            span = customTracer.spanBuilder("spark.job")
                    .setSpanKind(SpanKind.INTERNAL)
                    .startSpan();
            span.setAttribute("job.name", jobName);
            span.setAttribute("job.type", "spark");

            io.opentelemetry.context.Scope scope = span.makeCurrent();

            // ========== ByteBuddy가 삽입한 코드 끝 ==========
            // ========== 원본 코드 시작 ==========

            // Spark Session 생성
            SparkSession spark = SparkSession.builder().appName("WordCount").getOrCreate();

            // WordCount 로직
            // ...

            spark.stop();

            // ========== 원본 코드 끝 ==========
            // ========== ByteBuddy가 삽입한 코드 (OnMethodExit) 시작 ==========

        } catch (Throwable throwable) {
            if (throwable != null) {
                span.recordException(throwable);
                span.setStatus(StatusCode.ERROR, throwable.getMessage());
            }
        } finally {
            span.end();
            scope.close();
        }
        parentScope.close();

        // ========== ByteBuddy가 삽입한 코드 (OnMethodExit) 끝 ==========
    }
}
```

### 2.3.4 @Advice.OnMethodEnter 상세 분석

**역할**: 메서드 진입 시 코드 실행

```java
@Advice.OnMethodEnter
public static void onMethodEnter(
        @Advice.Argument(0) String[] args,                    // main()의 첫 번째 인자
        @Advice.Local("span") Span span,                       // 로컬 변수 선언
        @Advice.Local("parentContext") Context parentContext,   // 로컬 변수 선언
        @Advice.Local("parsedTraceId") String parsedTraceId) {  // 로컬 변수 선언

    // traceparent 읽기
    String traceParent = System.getProperty("traceparent", "");

    // Trace Context 파싱 및 재구성
    // ...

    // Span 생성
    span = tracer.spanBuilder("spark.job").startSpan();
}
```

**@Advice 어노테이션 설명**:

| 어노테이션 | 역할 | 예시 |
|-----------|------|------|
| `@Advice.OnMethodEnter` | 메서드 시작 시 실행 | main() 진입 직후 |
| `@Advice.OnMethodExit` | 메서드 종료 시 실행 | main() 반환 직전 |
| `@Advice.Argument(0)` | 메서드 인자 참조 | `String[] args` |
| `@Advice.Local("name")` | 로컬 변수 선언 | 메서드 내에서 사용 |
| `@Advice.Thrown` | 예외 객체 참조 | catch 블록에서 사용 |

### 2.3.5 @Advice.OnMethodExit 상세 분석

**역할**: 메서드 종료 시 정리 작업

```java
@Advice.OnMethodExit(onThrowable = Throwable.class)
public static void onMethodExit(
        @Advice.Local("span") Span span,                    // OnMethodEnter에서 생성
        @Advice.Local("scope") io.opentelemetry.context.Scope scope,
        @Advice.Thrown Throwable throwable) {               // 발생한 예외

    if (span == null) {
        return;
    }

    try {
        if (throwable != null) {
            // 예외 기록
            span.recordException(throwable);
            span.setStatus(StatusCode.ERROR, throwable.getMessage());
        } else {
            // 정상 완료
            span.setStatus(StatusCode.OK);
        }
    } finally {
        // Span 종료 (중요!)
        span.end();
        if (scope != null) {
            scope.close();
        }
    }
}
```

public void submitSparkJob(JobRequest request, String traceparent) {
    List<String> command = new ArrayList<>();
    command.add("spark-submit");
    command.add("--class");
    command.add("com.javaagent.test.spark.job.WordCountJob");
    command.add("--master");

    command.add("--conf");
    command.add("spark.driver.extraJavaOptions=-Dtraceparent=" + traceparent);
    command.add("--conf");
    command.add("spark.executor.extraJavaOptions=-Dtraceparent=" + traceparent);

    command.add("../spark-test-services/spark-job/target/spark-test-job-1.0.0.jar");
    // ...
}
```

**변경 이유**:
- `-Dtraceparent=`를 직접 사용하면 spark-submit 프로세스의 JVM 시스템 프로퍼티가 됨
- Spark Job의 JVM 시스템 프로퍼티로 전달하려면 `--conf`를 사용해야 함
- `spark.driver.extraJavaOptions`: Driver JVM의 Java 옵션
- `spark.executor.extraJavaOptions`: Executor JVM의 Java 옵션

### 3.2 JobRequest.java 수정

**변경 후**:

**변경 이유**:

    @Advice.OnMethodEnter
    public static void onMethodEnter(
            @Advice.Local("span") Span span,
            @Advice.Local("parentContext") Context parentContext,
            @Advice.Local("parsedTraceId") String parsedTraceId) {

        // 1. traceparent 읽기
        String traceParent = System.getProperty("traceparent", "");
        parsedTraceId = null;
        parentContext = Context.root();

        if (traceParent != null && !traceParent.isEmpty()) {
            try {
                // 2. W3C traceparent 파싱
                String[] parts = traceParent.split("-");
                if (parts.length == 4 && parts[0].equals("00")) {
                    String traceId = parts[1];
                    String spanId = parts[2];
                    parsedTraceId = traceId;

                    // 3. SpanContext 재구성
                    SpanContext parentSpanContext = SpanContext.create(
                        traceId, spanId,
                        TraceFlags.getSampled(),
                        TraceState.getDefault()
                    );

                    // 4. Parent Context 생성
                    Span parentSpan = Span.wrap(parentSpanContext);
                    parentContext = Context.root().with(parentSpan);
                }
            } catch (Exception e) {
                logger.warning("[SparkJobInstrumentation] Failed to parse traceparent: " + e.getMessage());
            }
        }

        // 5. 핵심: Context를 current로 설정
        io.opentelemetry.context.Scope parentScope = parentContext.makeCurrent();

        try {
            // 6. Span 생성 - Context.current()에서 Trace ID를 가져옴
            span = tracer.spanBuilder("spark.job")
                    .setSpanKind(SpanKind.INTERNAL)
                    .startSpan();

            // 7. Trace ID 일치 검증
            String actualTraceId = span.getSpanContext().getTraceId();
            if (parsedTraceId != null && parsedTraceId.equalsIgnoreCase(actualTraceId)) {
                logger.info("[SparkJobInstrumentation] ✓ Trace ID continuity VERIFIED!");
            } else {
                logger.warning("[SparkJobInstrumentation] ✗ Trace ID mismatch! Expected: " + parsedTraceId + ", Actual: " + actualTraceId);
            }
        } finally {
            parentScope.close();
        }
    }
}
```

## 9. 핵심 성공 요인

### 9.1 makeCurrent() 패턴

**가장 중요한 발견**:

```java
// ✅ 올바른 순서
parentContext = Context.root().with(parentSpan);
io.opentelemetry.context.Scope parentScope = parentContext.makeCurrent();
try {
    span = tracer.spanBuilder("spark.job")
            .setSpanKind(SpanKind.INTERNAL)
            .startSpan();
    // span이 Context.current()에서 Trace ID를 가져옴
} finally {
    parentScope.close();
}
```

**원리**:
- `SpanBuilder`는 내부적으로 `Context.current()`를 호출
- `makeCurrent()`로 parent context를 현재 스레드에 설정해야 함
- 그래야 `SpanBuilder`가 parent의 Trace ID를 가져옴

### 9.2 Spark Configuration 활용

```java
// ✅ 올바른 방식
command.add("--conf");
command.add("spark.driver.extraJavaOptions=-Dtraceparent=" + traceparent);
command.add("--conf");
command.add("spark.executor.extraJavaOptions=-Dtraceparent=" + traceparent);
```

**원리**:
- Spark는 driver와 executor를 별도의 JVM 프로세스로 실행
- JVM 시스템 프로퍼티를 전달하려면 `--conf`를 사용해야 함
- `-D`를 직접 사용하면 spark-submit 프로세스의 프로퍼티가 됨

### 9.3 W3C Trace Context 표준

```
traceparent: 00-traceid-spanid-traceflags

예시: 00-a8f44cb39818847d4785496ce60f1f41-7b6aaee6fb47346e-01
     ┬─┆─────────────────────────────────────┆──────────────┆┬─┆
     │  └─ Trace ID (32 hex)                └─ Span ID      │  └─ Trace Flags
     └─ Version (00 = W3C standard)                                   (01 = Sampled)
```
