## 개요

`hook.py`는 특정 프로세스 PID에 대한 커널 수준 이벤트를 캡처하는 Python 기반 eBPF 모니터링 도구입니다. **BCC (BPF Compiler Collection)**를 사용하여 eBPF 프로그램을 커널에 로드하고 **ctypes**를 사용하여 perf 버퍼에서 바이너리 이벤트 데이터를 파싱합니다.

### 주요 기능

- **13가지 이벤트 유형** 모니터링 (스케줄러, 시스템 콜, IRQ, 메모리 등)
- **TraceId별 파일 라우팅** - 스레드 이름에서 추출한 trace ID로 이벤트 분리
- **JVM 스레드 필터링** - 내부 JVM 스레드 건너뛰기 (GC, Compiler 등)
- **주기적 스레드 스캔** - /proc을 통해 빠른 스레드 이름 변경 감지
- **시스템 콜 이름 매핑** - 335개 x86_64 시스템 콜 매핑

## eBPF C 프로그램

BPF 프로그램은 Python 여러 줄 문자열(`bpf_text`)로 내장되어 런타임에 로드됩니다.

### 구조

```c
// Python에 내장 (라인 13-445)
bpf_text = """
#include <linux/sched.h>
#include <asm/ptrace.h>
#include <linux/prctl.h>

// 이벤트 구조체
struct sched_event_t { ... };
struct syscall_event_t { ... };

// Perf 버퍼
BPF_PERF_OUTPUT(sched_events);
BPF_PERF_OUTPUT(syscall_events);

// 타이밍용 해시 맵
BPF_HASH(start_times, u32, u64);
BPF_HASH(syscall_start_times, u32, struct syscall_timing_t);

// Tracepoints
TRACEPOINT_PROBE(sched, sched_switch) {
    if (tgid != TARGET_PID) return 0;  // PID 필터링
    // 이벤트 캡처 로직
}
"""
```

### TARGET_PID 플레이스홀더

```python
# 런타임 교체 (라인 980)
bpf_program = bpf_text.replace('TARGET_PID', str(args.pid))
```

**교체 전**: `if (tgid != TARGET_PID) return 0;`
**교체 후**: `if (tgid != 12345) return 0;`

### 이벤트 타이밍 전략

이벤트는 **페어**(진입/종료)로 캡처되어 지속 시간을 계산합니다:

```
진입 이벤트 (sys_enter):
  syscall_start_times[tid] = {start_time, syscall_id}

종료 이벤트 (sys_exit):
  timing = syscall_start_times.lookup(tid)
  event.start_time_ns = timing.start_time
  event.end_time_ns = bpf_ktime_get_ns()
  syscall_events.perf_submit(&event)
  syscall_start_times.delete(tid)
```

자연스러운 페어가 없는 이벤트(블록 I/O, cpu_frequency)의 경우 **추정 오버헤드**를 사용합니다:

```c
// 블록 I/O: 추정 1ms
u64 overhead_ns = 1000000;
event.end_time_ns = start_ts + overhead_ns;
```

## 헬퍼 함수

### get_output_file (577-603)

스레드 이름에서 추출한 traceId를 기반으로 이벤트를 파일로 라우팅합니다.

```python
def get_output_file(thread_name):
    # traceId 추출 (휴리스틱)
    parts = thread_name.split('-')
    trace_id = None

    for part in parts:
        if len(part) >= 12 and part.isalnum():
            trace_id = part[:6]  # ⚠️ 처음 6자
            break

    if not trace_id and len(thread_name) >= 12:
        trace_id = thread_name[:6]

    # 파일 핸들 열기 또는 재사용
    if trace_id and trace_id not in trace_files:
        filename = f"result_{trace_id}.json"
        trace_files[trace_id] = open(filename, 'w')
        return trace_files[trace_id]

    # 기본 파일
    if "default" not in trace_files:
        trace_files["default"] = open("result_unknown.json", 'w')
    return trace_files["default"]
```

**출력 파일**:
- `result_{traceId}.json` - 커스텀 스레드 이름이 있는 이벤트
- `result_unknown.json` - traceId가 없는 이벤트

### get_full_thread_name (606-623)

`/proc` 파일시스템에서 스레드 이름을 읽습니다.

```python
def get_full_thread_name(pid, tid):
    try:
        with open(f"/proc/{pid}/task/{tid}/comm", "r") as f:
            current_name = f.read().strip()

            # 본 모든 이름 추적
            if tid not in all_thread_names:
                all_thread_names[tid] = set()
            all_thread_names[tid].add(current_name)

            # 캐시 업데이트
            thread_name_cache[tid] = current_name
            return current_name

    except (FileNotFoundError, ProcessLookupError):
        return "[unknown]"
```

### should_process_thread (625-630)

JVM 내부 스레드를 필터링합니다.

```python
def should_process_thread(full_thread_name):
    jvm_internal_threads = [
        "GC", "Compiler", "Finalizer", "Signal Dispatcher",
        "Reference Handler", "Attach Listener", "VM Thread", "Service Thread"
    ]
    return not any(name in full_thread_name for name in jvm_internal_threads)
```

**필터링 예시**:
- ❌ "GC Thread" → False (건너뜀)
- ❌ "C2 CompilerThread" → False (건너뜀)
- ✅ "reactor-http-epoll-1" → True (처리)
- ✅ "1234abcd5678efgR1" → True (처리)

## 이벤트 콜백 패턴

모든 11개 이벤트 콜백이 동일한 패턴을 따릅니다:

```python
def process_X_event(cpu, data, size):
    # 1. 바이너리 데이터 파싱
    event = cast(data, POINTER(XEvent)).contents

    # 2. /proc에서 스레드 이름 가져오기
    full_thread_name = get_full_thread_name(event.pid, event.tid)

    # 3. JVM 스레드 필터링
    if not should_process_thread(full_thread_name):
        return

    # 4. 출력 파일 가져오기
    output_file = get_output_file(full_thread_name)

    # 5. 읽기 쉬운 타임스탬프 생성
    current_dt = datetime.datetime.now()
    timestamp_readable = current_dt.strftime("%Y%m%d%H%M%S") + \
                       f"{current_dt.microsecond // 1000:03d}"

    # 6. 커널 시간을 Unix epoch로 변환
    current_time_ns = int(time.time() * 1e9)
    kernel_to_unix_offset = current_time_ns - event.start_time_ns
    unix_start_time = event.start_time_ns + kernel_to_unix_offset
    unix_end_time = event.end_time_ns + kernel_to_unix_offset

    # 7. JSON 출력 생성
    output = {
        "eventName": "event:name",
        "timestamp": timestamp_readable,
        "startTimeUnixNano": unix_start_time,
        "endTimeUnixNano": unix_end_time,
        "processId": event.pid,
        "threadId": event.tid,
        "threadName": full_thread_name,
    }

    # 8. 파일에 쓰기
    output_file.write(json.dumps(output) + '\n')
```

### ❌ 1. 중복 코드 (심각)

모든 이벤트 콜백에 동일한 타임스탬프 변환 로직이 포함되어 있습니다:

**권장사항**: 헬퍼 함수로 추출:

```python
def convert_kernel_time(kernel_start_ns, kernel_end_ns):
    current_dt = datetime.datetime.now()
    timestamp_readable = current_dt.strftime("%Y%m%d%H%M%S") + \
                       f"{current_dt.microsecond // 1000:03d}"
    current_time_ns = int(time.time() * 1e9)
    offset = current_time_ns - kernel_start_ns
    return timestamp_readable, kernel_start_ns + offset, kernel_end_ns + offset
```

## 주기적 스레드 스캐너

```python
def periodic_thread_scanner():
    """빠른 이름 변경을 캡처하기 위해 10ms마다 모든 스레드 스캔"""
    while not stop_scanning:
        try:
            task_dir = f"/proc/{monitoring_pid}/task"
            if os.path.exists(task_dir):
                for tid_str in os.listdir(task_dir):
                    tid = int(tid_str)
                    get_full_thread_name(monitoring_pid, tid)
        except (FileNotFoundError, OSError):
            pass

        time.sleep(0.01)  # 10ms 간격
```

### 목적

eBPF가 이미 스레드 이름 변경을 캡처하는데 `/proc`을 스캔하나요?

1. **경쟁 조건**: 스레드 이름이 prctl과 이벤트 처리 사이에 변경될 수 있음
2. **누락된 이벤트**: 일부 스레드 이름 변경이 prctl kprobe를 우회할 수 있음
3. **초기 상태**: 모니터링 시작 전 스레드 이름 캡처

### 5. 시간 변환 개선

```python
def convert_kernel_time(kernel_ns):
    """커널 타임스탬프를 Unix epoch 나노초로 변환"""
    boot_time_ns = get_boot_time_ns()  # /proc/uptime에서 읽기
    return boot_time_ns + kernel_ns

def get_boot_time_ns():
    """시스템 부팅 시간을 나노초로 가져오기"""
    with open('/proc/uptime', 'r') as f:
        uptime_seconds = float(f.read().split()[0])
    return int((time.time() - uptime_seconds) * 1e9)
```
