## 1. 작동하는 코드

### SpanAdvice.java (최종 버전)

```java
package com.javaagent.bytebuddy.advices;

import net.bytebuddy.asm.Advice;
import com.javaagent.bytebuddy.helper.SpanHelper;

public class SpanAdvice {

    @Advice.OnMethodEnter(inline = true)
    public static SpanHelper onMethodEnter(@Advice.Origin String method) {
        System.err.println(">>> ADVICE ENTER: " + method);
        System.err.flush();

        // SpanHelper를 사용하여 span 생성
        SpanHelper spanHelper = SpanHelper.createSpan(method);
        System.err.println(">>> SPAN CREATED: " + (spanHelper != null && spanHelper.isValid()));
        System.err.flush();

        return spanHelper;
    }

    @Advice.OnMethodExit(inline = true)
    public static void onMethodExit(@Advice.Enter SpanHelper spanHelper) {
        System.err.println("<<< ADVICE EXIT");

        if (spanHelper != null && spanHelper.isValid()) {
            spanHelper.complete();
            System.err.println("<<< SPAN COMPLETED");
        } else {
            System.err.println("<<< SPAN WAS NULL!");
        }
        System.err.flush();
    }
}
```

#### 동적 계측 적용

```java
public static String createSpan(String pid, String className, String methodName) {
    // 1. 타겟 클래스 로드
    Class<?> targetClass = Class.forName(className);
    ClassLoader targetClassLoader = targetClass.getClassLoader();

    // 2. Helper injection (Advice를 타겟 ClassLoader에 주입)
    injectHelper(targetClassLoader);

    // 3. Advice 클래스 로드 (타겟의 ClassLoader에서)
    Class<?> adviceClass = targetClassLoader.loadClass("com.javaagent.bytebuddy.advices.SpanAdvice");

    // 4. ByteBuddy로 변환된 bytecode 생성
    byte[] transformedBytes = new ByteBuddy()
        .redefine(targetClass)
        .visit(Advice.to(adviceClass)
            .on(ElementMatchers.named(methodName)
                    .and(ElementMatchers.isMethod())
                    .and(ElementMatchers.not(ElementMatchers.isStatic()))))
        .make()
        .getBytes();

    // 5. Retransformation 적용
    ClassFileTransformer transformer = (loader, className2, classBeingRedefined,
                                         protectionDomain, classfileBuffer) -> {
        if (className2.replace('/', '.').equals(targetClass.getName())) {
            return transformedBytes;
        }
        return null;
    };

    instrumentation.addTransformer(transformer, true);  // canRetransform = true
    instrumentation.retransformClasses(targetClass);
    instrumentation.removeTransformer(transformer);

    return "SUCCESS: Created span for " + className + "." + methodName;
}
```

## 4. 성공 핵심 요소

### ✅ 작동하는 조건

1. **동적 Attach 필수**
   - `VirtualMachine.attach()`로 실행 중인 JVM에 attach
   - `-javaagent` 정적 로드는 작동하지 않음

2. **Shaded JAR**
   - Advice와 Helper가 모두 shaded JAR에 포함되어야 함
   - `loadHelperClassBytes()`가 System ClassLoader에서 bytecode를 읽음

3. **Helper Injection**
   - `ClassLoader.defineClass()`로 타겟 ClassLoader에 Advice 주입
   - `--add-opens` 불필요 (동적 attach에서는 필요 없음)

4. **Retransformation Strategy**
   - `instrumentation.retransformClasses()` 사용
   - `addTransformer(transformer, true)` - canRetransform = true

5. **같은 ClassLoader**
   - Advice를 타겟의 ClassLoader에서 로드
   - `targetClassLoader.loadClass("com.javaagent.bytebuddy.advices.SpanAdvice")`

6. **OpenTelemetry 초기화**
   - Agent attach 시점에 `SpanHelper.setTracer()` 호출
   - OTLP exporter를 `http://localhost:4317`로 설정

### ❌ 작동하지 않는 방식

1. `-javaagent` 정적 로드
2. `ClassReloadingStrategy.fromInstalledAgent()`
3. 별도 Advice JAR 분리
4. Bootstrap classpath (`-Xbootclasspath/a`)

## 9. 문제 해결 히스토리

### 시도한 방법 (실패)

1. ❌ `-javaagent` 정적 로드
   - `ClassReloadingStrategy.fromInstalledAgent()` 작동 안 함
   - "The Byte Buddy agent is not installed or not accessible"

2. ❌ 별도 Advice JAR
   - "Helper class file not found" 에러
   - Shaded JAR에 포함해야 함

3. ❌ Bootstrap classpath
   - "duplicate class definition" 에러
   - Injection으로 중복 해결

4. ❌ `--add-opens`
   - 동적 attach에서는 불필요
   - 정적 로드에서도 해결 못 함

### 최종 해결책 (성공)

1. ✅ **동적 Attach**
   - `VirtualMachine.attach(pid)`
   - 실행 중인 JVM에만 작동

2. ✅ **Shaded JAR**
   - Advice를 포함한 fat JAR
   - System ClassLoader에서 bytecode 로드

3. ✅ **Helper Injection**
   - `ClassLoader.defineClass()` reflection 호출
   - 타겟 ClassLoader에 Advice 주입

4. ✅ **Retransformation**
   - `instrumentation.retransformClasses()`
   - 이미 로드된 클래스 재정의
