## 1. EventAdvice.java

### 1.1 파일 개요

**위치:** `bytebuddy-advice/src/main/java/com/javaagent/bytebuddy/advices/EventAdvice.java`

**역할:** ByteBuddy Advice 클래스로, Spring MVC의 DispatcherServlet.doService() 메서드가 호출될 때 HTTP 요청 정보를 캡처합니다.

**특징:**
- ByteBuddy의 `@Advice` 애너테이션 사용
- 인라인 계측 (inline = true)으로 성능 최적화
- Jakarta Servlet API 사용 (Spring Boot 3.x 호환)

---

### 1.2 전체 소스

```java
package com.javaagent.bytebuddy.advices;

import com.javaagent.bytebuddy.helper.EventHelper;
import net.bytebuddy.asm.Advice;
import jakarta.servlet.http.HttpServletRequest;

/**
 * Spring Event Capture Advice
 *
 * DispatcherServlet.doService() 메서드를 계측하여
 * HTTP 요청의 헤더와 바디를 JSON 형식으로 출력합니다.
 */
public class EventAdvice {

    @Advice.OnMethodEnter(inline = true)
    public static void onMethodEnter(
            @Advice.Origin String method,
            @Advice.Argument(0) Object request
    ) {
        // HttpServletRequest인지 확인
        if (request instanceof HttpServletRequest) {
            HttpServletRequest httpRequest = (HttpServletRequest) request;

            try {
                // 요청 정보 출력
                EventHelper.printRequestInfo(httpRequest);

                // 헤더 출력
                EventHelper.printRequestHeaders(httpRequest);

                // 바디 출력
                EventHelper.printRequestBody(httpRequest);

            } catch (Exception e) {
                System.err.println("{\"error\":\"" + e.getMessage() + "\"}");
            }
        }
    }
}
```

#### 1.3.2 @Advice.OnMethodEnter

```java
@Advice.OnMethodEnter(inline = true)
public static void onMethodEnter(
        @Advice.Origin String method,
        @Advice.Argument(0) Object request
) {
```

**@Advice.OnMethodEnter 설명:**

ByteBuddy의 핵심 애너테이션으로, 타겟 메서드의 **진입 시점**에 코드를 주입합니다.

```
원본 메서드:                        계측 후:
┌─────────────────────┐           ┌──────────────────────────────┐
│ doService() {       │           │ doService() {                │
│     // 원본 로직     │    →      │     // Advice 코드 (주입)     │
│ }                   │           │     // 원본 로직              │
└─────────────────────┘           │ }                            │
                                  └──────────────────────────────┘
```

**파라미터 분석:**

| 파라미터 | 타입 | 설명 |
|----------|------|------|
| `inline = true` | boolean | 성능 최적화. 메서드 호출 오버헤드 제거, 코드 직접 삽입 |
| `method` | @Advice.Origin String | 계측된 메서드의 시그니처 (예: "doService") |
| `request` | @Advice.Argument(0) Object | 타겟 메서드의 **첫 번째 인자** |

**@Advice.Argument(0) 상세:**

```
DispatcherServlet.doService(HttpServletRequest request, HttpServletResponse response)
                                 ↑ 첫 번째 인자 (index 0)
                                 ↑ @Advice.Argument(0)로 캡처
```

---

#### 1.3.3 타입 체크와 캐스팅

```java
if (request instanceof HttpServletRequest) {
    HttpServletRequest httpRequest = (HttpServletRequest) request;
```

**왜 타입 체크를 하는가?**

1. **안전성:** ByteBuddy는 모든 클래스의 메서드를 계측할 수 있습니다
2. **유연성:** 다른 타입의 첫 번째 인자를 가진 메서드에도 적용 가능
3. **방어적 프로그래밍:** 타입 불일치로 인한 ClassCastException 방지

**타입 체크 흐름:**
```
request (Object)
    │
    ├─ instanceof check
    │   │
    │   ├─ YES → 캡처 진행
    │   │        └─ EventHelper 호출
    │   │
    │   └─ NO  → 무시 (다른 메서드일 수 있음)
```

#### 1.3.4 EventHelper 호출 체인

```java
try {

    EventHelper.printRequestHeaders(httpRequest);
    EventHelper.printRequestBody(httpRequest);
```

**호출 순서와 역할:**

| 순서 | 메서드 | 역할 | 출력 예시 |
|------|--------|------|----------|
| 1 | `printRequestInfo` | 기본 요청 정보 | method, URI, query string, content type |
| 2 | `printRequestHeaders` | 모든 HTTP 헤더 | host, user-agent, authorization, ... |
| 3 | `printRequestBody` | 요청 바디 | POST 데이터, JSON, 등 |

**예외 처리 이유:**
- `getInputStream()`은 한 번만 읽을 수 있음
- 이미 읽은 경우 예외 발생 가능
- 계측 오류로 애플리케이션 중단 방지

### 1.5 핵심 설계 원칙

#### 1.5.1 인라인 계측 (inline = true)

```java
@Advice.OnMethodEnter(inline = true)
```

**장점:**
- **성능:** 메서드 호출 오버헤드 제거
- **코드 크기:** 바이트코드 직접 삽입으로 작은 크기
- **최적화:** JIT 컴파일러 최적화 용이

**비교:**

```
inline = true (현재):
┌──────────────┐
│ doService()  │
│   코드:      │
│   EventHelper │  ← 직접 삽입
│   .printXXX() │
└──────────────┘

inline = false:
┌──────────────┐
│ doService()  │
│   코드:      │
│   EventAdvice│  ← 메서드 호출
│   .onEnter() │     (오버헤드)
└──────────────┘
```

#### 1.5.2 정적 메서드 사용

```java
public static void onMethodEnter(...)
```

**이유:**
- ByteBuddy Advice는 인스턴스 없이 호출
- `static`으로 컴파일 및 런타임 오류 방지
- 인스턴스 생성 오버헤드 제거

#### 1.5.3 Jakarta Servlet API

```java
import jakarta.servlet.http.HttpServletRequest;
```

**배경:**
- Spring Boot 3.x부터 Jakarta EE 마이그레이션
- `javax.servlet` → `jakarta.servlet` 변경
- 하위 호환성을 위해 두 패키지 모두 지원

## 2. EventHelper.java

### 2.1 파일 개요

**위치:** `bytebuddy-advice/src/main/java/com/javaagent/bytebuddy/helper/EventHelper.java`

**역할:** HttpServletRequest에서 HTTP 요청 정보를 추출하고 JSON 형식으로 출력하는 유틸리티 클래스입니다.

**특징:**
- 정적 메서드만으로 구성 (유틸리티 클래스)
- Jakarta Servlet API 사용
- JSON 이스케이프 처리
- 다중 헤더 값 지준
- System.err 사용 (Maven 로그 출력)

---

### 2.2 전체 소스

```java
package com.javaagent.bytebuddy.helper;

import jakarta.servlet.http.HttpServletRequest;
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.*;

public class EventHelper {

    private static final int MAX_BODY_LENGTH = 10000;

    public static void printRequestInfo(HttpServletRequest request) {
        System.err.println("{");
        System.err.println("  \"method\": \"" + request.getMethod() + "\",");
        System.err.println("  \"uri\": \"" + request.getRequestURI() + "\",");
        System.err.println("  \"queryString\": \"" +
            (request.getQueryString() != null ? request.getQueryString() : "") + "\",");
        System.err.print("  \"contentType\": \"" +
            (request.getContentType() != null ? request.getContentType() : "") + "\"");
        System.err.println();
        System.err.println("}");
        System.err.flush();
    }

    public static void printRequestHeaders(HttpServletRequest request) {
        System.err.println("{\"headers\":{");

        List<String> headerNames = Collections.list(request.getHeaderNames());
        if (headerNames.isEmpty()) {
            System.err.println("}");
            return;
        }

        for (int i = 0; i < headerNames.size(); i++) {
            String headerName = headerNames.get(i);
            List<String> headerValues = Collections.list(request.getHeaders(headerName));

            System.err.print("  \"" + headerName + "\":");
            if (headerValues.size() == 1) {
                System.err.print("\"" + escapeJson(headerValues.get(0)) + "\"");
            } else {
                System.err.print("[");
                for (int j = 0; j < headerValues.size(); j++) {
                    System.err.print("\"" + escapeJson(headerValues.get(j)) + "\"");
                    if (j < headerValues.size() - 1) System.err.print(", ");
                }
                System.err.print("]");
            }

            if (i < headerNames.size() - 1) {
                System.err.println(",");
            } else {
                System.err.println();
            }
        }

        System.err.println("}}");
        System.err.flush();
    }

    public static void printRequestBody(HttpServletRequest request) {
        System.err.print("{\"body\":\"");

        try {
            if (request.getContentLength() <= 0) {
                System.err.println("\"}");
                return;
            }

            // TODO: getInputStream() requires ServletInputStream class
            System.err.println("(body length: " +
                request.getContentLength() + " bytes - temporarily disabled)\"}");

        } catch (Exception e) {
            System.err.println("\"}");
        }

        System.err.flush();
    }

    private static String escapeJson(String s) {
        if (s == null) return "";
        return s.replace("\\", "\\\\")
                 .replace("\"", "\\\"")
                 .replace("\n", "\\n")
                 .replace("\r", "\\r")
                 .replace("\t", "\\t");
    }
}
```

#### 2.3.2 printRequestInfo()

```java
public static void printRequestInfo(HttpServletRequest request) {
    System.err.println("{");
    System.err.println("  \"method\": \"" + request.getMethod() + "\",");
    System.err.println("  \"uri\": \"" + request.getRequestURI() + "\",");
    System.err.println("  \"queryString\": \"" +
        (request.getQueryString() != null ? request.getQueryString() : "") + "\",");
    System.err.print("  \"contentType\": \"" +
        (request.getContentType() != null ? request.getContentType() : "") + "\"");
    System.err.println();
    System.err.println("}");
    System.err.flush();
}
```

**출력 형식:**
```json
{
  "method": "GET",
  "uri": "/test1",
  "queryString": "userId=user123&data=testdata",
  "contentType": "application/json"
}
```

**HttpServletRequest 메서드 분석:**

| 메서드 | 반환 타입 | 설명 | 예시 |
|--------|-----------|------|------|
| `getMethod()` | String | HTTP 메서드 | "GET", "POST", "PUT", "DELETE" |
| `getRequestURI()` | String | 요청 경로 (쿼리 제외) | "/test1" |
| `getQueryString()` | String | 쿼리 문자열 | "userId=123&data=abc" |
| `getContentType()` | String | Content-Type 헤더 | "application/json" |

**Null 체크 패턴:**
```java
(request.getQueryString() != null ? request.getQueryString() : "")
```

**이유:**
- `getQueryString()`은 쿼리가 없으면 `null` 반환
- JSON에 `null`이 아닌 빈 문자열 `""`로 출력
- `request.getQueryString() ?: ""` (Groovy 스타일)과 동일

**System.err 사용 이유:**
```
System.out vs System.err
├─ System.out: 표준 출력 (로그 파일로 감)
└─ System.err: 표준 에러 (콘솔에 즉시 출력)

Maven spring-boot:run 실행 시:
- System.out → 로그 파일만 (안 보임)
- System.err → 콘솔 + 로그 파일 (보임)
```

**flush() 호출:**
```java
System.err.flush();
```

**목적:** 버퍼 즉시 비우기
- 버퍼링 지연 방지
- 실시간 로그 확인
- 디버깅 용이성

#### 2.3.3 printRequestHeaders()

```java
public static void printRequestHeaders(HttpServletRequest request) {
    System.err.println("{\"headers\":{");

    List<String> headerNames = Collections.list(request.getHeaderNames());

        String headerName = headerNames.get(i);

        System.err.print("  \"" + headerName + "\":");
        if (headerValues.size() == 1) {
            System.err.print("\"" + escapeJson(headerValues.get(0)) + "\"");
        } else {
            System.err.print("[");
            for (int j = 0; j < headerValues.size(); j++) {
                System.err.print("\"" + escapeJson(headerValues.get(j)) + "\"");
                if (j < headerValues.size() - 1) System.err.print(", ");
            }
            System.err.print("]");
        }

        if (i < headerNames.size() - 1) {
            System.err.println(",");
        } else {
            System.err.println();
        }
    }

    System.err.println("}}");
    System.err.flush();
}
```

**출력 형식:**
```json
{
  "headers": {
    "host": "localhost:8081",
    "user-agent": "curl/8.18.0",
    "accept": "*/*",
    "content-type": "application/json",
    "authorization": "Bearer token123",
    "x-custom-header": "custom-value"
  }
}
```

**다중 헤더 값 처리:**

HTTP 헤더는 하나의 이름에 여러 값을 가질 수 있습니다.

```
단일 값:                    다중 값:
Accept: application/json    Accept: application/json
                            Accept: text/html
                            Accept: */*
```

**구현 로직:**

```java
List<String> headerValues = Collections.list(request.getHeaders(headerName));

if (headerValues.size() == 1) {
    // 단일 값: 문자열로 출력
    System.err.print("\"" + escapeJson(headerValues.get(0)) + "\"");
} else {
    // 다중 값: JSON 배열로 출력
    System.err.print("[");
    for (int j = 0; j < headerValues.size(); j++) {
        System.err.print("\"" + escapeJson(headerValues.get(j)) + "\"");
        if (j < headerValues.size() - 1) System.err.print(", ");
    }
    System.err.print("]");
}
```

**출력 예시:**

| 헤더 값 개수 | 출력 |
|--------------|------|
| 1개 | `"authorization": "Bearer token123"` |
| 여러 개 | `"accept": ["application/json", "text/html", "*/*"]` |

**Collections.list() 사용:**

```java
List<String> headerNames = Collections.list(request.getHeaderNames());
```

**배경:**
- `request.getHeaderNames()`는 `Enumeration<String>` 반환 (Java 1.0 레거시)
- `Collections.list()`로 `Enumeration` → `List` 변환
- 현대적인 for-each 루프 사용 가능

```java
// 레거시 방식 (Enumeration)
Enumeration<String> names = request.getHeaderNames();
while (names.hasMoreElements()) {
    String name = names.nextElement();
    // ...
}

// 현대적 방식 (List + for-each)
List<String> names = Collections.list(request.getHeaderNames());
for (String name : names) {
    // ...
}
```

#### 2.3.4 printRequestBody()

```java
public static void printRequestBody(HttpServletRequest request) {
    System.err.print("{\"body\":\"");

    try {
        if (request.getContentLength() <= 0) {
            System.err.println("\"}");
            return;
        }

        // TODO: getInputStream() requires ServletInputStream class
        System.err.println("(body length: " +
            request.getContentLength() + " bytes - temporarily disabled)\"}");

    } catch (Exception e) {
        System.err.println("\"}");
    }

    System.err.flush();
}
```

**출력 형식:**
```json
{"body": "(body length: 125 bytes - temporarily disabled)"}
```

또는 (비어 있을 때):
```json
{"body": ""}
```

**구현 제한 사항:**

```java
// TODO: getInputStream() requires ServletInputStream class
```

**문제:**
```java
// 현재 구현 (불가능):
BufferedReader reader = new BufferedReader(
    new InputStreamReader(request.getInputStream())
);
// ❌ ClassCastException: ServletInputStream cannot be cast to InputStream
```

**원인:**
- ByteBuddy 계측 시 `ServletInputStream` 클래스를 사용할 수 없음
- 클래스패스에 없거나 로드되지 않음

**임시 해결:**
- 바디 길이만 출력
- 실제 내용은 출력하지 않음

**getContentLength() 사용:**

```java
if (request.getContentLength() <= 0) {
    System.err.println("\"}");
    return;
}
```

**반환 값:**
- 바디가 있으면: 바이트 길이 (양수)
- 바디가 없으면: -1
- Content-Length 헤더가 없으면: -1

#### 2.3.5 escapeJson()

```java
private static String escapeJson(String s) {
    if (s == null) return "";
    return s.replace("\\", "\\\\")
             .replace("\"", "\\\"")
             .replace("\n", "\\n")
             .replace("\r", "\\r")
             .replace("\t", "\\t");
}
```

**JSON 이스케이프 규칙:**

| 문자 | 이스케이프 | 설명 |
|------|------------|------|
| `\` | `\\` | 백슬래시 |
| `"` | `\"` | 쌍따옴표 |
| `\n` | `\\n` | 줄바꿈 |
| `\r` | `\\r` | 캐리지 리턴 |
| `\t` | `\\t` | 탭 |

**예시:**

```
입력: "Bearer token123"
출력: "Bearer token123"

입력: "Line1\nLine2"
출력: "Line1\\nLine2"

입력: "Path: C:\Users\Test"
출력: "Path: C:\\Users\\Test"
```

**왜 필요한가?**

```java
// 이스케이프 없이 (잘못됨):
System.err.println("{\"authorization\": \"Bearer \"token\"\"}");
                                              ↑ JSON 오류

// 이스케이프 후 (올바름):
System.err.println("{\"authorization\": \"Bearer \\\"token\\\"\"}");
```

**순서 중요:**

```java
return s.replace("\\", "\\\\)  // 1. 먼저 백슬래시
         .replace("\"", "\\\"") // 2. 그 다음 쌍따옴표
```

**반대로 하면:**
```java
s.replace("\"", "\\\"")
 .replace("\\", "\\\\)  // 이미 추가된 \까지 이스케이프!
```

### 3.1 파일 개요

**역할:** Java Agent의 메인 클래스로, JVM에 attach되어 런타임에 클래스를 계측합니다. 독립적인 HTTP 서버를 운영하여 계측 명령을 받습니다.

### 3.5 createEventAdvice() 메서드

```java
public static String createEventAdvice(String className, String methodName) {
    try {
        if (instrumentation == null) {
            return "ERROR: Agent not initialized. Please attach to JVM first.";
        }

        String methodKey = className + "." + methodName;

        // 이미 적용된 메서드인지 확인
        if (appliedMethods.contains(methodKey)) {
            return "WARNING: EventAdvice already applied to " + methodKey;
        }

        // ByteBuddy 빌더로 계측 정의
        AgentBuilder builder = new AgentBuilder.Default()
            .type(named(className))
            .transform((builder2, typeDescription, classLoader, module) ->
                builder2.visit(advice(EventAdvice.class)
                    .on(named(methodName))
                    .to(typeDescription))
            );

        // 계측 적용
        builder.installOn(instrumentation);

        appliedMethods.add(methodKey);

        return "SUCCESS: Created event advice for " + methodKey;

    } catch (Exception e) {
        return "ERROR: " + e.getMessage();
    }
}
```

**ByteBuddy 계측 과정:**

```
1. 입력 검증
   │
   ├─ instrumentation == null?
   │  └─ YES → "ERROR: Agent not initialized"
   │
   └─ 이미 적용됨?
      └─ YES → "WARNING: Already applied"

2. AgentBuilder 생성
   │
   ├─ type(named(className))
   │  └─ 타겟 클래스 지정
   │
   ├─ transform(...)
   │  └─ 변환 정의
   │     ├─ advice(EventAdvice.class)
   │     │  └─ EventAdvice 로 주입
   │     │
   │     ├─ on(named(methodName))
   │     │  └─ 타겟 메서드 지정
   │     │
   │     └─ to(typeDescription)
   │        └─ 클래스에 적용

3. installOn(instrumentation)
   │
   └─ JVM에 계측 적용
      └─ 클래스 재정의 (Redefine Classes)

   │
   └─ 중복 적용 방지
```

### 4.2 attach() 메서드

```java
@PostMapping("/attach")
public Map<String, Object> attach(@RequestBody AttachRequest request) {
    try {
        String pid = request.getPid();

        // className이 제공되면 PID 찾기
        if ((pid == null || pid.isEmpty()) && request.getClassName() != null) {
            pid = com.javaagent.arthas.ArthasManager.findPidByClassName(
                request.getClassName());

            if (pid.startsWith("ERROR")) {
                return Map.of(
                    "success", false,
                    "error", "Class not found: " + request.getClassName()
                );
            }
        }

        if (pid == null || pid.isEmpty()) {
            return Map.of(
                "success", false,
                "error", "Either pid or className is required"
            );
        }

        // 이미 attach된 경우
        if (attachedPids.contains(pid)) {
            return Map.of(
                "success", true,
                "message", "Already attached to PID " + pid,
                "pid", pid
            );
        }

        // ByteBuddy Agent JAR 찾기
        String agentJar = findAgentJar();
        if (agentJar == null) {
            return Map.of(
                "success", false,
                "error", "ByteBuddy Agent JAR not found"
            );
        }

        // Attach 수행
        VirtualMachine vm = VirtualMachine.attach(pid);
        String agentArgs = request.getInstrumentation() != null ?
            "instrumentation=" + request.getInstrumentation() : "";
        vm.loadAgent(agentJar, agentArgs);
        vm.detach();

        attachedPids.add(pid);

        return Map.of(
            "success", true,
            "message", "ByteBuddy Agent attached to PID " + pid,
            "pid", pid,
            "agentJar", agentJar
        );

    } catch (Exception e) {
        return Map.of(
            "success", false,
            "error", e.getMessage()
        );
    }
}
```

**attach 과정:**

```
1. 요청 수신
   │
   │ POST /api/bytebuddy/attach
   │ {"pid": "12280"}
   │ 또는
   │ {"className": "com.test.service.test1.Test1Application"}
   ▼

2. PID 결정
   │
   ├─ pid가 직접 제공됨?
   │  └─ YES → 사용
   │
   └─ className이 제공됨?
      └─ YES → ArthasManager.findPidByClassName()
         └─ jps 명령으로 검색

3. 이미 attach됨?
   │
   └─ YES → "Already attached" 반환

4. Agent JAR 찾기
   │
   └─ findAgentJar()
      └─ bytebuddy-agent-1.0.0.jar 경로 반환

5. JVM Attach
   │
   ├─ VirtualMachine.attach(pid)
   │  └─ 타겟 JVM에 연결
   │
   ├─ vm.loadAgent(agentJar, args)
   │  └─ agent-main() 호출
```

### 5.2 HTTP 요청 처리 흐름

```
1. Attach 요청
   │
   ├─ 사용자: curl POST http://localhost:9999/api/bytebuddy/attach
   │             {"pid": "12280"}
   │
   ├─ agent-server: ByteBuddyController.attach()
   │               → VirtualMachine.attach("12280")
   │               → vm.loadAgent(bytebuddy-agent-1.0.0.jar)
   │
   └─ Target JVM: ByteBuddyAgent.agentmain()
                  → instrumentation 초기화
                  → HTTP Server 시작 (포트 10000)
                  → 응답: {"success": true}

2. EventAdvice 적용 요청
   │
   ├─ 사용자: curl POST http://localhost:10000/api/createEventAdvice
   │             {"className":"org.springframework.web.servlet.DispatcherServlet",
   │              "methodName":"doService"}
   │
   ├─ ByteBuddyAgent HTTP Server: createEventAdvice()
   │                           → ByteBuddy 빌더 생성
   │                           → DispatcherServlet.doService() 계측
   │                           → EventAdvice 코드 주입
   │
   └─ 응답: {"success": true,
             "message": "SUCCESS: Created event advice for ..."}

3. HTTP 요청 (계측된 메서드 호출)
   │
   ├─ 사용자: curl GET http://localhost:8081/test1?userId=123
   │
   ├─ Tomcat: DispatcherServlet.doService() 호출
   │
   ├─ EventAdvice.onMethodEnter() 실행
   │  → EventHelper.printRequestInfo()
   │  → EventHelper.printRequestHeaders()
   │  → EventHelper.printRequestBody()
   │
   ├─ System.err 출력 (Maven 로그):
   │  {
   │    "method": "GET",
   │    "uri": "/test1",
   │    "queryString": "userId=123",
   │    "contentType": "application/json"
   │  }
   │  {"headers":{"host":"localhost:8081",...}}
   │  {"body":""}
   │
   └─ 원본 doService() 로직 실행 → Controller → Response
```

## 6. 요약

### 6.1 핵심 컴포넌트

| 컴포넌트 | 역할 | 핵심 로직 |
|----------|------|----------|
| EventAdvice | 메서드 계측 정의 | @Advice.OnMethodEnter로 진입 시점 캡처 |
| EventHelper | HTTP 요청 추출 | HttpServletRequest → JSON 변환 |
| ByteBuddyAgent | Java Agent 본체 | JVM attach, HTTP 서버, 계측 수행 |
| ByteBuddyController | REST API 제공 | attach 요청 처리 |

### 6.2 주요 기술

| 기술 | 사용처 | 특징 |
|------|--------|------|
| ByteBuddy | 런타임 계측 | @Advice, AgentBuilder |
| Jakarta Servlet | HTTP API | HttpServletRequest (Spring Boot 3.x) |
| java.lang.instrument | JVM 계측 | Instrumentation, VirtualMachine |
| OpenTelemetry | 분산 추적 | Tracer, Span |
| com.sun.net.httpserver | 경량 HTTP 서버 | 포트 10000에서 API 제공 |

### 6.3 핵심 설계 원칙

1. **인라인 계측:** 성능 최적화를 위해 inline=true 사용
2. **정적 메서드:** ByteBuddy Advice는 인스턴스 없이 호출
3. **Null 안전성:** 모든 곳에 null 체크
4. **JSON 이스케이프:** 올바른 JSON 형식 보장
5. **System.err 사용:** Maven 로그에 즉시 출력
6. **중복 방지:** appliedMethods Set으로 이미 계측된 메서드 추적
7. **스레드 안전성:** AtomicBoolean, ConcurrentHashMap 사용
