## 1. EventAdvice 함수

### 1.1 onMethodEnter()

**함수 시그니처:**
```java
@Advice.OnMethodEnter(inline = true)
public static void onMethodEnter(
    @Advice.Origin String method,
    @Advice.Argument(0) Object request
)
```

**설명:**
ByteBuddy Advice의 진입점 메서드로, 타겟 메서드가 호출될 때 실행됩니다. HTTP 요청 정보를 캡처하여 JSON 형식으로 출력합니다.

---

#### 입력 (Input)

| 파라미터 | 타입 | 설명 | 예시 | 필수여부 |
|----------|------|------|------|----------|
| `method` | String | 계측된 메서드의 시그니처 | `"protected void org.springframework.web.servlet.DispatcherServlet.doService(...)"` | 자동 주입 |
| `request` | Object | 타겟 메서드의 첫 번째 인자 | `HttpServletRequest` 인스턴스 | 자동 주입 |

**입력 제약사항:**
- `request`는 `HttpServletRequest`의 인스턴스여야 함
- 다른 타입이면 무시됨 (`instanceof` 체크로 필터링)
- `null`인 경우 무시됨

---

#### 출력 (Output)

| 타입 | 설명 |
|------|------|
| `void` | 반환값 없음 (직접 System.err로 출력) |

**System.err 출력 포맷:**
```
{
  "method": "GET",
  "uri": "/test1",
  "queryString": "userId=user123&data=testdata",
  "contentType": "application/json"
}
{
  "headers": {
    "host": "localhost:8081",
    "user-agent": "curl/8.18.0",
    "accept": "*/*",
    "content-type": "application/json",
    "authorization": "Bearer token123",
    "x-custom-header": "custom-value"
  }
}
{
  "body": ""
}
```

---

#### 실행 예시

**입력 시나리오:**
```
타겟: DispatcherServlet.doService()
첫 번째 인자: HttpServletRequest (유효)
```

**출력 결과:**
```
{
  "method": "GET",
  "uri": "/test1",
  "queryString": "userId=user123",
  "contentType": "application/json"
}
{
  "headers": {
    "host": "localhost:8081",
    "user-agent": "curl/8.18.0",
    "accept": "*/*"
  }
}
{
  "body": "(body length: 125 bytes - temporarily disabled)"
}
```

---

#### 예외 상황

| 상황 | 동작 | 출력 |
|------|------|------|
| `request`가 `null` | 무시 | (출력 없음) |
| `request`가 `HttpServletRequest`가 아님 | 무시 | (출력 없음) |
| `EventHelper`에서 예외 발생 | 예외 캡처 및 JSON 에러 출력 | `{"error":"예외 메시지"}` |
| `getHeaderNames()` 실패 | 예외 캡처 | `{"error":"..."}` |

---

#### 관련 함수

```
onMethodEnter()
    ├─ EventHelper.printRequestInfo()
    ├─ EventHelper.printRequestHeaders()
    └─ EventHelper.printRequestBody()
```

## 2. EventHelper 함수

### 2.1 printRequestInfo()

**함수 시그니처:**
```java
public static void printRequestInfo(HttpServletRequest request)
```

**설명:**
HTTP 요청의 기본 정보 (메서드, URI, 쿼리 문자열, Content-Type)를 JSON 형식으로 출력합니다.

---

#### 입력 (Input)

| 파라미터 | 타입 | 설명 | 필수여부 |
|----------|------|------|----------|
| `request` | HttpServletRequest | HTTP 요청 객체 | 필수 |

**입력 제약사항:**
- `request`는 `null`이 아니어야 함
- `request`는 유효한 `HttpServletRequest` 인스턴스여야 함

---

#### 출력 (Output)

| 타입 | 설명 | 출력 대상 |
|------|------|----------|
| `void` | 반환값 없음 | `System.err` (Maven 콘솔) |

**출력 포맷:**
```json
{
  "method": "{HTTP_METHOD}",
  "uri": "{REQUEST_URI}",
  "queryString": "{QUERY_STRING}",
  "contentType": "{CONTENT_TYPE}"
}
```

---

#### 입력/출력 예시

**예시 1: GET 요청 (쿼리 있음)**

입력:
```java
// HttpServletRequest 정보
Method: GET
Request URI: /test1
Query String: userId=user123&data=testdata
Content-Type: application/json
```

출력:
```json
{
  "method": "GET",
  "uri": "/test1",
  "queryString": "userId=user123&data=testdata",
  "contentType": "application/json"
}
```

---

**예시 2: POST 요청 (쿼리 없음)**

입력:
```java
Method: POST
Request URI: /api/users
Query String: null
Content-Type: application/json; charset=UTF-8
```

출력:
```json
{
  "method": "POST",
  "uri": "/api/users",
  "queryString": "",
  "contentType": "application/json; charset=UTF-8"
}
```

---

**예시 3: null 쿼리 처리**

입력:
```java
QueryString: null
```

출력:
```json
{
  "queryString": "",
  ...
}
```

---

#### 예외 상황

| 상황 | 동작 | 결과 |
|------|------|------|
| `request.getMethod()`가 `null` | 빈 문자열로 대체 | `"method": ""` |
| `request.getQueryString()`가 `null` | 빈 문자열로 대체 | `"queryString": ""` |
| `request.getContentType()`가 `null` | 빈 문자열로 대체 | `"contentType": ""` |

---

#### 내부 동작

```
printRequestInfo(request)
    │
    ├─ request.getMethod()
    │  └─ "GET", "POST", "PUT", "DELETE", etc.
    │
    ├─ request.getRequestURI()
    │  └─ "/test1", "/api/users", etc.
    │
    ├─ request.getQueryString()
    │  ├─ null → ""
    │  └─ "key=value&key2=value2"
    │
    ├─ request.getContentType()
    │  ├─ null → ""
    │  └─ "application/json"
    │
    └─ System.err.println() (JSON 형식)
```

### 2.2 printRequestHeaders()

**함수 시그니처:**
```java
public static void printRequestHeaders(HttpServletRequest request)
```

**설명:**
HTTP 요청의 모든 헤더를 JSON 형식으로 출력합니다. 다중 헤더 값(같은 이름에 여러 값)을 지원합니다.

---

#### 입력 (Input)

| 파라미터 | 타입 | 설명 | 필수여부 |
|----------|------|------|----------|
| `request` | HttpServletRequest | HTTP 요청 객체 | 필수 |

---

#### 출력 (Output)

| 타입 | 설명 | 출력 대상 |
|------|------|----------|
| `void` | 반환값 없음 | `System.err` |

**출력 포맷:**
```json
{
  "headers": {
    "header-name-1": "header-value-1",
    "header-name-2": "header-value-2",
    "multi-value-header": ["value1", "value2", "value3"]
  }
}
```

---

#### 입력/출력 예시

**예시 1: 기본 헤더 (단일 값)**

입력:
```java
Headers:
  host: localhost:8081
  user-agent: curl/8.18.0
  accept: */*
  content-type: application/json
  authorization: Bearer token123
```

출력:
```json
{
  "headers": {
    "host": "localhost:8081",
    "user-agent": "curl/8.18.0",
    "accept": "*/*",
    "content-type": "application/json",
    "authorization": "Bearer token123"
  }
}
```

---

**예시 2: 다중 값 헤더**

입력:
```java
Headers:
  accept: application/json
  accept: text/html
  accept: */*
  cookie: session=abc123
  cookie: token=xyz789
```

출력:
```json
{
  "headers": {
    "accept": ["application/json", "text/html", "*/*"],
    "cookie": ["session=abc123", "token=xyz789"]
  }
}
```

---

**예시 3: 빈 헤더**

입력:
```java
Headers: (없음)
```

출력:
```json
{
  "headers": {
  }
}
```

---

#### 헤더 값 처리 로직

```
printRequestHeaders(request)
    │
    ├─ request.getHeaderNames()
    │  └─ Enumeration<String> → List<String>
    │
    ├─ 각 헤더 이름에 대해:
    │  │
    │  ├─ request.getHeaders(headerName)
    │  │  └─ Enumeration<String> → List<String>
    │  │
    │  ├─ 값 개수 확인
    │  │  │
    │  │  ├─ 1개 → 문자열로 출력
    │  │  │  └─ "header-name": "value"
    │  │  │
    │  │  └─ 여러 개 → JSON 배열로 출력
    │  │     └─ "header-name": ["value1", "value2", ...]
    │  │
    │  └─ escapeJson()로 값 이스케이프
    │     ├─ " → \"
    │     ├─ \ → \\
    │     ├─ \n → \\n
    │     └─ etc.
```

---

#### 예외 상황

| 상황 | 동작 | 결과 |
|------|------|------|
| 헤더가 없음 | 빈 객체 출력 | `{"headers": {}}` |
| 헤더 값에 특수 문자 | 이스케이프 처리 | `"value": "Line1\\nLine2"` |
| 헤더 값이 null | 빈 문자열로 처리 | `"value": ""` |

### 2.3 printRequestBody()

**함수 시그니처:**
```java
public static void printRequestBody(HttpServletRequest request)
```

**설명:**
HTTP 요청 바디를 출력합니다. 현재 구현에서는 바디 길이만 출력합니다 (실제 내용은 출력하지 않음).

---

#### 입력 (Input)

| 파라미터 | 타입 | 설명 | 필수여부 |
|----------|------|------|----------|
| `request` | HttpServletRequest | HTTP 요청 객체 | 필수 |

---

#### 출력 (Output)

| 타입 | 설명 | 출력 대상 |
|------|------|----------|
| `void` | 반환값 없음 | `System.err` |

**출력 포맷:**
```json
{
  "body": ""
}
```

또는 (바디가 있는 경우):
```json
{
  "body": "(body length: 125 bytes - temporarily disabled)"
}
```

---

#### 입력/출력 예시

**예시 1: 빈 바디 (GET 요청)**

입력:
```java
Content-Length: -1 (또는 0)
```

출력:
```json
{
  "body": ""
}
```

---

**예시 2: 바디 있음 (POST 요청)**

입력:
```java
Content-Length: 125
Body: {"username":"test","password":"secret"}
```

출력:
```json
{
  "body": "(body length: 125 bytes - temporarily disabled)"
}
```

---

#### 제한사항

**현재 구현:**
```java
// TODO: getInputStream() requires ServletInputStream class
System.err.println("(body length: " +
    request.getContentLength() + " bytes - temporarily disabled)\"}");
```

**이유:**
- ByteBuddy 계측 환경에서 `ServletInputStream` 클래스를 사용할 수 없음
- `request.getInputStream()` 호출 시 `ClassCastException` 발생 가능

**향후 개선:**
- Byte Code Manipulation을 통한 InputStream 접근
- 콘텐츠 복사 래퍼 사용
- Content-Length 기반 바디 읽기

---

### 2.4 escapeJson()

**함수 시그니처:**
```java
private static String escapeJson(String s)
```

**설명:**
문자열의 JSON 특수 문자를 이스케이프 처리합니다.

---

#### 입력 (Input)

| 파라미터 | 타입 | 설명 | 필수여부 |
|----------|------|------|----------|
| `s` | String | 이스케이프할 문자열 | 필수 |

---

#### 출력 (Output)

| 타입 | 설명 | 예시 |
|------|------|------|
| `String` | 이스케이프된 문자열 | `"Hello\\nWorld"` |

---

#### 이스케이프 규칙

| 입력 문자 | 이스케이프 결과 | 설명 |
|-----------|-----------------|------|
| `\` | `\\` | 백슬래시 |
| `"` | `\"` | 쌍따옴표 |
| `\n` | `\\n` | 줄바꿈 (LF) |
| `\r` | `\\r` | 캐리지 리턴 (CR) |
| `\t` | `\\t` | 탭 |

---

#### 입출력 예시

**예시 1: 기본 이스케이프**

입력: `"Hello "World""`

출력: `"Hello \"World\""`

---

**예시 2: 줄바꿈 처리**

입력: `"Line1\nLine2"`

출력: `"Line1\\nLine2"`

---

**예시 3: 백슬래시 처리**

입력: `"C:\Users\Test"`

출력: `"C:\\Users\\Test"`

---

**예시 4: 복합 이스케이프**

입력: `"Path: C:\Test\nFile: "data.json\""`

출력: `"Path: C:\\Test\\nFile: \"data.json\\""`

---

**예시 5: null 입력**

입력: `null`

출력: `""`

---

#### 처리 순서

```
escapeJson(s)
    │
    ├─ null 체크
    │  └─ null → "" 반환
    │
    ├─ 백슬래시 먼저 (중요!)
    │  └─ \ → \\
    │
    ├─ 쌍따옴표
    │  └─ " → \"
    │
    ├─ 줄바꿈
    │  └─ \n → \\n
    │
    ├─ 캐리지 리턴
    │  └─ \r → \\r
    │
    └─ 탭
       └─ \t → \\t
```

**순서가 중요한 이유:**
```java
// 잘못된 순서:
s.replace("\"", "\\\"")
 .replace("\\", "\\\\)  // 이미 추가된 \까지 이스케이프!

// 올바른 순서:
s.replace("\\", "\\\\)  // 먼저 백슬래시
 .replace("\"", "\\\"") // 그 다음 쌍따옴표
```

## 3. ByteBuddyAgent 함수

### 3.1 agentmain()

**함수 시그니처:**
```java
public static void agentmain(String args, Instrumentation inst)
```

**설명:**
Java Agent가 JVM에 attach될 때 호출되는 진입점입니다. Instrumentation API를 초기화하고 HTTP 서버를 시작합니다.

---

#### 입력 (Input)

| 파라미터 | 타입 | 설명 | 예시 | 필수여부 |
|----------|------|------|------|----------|
| `args` | String | Agent 인자 (옵션) | `"opentelemetry"` | 아니오 |
| `inst` | Instrumentation | JVM 계측 API | `Instrumentation` 인스턴스 | 필수 |

**Agent 인자 예시:**
```
# 특정 계측 활성화
"opentelemetry=true"

# 여러 옵션
"opentelemetry=true,auto-instrument=false"
```

---

#### 출력 (Output)

| 타입 | 설명 | 출력 대상 |
|------|------|----------|
| `void` | 반환값 없음 | `System.out`, `System.err` |

**출력 메시지:**
```
[ByteBuddy] premain called with args: {args}
[ByteBuddy] Agent initialized
[ByteBuddy] OpenTelemetry initialized
[ByteBuddy] Auto-instrumenting filters...
[ByteBuddy] HTTP server started on port 10000
```

---

#### 실행 흐름

```
agentmain(args, inst)
    │
    ├─ AtomicBoolean.compareAndSet(false, true)
    │  │
    │  ├─ true (첫 진입)
    │  │  │
    │  │  ├─ instrumentation = inst
    │  │  │
    │  │  ├─ initializeOpenTelemetry()
    │  │  │  ├─ OtlpGrpcSpanExporter 생성
    │  │  │  ├─ Tracer 생성
    │  │  │  └─ SpanProcessor 생성
    │  │  │
    │  │  └─ initializeAgentBuilder()
    │  │     └─ AgentBuilder 생성
    │  │
    │  └─ false (재진입)
    │     └─ 초기화 생략
    │
    ├─ autoInstrumentFilters()
    │  └─ Spring Filters 자동 계측 시도
    │
    ├─ startHttpServer()
    │  └─ 포트 10000에 HTTP 서버 시작
    │
    └─ processAgentArgs(args)
       └─ 인자 파싱 및 처리
```

---

#### 예외 상황

| 상황 | 동작 | 결과 |
|------|------|------|
| OpenTelemetry 초기화 실패 | 에러 로그 출력 | `[ByteBuddy] Failed to initialize OpenTelemetry: ...` |
| HTTP 서버 시작 실패 | 스택 트레이스 출력 | 에러 로그 |
| 이미 초기화됨 | 초기화 생략 | 재진입 안전성 보장 |

---

### 3.2 createEventAdvice()

**함수 시그니처:**
```java
public static String createEventAdvice(String className, String methodName)
```

**설명:**
ByteBuddy를 사용하여 지정된 클래스의 메서드에 EventAdvice를 적용합니다.

---

#### 입력 (Input)

| 파라미터 | 타입 | 설명 | 예시 | 필수여부 |
|----------|------|------|------|----------|
| `className` | String | 타겟 클래스의 완전한 이름 | `"org.springframework.web.servlet.DispatcherServlet"` | 필수 |
| `methodName` | String | 타겟 메서드 이름 | `"doService"` | 필수 |

**입력 제약사항:**
- `className`은 null이거나 비어있으면 안 됨
- `methodName`은 null이거나 비어있으면 안 됨
- 클래스는 클래스패스에 존재해야 함
- 메서드는 클래스에 존재해야 함

---

#### 출력 (Output)

| 타입 | 설명 | 예시 |
|------|------|------|
| `String` | 실행 결과 메시지 | `"SUCCESS: Created event advice for org.springframework.web.servlet.DispatcherServlet.doService"` |

**반환 메시지 포맷:**

| 상황 | 반환 메시지 | 성공여부 |
|------|-------------|----------|
| 성공 | `"SUCCESS: Created event advice for {className}.{methodName}"` | true |
| Agent 미초기화 | `"ERROR: Agent not initialized. Please attach to JVM first."` | false |
| 이미 적용됨 | `"WARNING: EventAdvice already applied to {className}.{methodName}"` | true (경고) |
| 에러 | `"ERROR: {에러 메시지}"` | false |

---

#### 입출력 예시

**예시 1: 정상 적용**

입력:
```json
{
  "className": "org.springframework.web.servlet.DispatcherServlet",
  "methodName": "doService"
}
```

#### 내부 처리 과정

```
createEventAdvice(className, methodName)
    │
    ├─ 1. 초기화 확인
    │  └─ instrumentation == null?
    │     └─ YES → "ERROR: Agent not initialized"
    │
    ├─ 2. 중복 확인
    │  └─ appliedMethods.contains(methodKey)?
    │     └─ YES → "WARNING: Already applied"
    │
    ├─ 3. AgentBuilder 생성
    │  │
    │  ├─ type(named(className))
    │  │  └─ 타겟 클래스 선택
    │  │
    │  ├─ transform(...)
    │  │  └─ 변환 정의
    │  │
    │  ├─ advice(EventAdvice.class)
    │  │  └─ EventAdvice 주입
    │  │
    │  └─ on(named(methodName))
    │     └─ 타겟 메서드 선택
    │
    ├─ 4. 계측 적용
    │  └─ builder.installOn(instrumentation)
    │     └─ JVM에 클래스 재정의 요청
    │
    ├─ 5. 추적
    │  └─ appliedMethods.add(methodKey)
    │
    └─ 6. 성공 메시지 반환
```

---

#### ByteBuddy AgentBuilder DSL

```java
AgentBuilder builder = new AgentBuilder.Default()
    .type(named(className))                              // [1] 타겟 클래스
    .transform((builder2, typeDescription, classLoader, module) ->
        builder2.visit(advice(EventAdvice.class)          // [2] Advice 클래스
            .on(named(methodName))                        // [3] 타겟 메서드
            .to(typeDescription))                         // [4] 적용 대상
    );

builder.installOn(instrumentation);                      // [5] JVM에 적용
```

**단계별 설명:**

| 단계 | 코드 | 설명 |
|------|------|------|
| [1] | `.type(named(className))` | 타겟 클래스를 선택합니다 |
| [2] | `.visit(advice(EventAdvice.class))` | EventAdvice 코드를 주입합니다 |
| [3] | `.on(named(methodName))` | 타겟 메서드를 선택합니다 |
| [4] | `.to(typeDescription)` | 선택된 타입에 적용합니다 |
| [5] | `.installOn(instrumentation)` | JVM에 계측을 적용합니다 |

**계측 전:**
```java

    }
}
```

**계측 후:**

```
    │

    │  └─ attachedPids.add(pid)
    │
    └─ 7. 성공 응답 반환
```
