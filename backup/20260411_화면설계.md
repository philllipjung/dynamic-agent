화면은 3개이고, 탭으로 구분해.
1. 스팬, 속성, 링크를 생성
2. 이벤트를 생성
3. arthas-manager 는 20260410_아르타스.md를 읽고, 화면을 설계

mapParameters, createSpanAttribute 는 적용하지 않고, test1, test2 에 "http://localhost:8080/api/bytebuddy/attach" 호출

userId는 파라미터인데, arthas watch 사용해서, 파라미터를 식별할거야.

ui에서는 소스와 타겟의 대상 클래스, 감지할 메서드 이름, Span에 추가할 속성 이름만 선택
  1. com.test.service.test1.controller.Test1Controller - 대상 클래스 (Fully Qualified Class Name)
  2. test1 - 감지할 메서드 이름
  3. userId - Span에 추가할 속성 이름


# 새로운 PID로 attach
curl -X POST "http://localhost:8080/api/bytebuddy/attach" \
  -H "Content-Type: application/json" \
  -d '{
    "pid": "<new-pid>",
    "instrumentation": "com.test.service.test1.controller.Test1Controller:test1:spanAttribute:userId"
  }'

instrumentation 파라미터의 포맷은 다음과 같이 구성됩니다:

  <클래스 전체 이름>:<메서드 이름>:<액션 타입>:<속성 이름>                                              
                                                                                                          각 부분의 의미:                                                                                       
                                                                                                        
  1. com.test.service.test1.controller.Test1Controller - 대상 클래스 (Fully Qualified Class Name)
  2. test1 - 감지할 메서드 이름
  3. spanAttribute - 수행할 작업 유형 (예: spanAttribute, watchAndCreateSpan 등)
  4. userId - Span에 추가할 속성 이름

  이 포맷은 ByteBuddyAgent에서 파싱하여 사용합니다. 해당 코드를 확인하면 정확한 동작을 알 수 있습니다.     
  Searched for 2 patterns, read 1 file (ctrl+o to expand)                                               

코드를 확인했는데, 파싱 로직은 다음과 같습니다:
  
  String[] parts = instruction.trim().split(":");
  String className = parts[0];        // 클래스                                                           
  String methodName = parts[1];        // 메서드
  String type = parts[2];             // 액션 타입                                                      
  
  // type이 "spanAttribute"일 경우:
  String[] params = parts[3].split(","); // 파라미터들 (콤마로 구분)

  "com.test.service.test1.controller.Test1Controller:test1:spanAttribute:userId" 의미                      
  
  동작: test1() 메서드가 호출될 때 첫 번째 파라미터의 값을 userId라는 이름으로 OpenTelemetry Span에       속성(arthas.attribute.userId)으로 추가합니다.
                                                                                                          
여러 파라미터인 경우:
  Controller:method:spanAttribute:userId,sessionId
  → userId, sessionId 두 개의 속성을 Span에 추가합니다.
