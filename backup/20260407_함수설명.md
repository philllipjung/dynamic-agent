## 1. ArthasManager 함수

### 1.1 연결 관리

#### `attachByPid(String pid)`

**설명:** 프로세스 ID(PID)로 Arthas를 실행 중인 JVM에 연결합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `pid` | String | Yes | 대상 JVM의 프로세스 ID | "12345" |

**반환값:**
| 타입 | 설명 | 예시 |
|------|------|------|
| String | 결과 메시지 | `"SUCCESS: Arthas attached to PID 12345 (Session: arthas-12345-1712345678901), HTTP API on port 8563"` |

**에러:**
| 상황 | 반환값 |
|------|--------|
| PID가 존재하지 않음 | `"ERROR: No JVM found with PID: 12345"` |
| 이미 연결됨 | `"Arthas already attached to PID 12345"` |
| arthas-boot.jar 없음 | `"ERROR: Arthas boot jar not found: /path/to/arthas-boot.jar"` |

**내부 동작:**
```
1. activeSessions에서 중복 확인
   └─> 이미 연결되었으면 메시지 반환

2. Arthas 설치 확인
   └─> ~/.arthas/ 디렉토리 확인
   └─> 없으면 생성

3. arthas-boot.jar 실행
   └─> Windows: cmd.exe /c start /B java -jar arthas-boot.jar --target-ip 127.0.0.1 --http-port 8563 {pid}
   └─> Unix:   java -jar arthas-boot.jar --target-ip 127.0.0.1 --http-port 8563 {pid} &

4. HTTP API 시작 대기 (3초)

5. 세션 생성 및 등록
   └─> activeSessions.put(pid, session)
```

**사용 예시:**
```java
// 1. PID로 직접 연결
String result = ArthasManager.attachByPid("12345");
if (result.startsWith("SUCCESS")) {
    System.out.println("연결 성공!");
}

// 2. jps로 PID 먼저 찾기
// $ jps -l
// 12345 com.example.OrderService
// 12346 com.example.PaymentService

String pid = "12345";
String result = ArthasManager.attachByPid(pid);
```

#### `attachByClassName(String className)`

**설명:** 클래스 이름으로 실행 중인 JVM을 찾아 Arthas를 연결합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `className` | String | Yes | 대상 클래스 이름 (부분 일치 허용) | "com.example.OrderService" 또는 "OrderService" |

**반환값:**
| 타입 | 설명 | 예시 |
|------|------|------|
| String | 결과 메시지 | `"SUCCESS: Arthas attached to PID 12345"` 또는 `"ERROR: No JVM found with class name: UnknownClass"` |

**내부 동작:**
```
1. findPidByClassName()으로 PID 검색
   └─> jps -l 실행
   └─> 클래스 이름 부분 일치 검색

2. PID를 찾으면 attachByPid() 호출
```

**사용 예시:**
```java
// 1. 전체 클래스명으로 검색
String result = ArthasManager.attachByClassName("com.example.OrderService");

// 2. 부분 클래스명으로 검색
String result = ArthasManager.attachByClassName("OrderService");

// 3. 여러 JVM이 실행 중인 경우
// $ jps -l
// 12345 com.example.OrderService
// 12346 com.example.test.OrderService
// └─> 둘 다 "OrderService"를 포함하므로 먼저 발견된 것 반환
```

### 1.2 명령 실행

#### `executeTunnelApiCommand(String pid, String command)`

**설명:** Arthas HTTP API를 통해 명령을 실행하고 결과를 JSON으로 반환합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `pid` | String | Yes | 대상 프로세스 ID | "12345" |
| `command` | String | Yes | Arthas 명령 | "watch com.example.OrderService processOrder params -n 1" |

**반환값:**
| 타입 | 설명 | 예시 |
|------|------|------|
| String | JSON 응답 문자열 | 성공: `{"state": "SUCCEEDED", "body": {"results": [...]}}`<br>실패: `{"error": "API call failed: ..."}` |

**Arthas HTTP API 형식:**
```json
// 요청
POST http://127.0.0.1:8563/api
Content-Type: application/json

{
  "action": "exec",
  "command": "watch com.example.OrderService processOrder params -n 1",
  "execTimeout": "60000"
}

// 응답 (성공)
{
  "state": "SUCCEEDED",
  "body": {
    "results": [
      {
        "type": "watch",
        "value": "params[0]=\"user123\", params[1]=\"ORD-001\""
      }
    ]
  }
}

// 응답 (실패)
{
  "state": "FAILED",
  "error": "Class not found: com.example.UnknownClass"
}
```

**내부 동작:**
```
1. RestTemplate으로 HTTP POST 요청
   └─> URL: http://{host}:{port}/api
   └─> Body: {"action": "exec", "command": "...", "execTimeout": "60000"}
   └─> Timeout: 60초

2. 응답을 그대로 반환 (호출자가 파싱)
```

**사용 예시:**
```java
// 1. watch 명령 실행
String result = ArthasManager.executeTunnelApiCommand("12345",
    "watch com.example.OrderService processOrder params -n 1");
JSONObject json = new JSONObject(result);

// 2. 응답 파싱
if (json.has("body")) {
    JSONObject body = json.getJSONObject("body");
    JSONArray results = body.getJSONArray("results");
    // ...
}
```

#### `watchWithParameters(String className, String methodName, int limit)`

**설명:** 메서드의 파라미터 값을 감시하고 JSON으로 반환합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `className` | String | Yes | 대상 클래스 전체 이름 | "com.example.OrderService" |
| `methodName` | String | Yes | 대상 메서드 이름 | "processOrder" |
| `limit` | int | Yes | 캡처할 호출 횟수 | 5 |

**반환값:**
| 타입 | 설명 | 예시 |
|------|------|------|
| JSONObject | 파라미터 정보 | ```json<br>{<br>  "success": true,<br>  "className": "com.example.OrderService",<br>  "methodName": "processOrder",<br>  "parameters": [<br>    {"index": 0, "name": "userId", "value": "user123"},<br>    {"index": 1, "name": "orderId", "value": "ORD-001"}<br>  ],<br>  "parameterCount": 2<br>}<br>``` |

**에러:**
| 상황 | 반환값 |
|------|--------|
| 클래스 없음 | `{"error": "No running JVM found"}` |
| 명령 실패 | `{"error": "Command execution failed: ..."}` |

**내부 동작:**
```
1. watch 명령 생성
   └─> "watch {className} {methodName} params -n {limit} -x 1"

2. executeTunnelApiCommand() 실행

3. 응답 파싱
   └─> body.results[].value에서 파라미터 추출
   └─> params[N] 패턴 파싱

4. 파라미터 배열 생성
   └─> 각 파라미터의 index, name, value 설정
```

**사용 예시:**
```java
// 1. 파라미터 감시
JSONObject result = ArthasManager.watchWithParameters(
    "com.example.OrderService",
    "processOrder",
    3
);

// 2. 결과 확인
if (result.has("parameters")) {
    JSONArray params = result.getJSONArray("parameters");
    for (int i = 0; i < params.length(); i++) {
        JSONObject param = params.getJSONObject(i);
        int index = param.getInt("index");
        String name = param.getString("name");
        String value = param.getString("value");
        System.out.println("param[" + index + "] " + name + " = " + value);
    }
}
```

### 1.3 검색

#### `searchClass(String pattern)`

**설명:** 패턴으로 클래스를 검색합니다 (sc 명령).

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `pattern` | String | Yes | 검색 패턴 (와일드카드 * 지원) | "com.example.*" 또는 "OrderService" |

| 타입 | 설명 |
|------|------|

```
   └─> "OrderService" → "*OrderService*"

2. sc 명령 실행
   └─> "sc {pattern}"
```

**사용 예시:**
```java
// 1. 와일드카드로 검색
String result = ArthasManager.searchClass("com.example.*");

// 2. 부분 일치로 검색

// → "*OrderService*"로 변환되어 검색
```

### 1.4 비동기 작업

#### `startWatchAsync(String className, String methodName, int limit)`

**설명:** 비동기로 watch 작업을 시작하고 즉시 작업 ID를 반환합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `className` | String | Yes | 대상 클래스 | "com.example.OrderService" |
| `methodName` | String | Yes | 대상 메서드 | "processOrder" |
| `limit` | int | Yes | 캡처 횟수 | 5 |

**반환값:**
| 타입 | 설명 | 예시 |
|------|------|------|
| String | 작업 ID | `"watch-1712345678901"` |

**내부 동작:**
```
1. jobId 생성
   └─> "watch-" + System.currentTimeMillis()

2. 상태를 "RUNNING"으로 설정
   └─> watchStatus.put(jobId, "RUNNING")

3. 백그라운드 스레드에서 watch 실행
   └─> watchExecutor.submit(() -> { ... })

4. 즉시 jobId 반환
```

**사용 예시:**
```java
// 1. 비동기 watch 시작
String jobId = ArthasManager.startWatchAsync(
    "com.example.OrderService",
    "processOrder",
    5
);
System.out.println("Watch 시작: " + jobId);

// 2. 대상 메서드 호출 (애플리케이션에서)
// orderService.processOrder("user123", "ORD-001");

// 3. 결과 조회
Thread.sleep(5000);  // 대기
JSONObject result = ArthasManager.getWatchResult(jobId);
String status = result.getString("status");  // "COMPLETED"
```

---

#### `getWatchResult(String jobId)`

**설명:** 비동기 watch 작업의 결과를 조회합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `jobId` | String | Yes | 작업 ID | "watch-1712345678901" |

**반환값:**
| 타입 | 설명 | 예시 |
|------|------|------|
| JSONObject | 작업 상태 및 결과 | ```json<br>{<br>  "jobId": "watch-1712345678901",<br>  "status": "COMPLETED",<br>  "success": true,<br>  "parameters": [...]<br>}<br>``` |

**상태 값:**
| 상태 | 설명 |
|------|------|
| "RUNNING" | 작업 진행 중 |
| "COMPLETED" | 작업 완료 (data 필드에 결과) |
| "FAILED" | 작업 실패 (error 필드에 이유) |
| "UNKNOWN" | 알 수 없는 작업 ID |

**사용 예시:**
```java
// 1. 결과 조회 (반복)
String jobId = "watch-1712345678901";

while (true) {
    JSONObject result = ArthasManager.getWatchResult(jobId);
    String status = result.getString("status");

    if ("COMPLETED".equals(status)) {
        // 결과 처리
        JSONArray params = result.getJSONArray("parameters");
        break;
    } else if ("FAILED".equals(status)) {
        // 에러 처리
        String error = result.getString("error");
        System.err.println("Error: " + error);
        break;
    }

    Thread.sleep(1000);  // 1초 대기 후 재시도
}
```

```java
// 3. 결과로 Arthas 연결
if (!pid.startsWith("ERROR")) {
    ArthasManager.attachByPid(pid);
}
```

#### `listJavaProcesses()`

**설명:** 모든 Java 프로세스 목록을 반환합니다.

**파라미터:**
없음

**반환값:**
| 타입 | 설명 | 예시 |
|------|------|------|
| List<Map<String, String>> | 프로세스 정보 리스트 | ```[<br>  {"pid": "12345", "className": "com.example.OrderService"},<br>  {"pid": "12346", "className": "com.example.PaymentService"}<br>]``` |

**내부 동작:**
```
1. jps -l 실행 (또는 OS별 대안 명령어)
   └─> Windows: tasklist + wmic
   └─> Unix: ps -eo pid,command | grep java

2. 시스템 프로세스 필터링
   └─> jps, arthas-boot, GradleDaemon 제외

3. 파싱하여 리스트 생성
```

## 2. ByteBuddyAgent 함수

### 2.1 에이전트 수명주기

#### `premain(String args, Instrumentation inst)`

**설명:** JVM 시작 시 에이전트가 로드될 때 호출됩니다 (`-javaagent` 옵션 사용 시).

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `args` | String | No | 에이전트 인자 문자열 | "instrumentation=com.example.Service1:method1:span" |
| `inst` | Instrumentation | Yes | JVM Instrumentation 객체 | JVM에서 자동 전달 |

**반환값:**
void

**내부 동작:**
```
1. 초기화 중복 방지 (AtomicBoolean)
   └─> 이미 초기화되었으면 건너뜀

2. OpenTelemetry 초기화
   └─> initializeOpenTelemetry()
       └─> OtlpGrpcSpanExporter로 Jaeger 연결
       └─> Tracer 생성
       └─> SpanHelper.setTracer(tracer)

3. AgentBuilder 생성
   └─> RedefinitionStrategy.RETRANSFORMATION
   └─> Transformation Listener 등록

4. 에이전트 인자 처리
   └─> processAgentArgs(args)
       └─> 자동 계측 지시 실행

5. 자동 필터 계측
   └─> instrumentFilters()

6. HTTP 서버 시작
   └─> startHttpServer()
       └─> 포트 9999-10010 중 사용 가능한 포트
```

**에이전트 인자 형식:**
```
instrumentation=className1:methodName1:type1;className2:methodName2:type2[,paramNames]

타입:
  - span: Span 생성만
  - spanAttribute: Span + 속성 추가

예시:
  "instrumentation=com.example.Service1:method1:span;com.example.Service2:method2:spanAttribute:param1,param2"
```

**사용 예시:**
```bash
# 1. JVM 시작 시 에이전트 로드
java -javaagent:bytebuddy-agent-1.0.0.jar=instrumentation=com.example.OrderService:processOrder:span \
     -jar your-app.jar

# 2. 자동 계측 지정
java -javaagent:bytebuddy-agent-1.0.0.jar=instrumentation=com.example.*:*:span \
     -jar your-app.jar
```

### 2.2 계측 (Instrumentation)

#### `createSpan(String pid, String className, String methodName)`

**설명:** 메서드에 Span 생성 Advice를 적용합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `pid` | String | No | 프로세스 ID (호환성용, 현재 미사용) | "12345" |
| `className` | String | Yes | 대상 클래스 전체 이름 | "com.example.OrderService" |
| `methodName` | String | Yes | 대상 메서드 이름 | "processOrder" |

**반환값:**
| 타입 | 설명 | 예시 |
|------|------|------|
| String | 결과 메시지 | `"SUCCESS: Created span for com.example.OrderService.processOrder"`<br>실패: `"ERROR: Failed to create span - ..."` |

**에러:**
| 상황 | 에러 메시지 |
|------|------------|
| 에이전트 미초기화 | `"ERROR: Agent not initialized. Please attach to JVM first."` |
| 클래스를 찾을 수 없음 | `"ERROR: Failed to create span - ClassNotFoundException: ..."` |
| 이미 적용됨 | 로그에 "Already applied" 메시지 |

**내부 동작:**
```
1. Instrumentation 확인
   └─> null이면 에러 반환

2. 대상 클래스 로드
   └─> Class.forName(className)
   └─> targetClassLoader = targetClass.getClassLoader()

3. Helper 클래스 주입
   └─> injectHelper(targetClassLoader)
       ├─> HELPER_CLASSES 목록 순회
       ├─> 각 클래스의 바이트코드 로드
       └─> ClassLoader.defineClass()로 주입

4. SpanAdvice 로드
   └─> targetClassLoader.loadClass("com.javaagent.bytebuddy.advices.SpanAdvice")

5. ByteBuddy로 바이트코드 생성
   └─> new ByteBuddy()
       .redefine(targetClass)
       .visit(Advice.to(SpanAdvice)
           .on(named(methodName).and(isMethod()).and(not(isStatic()))))
       .make()
       .getBytes()

6. 재변환 적용
   └─> instrumentation.addTransformer(transformer, true)
   └─> instrumentation.retransformClasses(targetClass)
   └─> instrumentation.removeTransformer(transformer)
```

**사용 예시:**
```java
// 1. Span 생성
String result = ByteBuddyAgent.createSpan(
    null,  // PID (미사용)
    "com.example.OrderService",
    "processOrder"
);

// 2. 결과 확인
if (result.startsWith("SUCCESS")) {
    System.out.println("계측 성공!");
    // 이후 메서드 호출 시 자동으로 Span 생성됨
}
```

#### `createSpanAttributeAdvice(String pid, String className, String methodName, Map<Integer, String> paramMapping)`

**설명:** 메서드 파라미터를 Span 속성으로 캡처하는 Advice를 적용합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `pid` | String | No | 프로세스 ID (호환성용) | "12345" |
| `className` | String | Yes | 대상 클래스 | "com.example.OrderService" |
| `methodName` | String | Yes | 대상 메서드 | "processOrder" |
| `paramMapping` | Map<Integer, String> | Yes | 파라미터 인덱스→이름 매핑 | `{0: "userId", 1: "orderId"}` |

**반환값:**
| 타입 | 설명 | 예시 |
|------|------|------|
| String | 결과 메시지 | `"SUCCESS: Created span attribute advice for com.example.OrderService.processOrder"` |

**내부 동작:**
```
1. 파라미터 매핑 등록
   └─> SpanAdvice.setParameterMapping(className, methodName, paramMapping)
   └─> parameterMappings.put(key, mapping)

2. createSpan()과 동일한 방식으로 계측
   └─> 차이점: SpanAdvice에 매핑 정보가 이미 등록되어 있음
```

**사용 예시:**

```java
mapping.put(1, "orderId");
mapping.put(2, "amount");

// 2. Advice 적용

    null,
    "com.example.PaymentService",
    "processPayment",
    mapping
);

// 3. 런타임에 자동으로 속성 추가됨
// Span attributes:
//   - arthas.attribute.userId: "user123"
```

```
   └─> linkedContexts 순회하며 builder.addLink(context)

3. Span 시작
   └─> builder.startSpan()

4. 현재 컨텍스트로 설정
   └─> span.makeCurrent()
   └─> 이후 Span.current()로 접근 가능

5. SpanHelper 반환
```

**사용 예시:**
```java
// 1. Jaeger에서 이전 Span 검색
List<SpanContext> linkedContexts = JaegerLinkLookupService.findTargetSpanContexts(
    "unknown_service:java",
    "com.example.ServiceB.handleRequest",
    "user123"
);

// 2. Link와 함께 Span 생성
SpanHelper span = SpanHelper.createSpanWithLinks("processOrder", linkedContexts);
span.addAttribute("userId", "user123");
span.complete();
```

### 4.3 ThreadNameHelper

#### `renameThread(Span span)`

**설명:** Span에서 TraceId와 SpanId를 추출하여 스레드명을 변경합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| `span` | Span | Yes | Span 객체 |

**반환값:** void

**스레드명 형식:**
```
{TraceId 앞 7자}{SpanId 앞 6자}{이니셜}{0-9999}

예: 3a5b7c1d4567ABCTSRV1234
    └─┬──┘└─┬─┘└┬┘└─┬┘
      │     │    │   └─ 0-9999 난수
      │     │    └───── 이니셜 (SRV, API, WEB, etc.)
      │     └────────── SpanId 앞 6자
      └──────────────── TraceId 앞 7자
```

**내부 동작:**
```
1. SpanContext 추출
   └─> span.getSpanContext()
       ├─> context.getTraceId()
       └─> context.getSpanId()

2. 스레드명 생성
   └─> formatThreadName(traceId, spanId)

3. 스레드명 변경
   └─> Thread.currentThread().setName(newName)

4. 로그 출력
   └─> 원래 이름, 새 이름, TraceId, SpanId
```

**사용 예시:**
```java
// SpanAdvice에서 사용
Span span = spanHelper.getSpan();
String originalName = Thread.currentThread().getName();  // "reactor-http-nio-3"
ThreadNameHelper.renameThread(span);
// → "3a5b7c1d4567ABCTSRV1234"

// 나중에 복원
Thread.currentThread().setName(originalName);
```

## 5. Advice 클래스 함수

### 5.1 SpanAdvice

#### `setParameterMapping(String className, String methodName, Map<Integer, String> mapping)`

**설명:** 메서드의 파라미터 매핑을 등록합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |
|----------|------|------|------|------|
| `className` | String | Yes | 클래스 전체 이름 | "com.example.OrderService" |
| `methodName` | String | Yes | 메서드 이름 | "processOrder" |
| `mapping` | Map<Integer, String> | Yes | 인덱스→이름 매핑 | `{0: "userId", 1: "orderId"}` |

**저장 위치:**
```java
public static final Map<String, Map<Integer, String>> parameterMappings
    = new ConcurrentHashMap<>();

// 키: "className.methodName"
// 값: {index: paramName, ...}
```

**사용 예시:**
```java
// 1. 매핑 등록
Map<Integer, String> mapping = new LinkedHashMap<>();
mapping.put(0, "userId");
mapping.put(1, "orderId");

SpanAdvice.setParameterMapping(
    "com.example.OrderService",
    "processOrder",
    mapping
);

// 2. onMethodEnter에서 사용
String key = "com.example.OrderService.processOrder";
Map<Integer, String> mapping = parameterMappings.get(key);
// → {0: "userId", 1: "orderId"}
```

---

#### `onMethodEnter(...)` (Advice)

**설명:** 메서드 진입 시 자동으로 호출됩니다.

**자동 주입 파라미터:**
| 파라미터 | 타입 | 설명 | 예시 |
|----------|------|------|------|
| `@Advice.Origin String method` | String | 전체 메서드 서명 | "public void com.example.OrderService.processOrder(String, String)" |
| `@Advice.This Object target` | Object | this 객체 (메서드가 호출된 인스턴스) | OrderService@12345 |
| `@Advice.AllArguments Object[] allArguments` | Object[] | 모든 파라미터 배열 | ["user123", "ORD-001"] |

**반환값:**
| 타입 | 설명 |
|------|------|
| SpanHelper | Span 래퍼 객체 (onMethodExit로 전달) |

**동작 순서:**
```
1. 원래 스레드명 저장
2. className, methodName 추출
3. parameterMappings에서 매핑 조회
4. userId 추출 (매핑에서 "userId"인 파라미터)
5. Jaeger Link 검색 (userId가 있으면)
6. Span 생성 (Links 포함)
7. 스레드명 변경 (TraceId + SpanId)
8. 파라미터를 Span 속성으로 추가
9. SpanHelper 반환
```

---

#### `onMethodExit(...)` (Advice)

**설명:** 메서드 종료 시 자동으로 호출됩니다.

**자동 주입 파라미터:**
| 파라미터 | 타입 | 설명 |
|----------|------|------|
| `@Advice.Enter SpanHelper spanHelper` | SpanHelper | onMethodEnter에서 반환한 값 |

**동작:**
```
1. Span 완료
   └─> spanHelper.complete()
2. 스레드명 복원
   └─> Thread.currentThread().setName(originalName)
```

### 6.1 ParameterMappingService

#### `saveMapping(String className, String methodName, Map<Integer, String> mapping)`

**설명:** 파라미터 매핑을 Redis에 JSON 형식으로 저장합니다.

**파라미터:**
| 파라미터 | 타입 | 필수 | 설명 | 예시 |

**저장 형식:**
```
Redis Key: "paramMapping:{className}:{methodName}"
Redis Value: JSON 문자열
  예: {"0":"userId","1":"orderId"}
```

**사용 예시:**
```java
Map<Integer, String> mapping = new LinkedHashMap<>();

parameterMappingService.saveMapping(
    "com.example.OrderService",
    "processOrder",
    mapping
);

// Redis에 저장됨:
// key: paramMapping:com.example.OrderService:processOrder
// value: {"0":"userId","1":"orderId"}
```

## 7. 데이터 흐름 예시

### 7.1 완전한 계측 흐름

```
1. 사용자가 UI에서 메서드 선택 및 계측 요청
   ↓
   POST /api/arthas/watch
   {
     "className": "com.example.OrderService",
     "methodName": "processOrder",
     "limit": 1
   }
   ↓
2. agent-server → ArthasManager.watchWithParameters()
   ↓
   {"success": true, "parameters": [
     {"index": 0, "value": "user123"},
     {"index": 1, "value": "ORD-001"}
   ]}
   ↓
3. 사용자가 파라미터 매핑
   ↓
   POST /api/arthas/mapParameters
   {
     "className": "com.example.OrderService",
     "methodName": "processOrder",
     "mapping": {"0": "userId", "1": "orderId"}
   }
   ↓
4. 저장
   ├─ Redis: paramMapping:com.example.OrderService:processOrder
   └─ SpanAdvice: parameterMappings.put(key, mapping)
   ↓
5. Span 속성 Advice 적용
   ↓
   POST /api/bytebuddy/createSpanAttribute
   {
     "className": "com.example.OrderService",
     "methodName": "processOrder",
     "parameters": ["userId", "orderId"]
   }
   ↓
6. ByteBuddyAgent.createSpanAttributeAdvice()
   ├─ Helper 클래스 주입
   ├─ SpanAdvice 로드
   └─ 재변환 적용
   ↓
7. 런타임: 메서드 호출
   ↓
   orderService.processOrder("user123", "ORD-001")
   ↓
8. SpanAdvice.onMethodEnter() 자동 호출
   ├─> SpanHelper.createSpanWithLinks()
   │   └─> tracer.spanBuilder("processOrder").startSpan()
   ├─> ThreadNameHelper.renameThread()
   │   └─> Thread.setName("3a5b7c1d4567ABCTSRV1234")
   └─> SpanAttributeHelper.setParameterAttribute()
       ├─> span.setAttribute("arthas.attribute.userId", "user123")
       └─> span.setAttribute("arthas.attribute.orderId", "ORD-001")
   ↓
9. 비즈니스 로직 실행
   ↓
10. SpanAdvice.onMethodExit() 자동 호출
    ├─> SpanHelper.complete()
    │   └─> span.end() → OTLP로 Jaeger 전송
    └─> Thread.currentThread().setName("reactor-http-nio-3")
    ↓
11. Jaeger Collector 수신
    ↓
12. OpenSearch에 저장
    └─> jaeger-span-2026-03-28 인덱스
        {
          "traceID": "3a5b7c1d4567e8f9...",
          "spanID": "0123456789abcdef...",
          "operationName": "processOrder",
          "tags": [
            {"key": "arthas.attribute.userId", "value": "user123"},
            {"key": "arthas.attribute.orderId", "value": "ORD-001"}
          ]
        }
    ↓
13. UI에서 조회 가능
    ├─ GET /api/bytebuddy/spanNames
    └─ GET /api/bytebuddy/spanAttributes?spanName=processOrder
```
