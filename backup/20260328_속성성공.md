# 20260328 - SpanAttributeAdvice ClassLoader 문제 해결

## 🎯 목표
SpanAttributeAdvice를 사용하여 메서드 파라미터를 OpenTelemetry Span의 속성으로 자동 추가

## ❌ 원래 시도한 방식 (실패)

### 아키텍처
```
1. SpanAdvice 적용 → Span 생성 및 makeCurrent()
2. SpanAttributeAdvice 적용 → 현재 span에 속성 추가
```

### 문제 발생
```java
// ByteBuddyController.java
@PostMapping("/createSpanAttribute")
public Map<String, Object> createSpanAttribute(@RequestBody CreateSpanAttributeRequest request) {
    // pid 파라미터 누락으로 HTTP 500 에러!
    com.javaagent.bytebuddy.ByteBuddyAgent.createSpanAttributeAdvice(
        request.getClassName(),  // ❌ pid 없음
        request.getMethodName(),
        paramMapping
    );
}
```

**에러:** `HTTP Status 500 – Internal Server Error`

---

## 🔍 문제 분석

### 문제 1: HTTP 500 에러

**원인:** `createSpanAttribute` 엔드포인트가 `pid` 파라미터를 누락

```java
// Before (line 496)
ByteBuddyAgent.createSpanAttributeAdvice(className, methodName, paramMapping);

// After (수정 완료)
ByteBuddyAgent.createSpanAttributeAdvice(pid, className, methodName, paramMapping);
```

### 문제 2: Java 21 Module System이 Reflection 차단

#### 실패 로그
```
[ByteBuddy] Failed to inject helper classes: Unable to make protected final
java.lang.Class java.lang.ClassLoader.defineClass(java.lang.String,byte[],int,int)
throws java.lang.ClassFormatError accessible: module java.base does not "opens java.lang"
to unnamed module @4bb4de6a
```

**원인:**
- Java 9+부터 module system 도입
- `ClassLoader.defineClass()`가 strongly encapsulated
- Reflection으로 접근하려면 `--add-opens` 필요하지만 동적 attach에서는 사용 불가

### 문제 3: 같은 메서드에 여러 번 Advice 적용

**시도한 instruction:**
```
instrumentation=com.test.service.test1.controller.Test1Controller:test1:span;
                  com.test.service.test1.controller.Test1Controller:test1:spanAttribute:userId
```

**결과:**
- 첫 번째 advice (SpanAdvice) 적용 성공
- 두 번째 advice (SpanAttributeAdvice) 적용 시도 → Helper injection 실패
- Module system이 두 번째 helper injection을 차단

### 문제 4: IllegalAccessError - Private 필드 접근

```
java.lang.IllegalAccessError: class com.test.service.test1.controller.Test1Controller
tried to access private field com.javaagent.bytebuddy.advices.SpanAdvice.parameterMappings
```

**원인:**
- Advice 코드가 inline되어 target class에 복사됨
- Private 필드에 접근 불가

### 문제 5: Method Name 추출 오류

```java
// 입력: "public java.lang.String com.test.service.test1.controller.Test1Controller.test1(java.lang.String)"

// 잘못된 추출 로직:
int lastDot = method.lastIndexOf('.');  // "java.lang.String"의 마지막 dot을 찾음
methodName = method.substring(lastDot + 1);  // "String)" 추출됨!

// 올바른 키: "com.test.service.test1.controller.Test1Controller.test1"
// 실제 키: "com.test.service.test1.controller.Test1Controller.String)"  ❌
```

## ✅ 해결 방법

### 핵심 아이디어: SpanAdvice에 속성 설정 기능 통합

두 개의 별도 advice를 적용하는 대신, **하나의 advice**에서 두 기능을 모두 수행

### 1. SpanAdvice 수정

#### Before (속성 설정 없음)
```java
public class SpanAdvice {
    @Advice.OnMethodEnter(inline = true)
    public static SpanHelper onMethodEnter(@Advice.Origin String method) {
        SpanHelper spanHelper = SpanHelper.createSpan(method);
        return spanHelper;
    }
}
```

#### After (속성 설정 포함)
```java
public class SpanAdvice {
    // Parameter mappings를 저장할 static Map
    public static final Map<String, Map<Integer, String>> parameterMappings = new ConcurrentHashMap<>();

    public static void setParameterMapping(String className, String methodName,
                                           Map<Integer, String> mapping) {
        String key = className + "." + methodName;
        parameterMappings.put(key, mapping);
    }

    @Advice.OnMethodEnter(inline = true)
    public static SpanHelper onMethodEnter(
            @Advice.Origin String method,
            @Advice.This Object target,
            @Advice.AllArguments Object[] allArguments
    ) {
        // 1. Span 생성
        SpanHelper spanHelper = SpanHelper.createSpan(method);

        // 2. 파라미터 속성 추가 (같은 advice 내에서 실행)
        if (spanHelper != null && target != null) {
            String className = target.getClass().getName();
            String methodName = extractMethodName(method);  // 올바른 추출
            String key = className + "." + methodName;

            Map<Integer, String> paramMapping = parameterMappings.get(key);
            if (paramMapping != null && allArguments != null) {
                SpanAttributeHelper attrHelper = new SpanAttributeHelper(spanHelper.getSpan());
                for (Map.Entry<Integer, String> entry : paramMapping.entrySet()) {
                    attrHelper.setParameterAttribute(entry.getValue(),
                                                     allArguments[entry.getKey()]);
                }
            }
        }
        return spanHelper;
    }

    // 올바른 method name 추출
    private static String extractMethodName(String fullMethod) {
        int parenIndex = method.indexOf('(');
        if (parenIndex > 0) {
            String beforeParen = method.substring(0, parenIndex);
            int lastSpace = beforeParen.lastIndexOf(' ');
            int lastDot = beforeParen.lastIndexOf('.');
            if (lastDot > lastSpace) {
                return beforeParen.substring(lastDot + 1);
            }
        }
        return method;
    }
}
```

### 2. Agent Args 처리 수정

#### Before (별도 advice 적용 시도)
```java
private static void processAgentArgs(String args) {
    if ("spanAttribute".equals(type)) {
        // ❌ 별도의 advice 적용 → helper injection 실패
        String result = createSpanAttributeAdvice(null, className, methodName, paramMapping);
    }
}
```

#### After (Parameter mapping만 등록)
```java
private static void processAgentArgs(String args) {
    if ("spanAttribute".equals(type)) {
        // ✅ SpanAdvice에 parameter mapping만 등록
        com.javaagent.bytebuddy.advices.SpanAdvice.setParameterMapping(
            className, methodName, paramMapping
        );
    }
}
```

### 3. 필드 접근성 수정

```java
// Before
private static final Map<String, Map<Integer, String>> parameterMappings = new ConcurrentHashMap<>();

// After
public static final Map<String, Map<Integer, String>> parameterMappings = new ConcurrentHashMap<>();
```

**이유:** Advice가 inline될 때 target class의 코드가 되므로 public이어야 함

## 📊 최종 아키텍처

### 단일 Advice 접근 방식

```
┌─────────────────────────────────────────────────────────┐
│                    SpanAdvice                          │
│  ┌──────────────────────────────────────────────────┐  │
│  │ 1. SpanHelper.createSpan(method)                │  │
│  │    → Span 생성 및 makeCurrent()                 │  │
│  └──────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────┐  │
│  │ 2. parameterMappings.get(key)                  │  │
│  │    → {0: "userId", 1: "sessionId", ...}        │  │
│  └──────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────┐  │
│  │ 3. SpanAttributeHelper.setAttribute()           │  │
│  │    → arthas.attribute.userId = "Alice123"      │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                        ↓
            모든 기능이 하나의 Advice에서 실행
            → Helper injection 1회만 필요
            → Module system 문제 회피
```

## 📝 배운 점

### 1. Java 9+ Module System의 제약

| 문제 | 원인 | 해결 |
|------|------|------|
| `ClassLoader.defineClass()` 접근 불가 | Strong encapsulation | 단일 advice 사용으로 injection 횟수 최소화 |
| Private 필드 접근 불가 | Inline 코드는 target class의 일부가 됨 | Public 필드 사용 |

### 2. ByteBuddy Advice Inline의 특성

```java
@Advice.OnMethodEnter(inline = true)  // ❌ inline = true
public static void method() {
    // 이 코드가 target class에 복사됨
    // Private 필드/메서드 접근 불가
}

@Advice.OnMethodEnter(inline = false)  // ✅ inline = false
public static void method() {
    // Helper class의 메서드 호출
    // Private 접근 가능
}
```

**현재 프로젝트에서는 `inline = true`를 유지하며 public 필드 사용**

### 3. Method Signature 파싱

```java
// 전체 시그니처
"public java.lang.String com.example.MyClass.myMethod(java.lang.String, int)"

// 올바른 파싱 순서
1. '(' 위치 찾기
2. '(' 앞부분 추출 → "com.example.MyClass.myMethod"
3. 마지막 '.' 찾기
4. '.' 뒤부분 추출 → "myMethod"

// 주의: "java.lang.String"의 '.'에 속지 말 것!
```

### 4. 동적 Attach에서의 제약사항

| 작동 환경 | Helper Injection | --add-opens |
|-----------|------------------|-------------|
| 정적 로드 (`-javaagent`) | ✅ 작동 | ❌ 필요 |
| 동적 Attach (Java 8) | ✅ 작동 | ❌ 불필요 |
| 동적 Attach (Java 9+) | ⚠️ 제한적 | ❌ 사용 불가 |

**해결:** Injection 횟수 최소화 → 단일 Advice로 통합

## 🎯 사용법

### REST API로 Span + 속성 적용

```bash
curl -X POST "http://localhost:8080/api/bytebuddy/attach" \
  -H "Content-Type: application/json" \
  -d '{
    "pid": "12345",
    "instrumentation": "com.example.MyClass:myMethod:span;com.example.MyClass:myMethod:spanAttribute:userId,sessionId"
  }'
```

**설명:**
- `:span` → Span 적용
- `:spanAttribute:userId,sessionId` → userId, sessionId 파라미터를 속성으로 등록

## ✅ 성공 요인

1. **단일 Advice 통합** - Helper injection 1회만 수행
2. **Public 필드 사용** - Inline 제약 회피
3. **올바른 method name 추출** - "java.lang.String"의 '.'에 속지 않음
