# 스팬 속성 생성 완전 가이드

**작성일:** 2026-04-08
**목적:** Java Agent에서 스팬 속성을 생성하는 전체 과정 상세 설명

---

## 1. 개요

이 문서는 **Java Agent에서 메서드의 파라미터를 추출하여 OpenTelemetry Span의 속성(Attribute)으로 저장하는 전체 과정**을 단계별로 설명합니다.

### 왜 스팬 속성이 필요한가요?

```
문제 상황:
- 로그: "userId=user123가 주문을 생성했습니다"
- 스팬: "OrderService.createOrder()"
- 문제: 이 스팬이 userId=user123인 것을 어떻게 아나요?

해결:
- 스팬 속성 추가: arthas.attribute.userId = "user123"
- 이제 OpenSearch에서 "userId=user123인 모든 스팬 조회" 가능!
```

## 3. 5단계 상세 설명

### 단계 1: Arthas로 파라미터 추출

**목표:** 메서드의 파라미터 값을 실제로 확인

#### 1.1 API 호출

```bash
POST http://localhost:8080/api/arthas/watch
Content-Type: application/json

{
  "className": "com.test.service.UserIdService",
  "methodName": "setUserId",
  "limit": 5
}
```

#### 1.2 서버 처리 (ArthasController)

```java
// ArthasController.java line 80-111
@PostMapping("/watch")
public Map<String, Object> watch(@RequestBody WatchRequest request) {
    // Arthas HTTP API로 파라미터 추출 요청
    org.json.JSONObject result = ArthasManager.watchWithParameters(
        request.getClassName(),
        request.getMethodName(),
        request.getLimit()
    );

    return Map.of(
        "success", true,
        "data", result.toMap()
    );
}
```

#### 1.3 Arthas HTTP API 통신

```java
// ArthasManager.java
public static JSONObject watchWithParameters(String className, String methodName, int limit) {
    // Arthas HTTP API (포트 8563) 호출
    String url = "http://127.0.0.1:8563/api";
    JSONObject requestBody = new JSONObject();
    requestBody.put("action", "exec");
    requestBody.put("command", "watch " + className + " " + methodName + " '{params}' -n " + limit + " -x 2");
    requestBody.put("execTimeout", "30000");

    // HTTP POST 전송
    HttpResponse response = httpClient.post(url, requestBody.toString());

    // 응답 파싱: body.results[].value 에서 파라미터 추출
    JSONObject jsonResponse = new JSONObject(response.body());
    JSONArray results = jsonResponse.getJSONObject("body").getJSONArray("results");

    // 파라미터 값 추출 (예: ["user123", "session456"])
    return extractParameters(results);
}
```

#### 1.4 결과

```json
{
  "success": true,
  "data": {
    "parameters": ["user123", "session456"],
    "className": "com.test.service.UserIdService",
    "methodName": "setUserId"
  }
}
```

**결과:** 파라미터 값 `[0번째: "user123", 1번째: "session456"]` 확인 완료

### 단계 2: 파라미터 매핑 저장

**목표:** 파라미터 인덱스(0, 1)에 이름(userId, sessionId) 부여

#### 2.1 API 호출

```bash
POST http://localhost:8080/api/arthas/mapParameters
Content-Type: application/json

{
  "className": "com.test.service.UserIdService",
  "methodName": "setUserId",
  "mapping": {
    "0": "userId",
    "1": "sessionId"
  }
}
```

#### 2.2 서버 처리 (ArthasController)

```java
// ArthasController.java line 360-404
@PostMapping("/mapParameters")
public Map<String, Object> mapParameters(@RequestBody MapParameterRequest request) {
    // 1. Map<String, String>을 Map<Integer, String>으로 변환
    Map<Integer, String> paramMapping = new LinkedHashMap<>();
    for (Map.Entry<String, String> entry : request.getMapping().entrySet()) {
        int index = Integer.parseInt(entry.getKey());  // "0" -> 0
        paramMapping.put(index, entry.getValue());     // 0 -> "userId"
    }

    // 2. Redis에 저장
    parameterMappingService.saveMapping(
        request.getClassName(),
        request.getMethodName(),
        paramMapping
    );

    // 3. Advice 캐시에도 저장 (런타임에서 바로 사용)
    SpanAdvice.setParameterMapping(
        request.getClassName(),
        request.getMethodName(),
        paramMapping
    );

    return Map.of("success", true, "message", "매핑 저장 완료");
}
```

#### 2.3 Redis 저장 (ParameterMappingService)

```java
// bytebuddy-agent/.../redis/ParameterMappingService.java line 50-74
public static void saveMapping(String className, String methodName, Map<Integer, String> mapping) {
    try (Jedis jedis = jedisPool.getResource()) {
        // 키 생성: "paramMapping:com.test.service.UserIdService:setUserId"
        String key = KEY_PREFIX + className + ":" + methodName;

        // Map<Integer, String>을 JSON으로 변환
        // {0: "userId", 1: "sessionId"} -> {"0":"userId","1":"sessionId"}
        Map<String, String> stringMap = new HashMap<>();
        for (Map.Entry<Integer, String> entry : mapping.entrySet()) {
            stringMap.put(entry.getKey().toString(), entry.getValue());
        }
        String jsonValue = objectMapper.writeValueAsString(stringMap);

        // Redis에 저장
        jedis.set(key, jsonValue);
    }
}
```

**Redis 저장 결과:**
```

Value: {"0":"userId","1":"sessionId"}
```

### 단계 3: ByteBuddy Agent로 Advice 적용

**목표:** 타겟 JVM의 메서드에 자동으로 실행되는 코드(Advice) 주입

#### 3.1 API 호출

```bash
POST http://localhost:8080/api/bytebuddy/createSpanAttribute
Content-Type: application/json

{
  "pid": "12345",
  "className": "com.test.service.UserIdService",
  "methodName": "setUserId",
  "parameters": ["userId", "sessionId"]
}
```

#### 3.2 서버 처리 (ByteBuddyController)

```java
// ByteBuddyController.java line 439-484
@PostMapping("/createSpanAttribute")
public Map<String, Object> createSpanAttribute(@RequestBody CreateSpanAttributeRequest request) {
    // 1. 파라미터 매핑 생성
    Map<Integer, String> paramMapping = new LinkedHashMap<>();
    int index = 0;
    for (String paramName : request.getParameters()) {
        paramMapping.put(index++, paramName);
    }

    // 2. ByteBuddy Agent 호출
    String result = ByteBuddyAgent.createSpanAttributeAdvice(
        request.getPid(),
        request.getClassName(),
        request.getMethodName(),
        paramMapping
    );

    return Map.of("success", result.startsWith("SUCCESS"), "message", result);
}
```

#### 3.3 ByteBuddy Agent 처리

```java
// bytebuddy-agent/.../ByteBuddyAgent.java line 605-650
public static String createSpanAttributeAdvice(String pid, String className, String methodName,
                                               Map<Integer, String> paramMapping) {
    // 1. Advice 캐시에 매핑 저장
    SpanAdvice.setParameterMapping(className, methodName, paramMapping);

    // 2. 타겟 클래스 로드
    Class<?> targetClass = Class.forName(className);
    ClassLoader targetClassLoader = targetClass.getClassLoader();

    // 3. ByteBuddy로 Advice 적용
    new ByteBuddy()
        .redefine(targetClass)
        .visit(Advice.to(SpanAdvice.class)
            .on(named(methodName))
            .to(targetClassLoader))
        .make()
        .load(targetClassLoader, ClassReloadingStrategy.fromInstrumentation(instrumentation));

    return "SUCCESS: Advice applied to " + className + "." + methodName;
}
```

#### 3.4 적용 결과

```
원래 코드:
public void setUserId(String userId, String sessionId) {
    // ...
}

ByteBuddy 변환 후:
public void setUserId(String userId, String sessionId) {
    SpanHelper spanHelper = SpanAdvice.onMethodEnter(this, userId, sessionId);  // 자동 삽입
    try {
        // 원래 코드
        // ...
    } finally {
        SpanAdvice.onMethodExit(spanHelper);  // 자동 삽입
    }
}
```

### 단계 4: 런타임에 스팬 생성 + 속성 추가

**목표:** 메서드가 실제로 실행될 때 스팬 생성 및 속성 추가

#### 4.1 메서드 호출

```java
// 사용자 애플리케이션
userIdService.setUserId("user123", "session456");
```

#### 4.2 ByteBuddy가 자동으로 Advice 실행

```java
// bytebuddy-advice/.../SpanAdvice.java line 33-129
@Advice.OnMethodEnter(inline = false)
public static SpanHelper onMethodEnter(
        @Advice.Origin String method,
        @Advice.This Object target,
        @Advice.AllArguments Object[] allArguments) {

    // 1. 클래스, 메서드 이름 추출
    String className = target.getClass().getName();  // "com.test.service.UserIdService"
    String methodName = extractMethodName(method);   // "setUserId"
    String key = className + "." + methodName;       // "com.test.service.UserIdService.setUserId"

    // 2. 파라미터 매핑 조회
    Map<Integer, String> paramMapping = parameterMappings.get(key);
    // 결과: {0: "userId", 1: "sessionId"}

    // 3. 스팬 생성
    SpanHelper spanHelper = SpanHelper.createSpanWithLinks(method, Collections.emptyList());

    // 4. 속성 추가
    if (paramMapping != null && allArguments != null) {
        SpanAttributeHelper attrHelper = new SpanAttributeHelper(spanHelper.getSpan());

        for (Map.Entry<Integer, String> entry : paramMapping.entrySet()) {
            int paramIndex = entry.getKey();    // 0, 1
            String paramName = entry.getValue(); // "userId", "sessionId"

            if (paramIndex < allArguments.length) {
                Object paramValue = allArguments[paramIndex];  // "user123", "session456"

                // 속성 추가 (자동으로 "arthas.attribute." prefix 추가)
                attrHelper.setParameterAttribute(paramName, paramValue);
                // 결과: arthas.attribute.userId = "user123"
                //       arthas.attribute.sessionId = "session456"
            }
        }
    }

    return spanHelper;  // onMethodExit에서 사용
}

@Advice.OnMethodExit(inline = false)
public static void onMethodExit(@Advice.Enter SpanHelper spanHelper) {
    if (spanHelper != null && spanHelper.isValid()) {
        spanHelper.complete();  // span.end() 호출
    }
}
```

#### 4.3 SpanAttributeHelper 상세

```java
// bytebuddy-advice/.../helper/SpanAttributeHelper.java
public class SpanAttributeHelper {
    private static final String ATTRIBUTE_PREFIX = "arthas.attribute.";
    private final Span span;

    // 파라미터 속성 추가
    public SpanAttributeHelper setParameterAttribute(String paramName, Object paramValue) {
        String value = convertToString(paramValue);  // "user123"
        setAttribute(paramName, value);              // "userId", "user123"
        return this;
    }

    // 속성 추가 (자동으로 prefix 추가)
    public SpanAttributeHelper setAttribute(String key, String value) {
        String fullKey = ATTRIBUTE_PREFIX + key;  // "arthas.attribute.userId"
        span.setAttribute(fullKey, value);         // OpenTelemetry API
        return this;
    }

    // 스팬 완료
    public void complete() {
        span.setStatus(StatusCode.OK);
        span.end();  // OpenTelemetry Exporter로 전송
    }
}
```

## 8. 참고

### 속성 이름 규칙

- 모든 속성은 **`arthas.attribute.`** prefix가 자동으로 추가됩니다
- 예: `userId` → `arthas.attribute.userId`
- 이 규칙을 통해 OpenSearch에서 쉽게 필터링 가능합니다

### Redis vs 메모리 캐시

| 저장소 | 장점 | 단점 | 용도 |
|--------|------|------|------|
| **Redis** | 영구 저장, 여러 JVM 공유 | Redis 의존성 | 매핑 영구 저장 |
| **메모리 캐시** | 빠른 접근, 별도 의존성 없음 | JVM 재시작 시 소실 | 런타임 빠른 조회 |

실제로는 **두 곳 모두에 저장**하여 안정성과 성능을 모두 확보합니다.

### OpenTelemetry Span 라이프사이클

```
1. Span 생성 (SpanHelper.createSpanWithLinks)
2. 속성 추가 (SpanAttributeHelper.setAttribute)
3. 상태 설정 (setStatus OK)
4. 스팬 종료 (span.end)
5. Exporter로 전송 (자동)
```
