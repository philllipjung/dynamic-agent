## 📋 개요
**날짜**: 2026-03-29
**기능**: Jaeger API를 사용한 실시간 Span Link 생성
**상태**: ✅ 완료 및 테스트 성공
**테스트 환경**: Windows 11, Java 21, OpenTelemetry 1.38.0

## 🎯 테스트 목표

test1과 test2 서비스에서 **같은 userId**를 가진 span 간에 **OpenTelemetry 표준 Span Link**가 정상적으로 생성되는지 검증

### 핵심 검증 항목
1. ✅ ByteBuddy Agent가 두 서비스에 정상 attach 되는지
2. ✅ Span이 정상적으로 생성되는지
3. ✅ Jaeger API를 통해 연결된 span을 찾을 수 있는지
4. ✅ SpanContext를 정상적으로 생성할 수 있는지
5. ✅ OpenTelemetry 표준 `SpanBuilder.addLink()`로 link를 추가할 수 있는지
6. ✅ Jaeger API 응답에서 link가 확인되는지

## 🐛 해결된 문제 (리팩토링)

### 문제 1: Java 21 모듈 시스템 접근 제어
**원인**: inline Advice에서 private 메서드 접근 실패
**해결**: `findLinkedContexts()`를 public으로 변경

### 문제 2: Jaeger API 태그 형식
**원인**: `tag=key=value` 형식 사용 → HTTP 400
**해결**: `tag=key:value` 형식으로 수정

### 문제 3: JSON 파싱 복잡도
**원인**: 정규식이 중첩 JSON 처리 실패
**해결**: 2단계 파싱 (traceID → spanID/operationName)

### 문제 4: SpanContext 클래스 없음
**원인**: `ImmutableSpanContext` 클래스를 찾을 수 없음
**해결**: `SimpleSpanContext` 직접 구현

### 문제 5: 중복 Link 생성
**원인**: Jaeger API 응답에 같은 span이 여러 번 나타남
**해결**: 전역 Set으로 중복 제거
