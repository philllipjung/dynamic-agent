## 🔥 Part 1: 핵심 기능 요약 (빠른 참조용)

### 1. 시스템 개요

```
실행 중인 Java 애플리케이션의 코드 수정 없이 런타임에:
✅ 메서드 파라미터 추출 (Arthas Watch - 비동기)
✅ 메서드 호출 스택 추적 (Arthas Stack - 비동기)
✅ 메서드 내부 호출 트리 분석 (Arthas Trace - 비동기)
✅ Span 생성 및 속성 추가 (ByteBuddy)
✅ HTTP 요청/응답 캡처 (EventAdvice)
✅ 서비스 간 Span 연결 (Span Link)
✅ UI 기반 파라미터 선택 및 Link 설정
```

### 2. 핵심 모듈

| 모듈 | 역할 | 핵심 클래스 |
|------|------|-----------|
| **arthas-manager** | Arthas HTTP API 통신, 비동기 작업 관리 | ArthasManager |
| **bytebuddy-agent** | ByteBuddy Agent, Advice 적용 | ByteBuddyAgent |
| **bytebuddy-advice** | 런타임에 실행될 코드 | SpanAdvice, EventAdvice |
| **agent-server** | REST API 서버 | ArthasController, ByteBuddyController |
| **UI (예정)** | React 기반 웹 대시보드 | 3개 탭 |

### 3. 핵심 데이터 흐름 (비동기 패턴)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  스팬 속성 생성 과정 (UI 통합)                                              │
└─────────────────────────────────────────────────────────────────────────────┘

1. UI에서 Source Method 입력
   ↓
2. [Watch로 파라미터 확인] 클릭
   ↓
3. POST /api/arthas/startWatch
   → jobId 반환 (예: watch-1774679668818)
   ↓
4. 메서드 호출 (사용자가 직접 또는 UI 안내)
   ↓
5. GET /api/arthas/getWatchResult/{jobId}
   → 파라미터 리스트 반환
   ↓
6. UI에서 파라미터 선택 (체크박스)
   ↓
7. POST /api/bytebuddy/attach
   → instrumentation: "Class:method:spanAttribute:param1,param2"
   ↓
8. 런타임에 스팬 생성 + 속성 추가 + 링크 생성
   → arthas.attribute.userId = "user123"
   ↓
9. OpenSearch에 저장
```

## 📦 Part 4: 모듈 상세 분석

### 1. arthas-manager (비동기 기능 추가)

**역할**: Arthas HTTP API 통신, 비동기 작업 관리

**핵심 기능**:
```java
public class ArthasManager {
    // 비동기 Watch
    private static final ExecutorService watchExecutor = Executors.newCachedThreadPool();
    private static final Map<String, String> watchResults = new ConcurrentHashMap<>();
    private static final Map<String, String> watchStatus = new ConcurrentHashMap<>();

    public static String startWatchAsync(String className, String methodName, int limit) {
        String jobId = "watch-" + System.currentTimeMillis();
        watchStatus.put(jobId, "RUNNING");

        // 백그라운드 실행
        watchExecutor.submit(() -> {
            try {
                String result = watchWithParameters(className, methodName, limit);
                watchResults.put(jobId, result);
                watchStatus.put(jobId, "COMPLETED");
            } catch (Exception e) {
                watchResults.put(jobId, "ERROR: " + e.getMessage());
                watchStatus.put(jobId, "FAILED");
            }
        });

        return jobId;
    }

    public static JSONObject getWatchResult(String jobId) {
        JSONObject result = new JSONObject();
        result.put("jobId", jobId);
        result.put("status", watchStatus.getOrDefault(jobId, "UNKNOWN"));

        if ("COMPLETED".equals(status) || "FAILED".equals(status)) {
            result.put("result", watchResults.get(jobId));
        }

        return result;
    }

    // 동일한 패턴으로 Stack, Trace 구현
    public static String startStackAsync(...)
    public static JSONObject getStackResult(String jobId)

    public static String startTraceAsync(...)
    public static JSONObject getTraceResult(String jobId)
}
```

```java
public class EventAdvice {
```

## 🔧 Part 5: 비동기 API 상세

### 5.1 비동기 패턴이 필요한 이유

**문제**: 동기 방식은 메서드가 호출될 때까지 무한 대기

```bash
# 동기 방식 (문제)
POST /api/arthas/watch
→ 60~120초 동안 메서드 호출을 기다림
→ 타임아웃 발생
```

**해결**: 비동기 방식으로 요청-대기-조회 패턴 사용

```bash
# 비동기 방식 (권장)
POST /api/arthas/startWatch  → 즉시 jobId 반환
메서드 호출                    → 실제로 메서드 실행
GET /api/arthas/getWatchResult/{jobId} → 결과 조회
```

### 5.2 Watch 상세 흐름

```bash
# 단계 1: Watch 시작
POST /api/arthas/startWatch
{
  "className": "com.test.service.test1.controller.Test1Controller",
  "methodName": "test1",
  "limit": 1
}

# 응답
{
  "success": true,
  "jobId": "watch-1774679668818",
  "status": "RUNNING",
  "message": "Watch job started. Call the method, then check result with /api/arthas/getWatchResult/watch-1774679668818"
}

# 단계 2: 메서드 호출
curl "http://localhost:8081/test1?userId=0004"

# 단계 3: 결과 조회
GET /api/arthas/getWatchResult/watch-1774679668818

# 응답
{
  "jobId": "watch-1774679668818",
  "success": true,
  "status": "COMPLETED",
  "parameterCount": 3,
  "parameters": [
    {"name": "param0", "index": 0, "type": "Object", "value": "@Object"},
    {"name": "param1", "index": 1, "type": "Object", "value": "@String"},
    {"name": "param2", "index": 2, "type": "Object", "value": "0004"}
  ]
}
```

### 5.3 파라미터 분석

| 파라미터 | 인덱스 | 타입 | 값 | 설명 |
|---------|--------|------|-----|------|
| param0 | 0 | Object | @Object | HttpServletRequest (Spring 자동 주입) |
| param1 | 1 | Object | @String | @RequestParam 애너테이션 |
| param2 | 2 | Object | "0004" | **실제 사용자 입력 값** |

### 테스트 2: Span & Link (양쪽 서비스 적용)

⚠️ **Span Link 생성을 위한 필수 조건: 소스와 타겟 양쪽 모두 적용**

```bash
# 1️⃣ 타겟 서비스 적용 (먼저 실행될 test2)
curl -X POST "http://localhost:8080/api/bytebuddy/attach" \
  -H "Content-Type: application/json" \
  -d '{
    "pid": "<test2-pid>",
    "instrumentation": "com.test.service.test2.controller.Test2Controller:test2:spanAttribute:userId"
  }'

# 2️⃣ 소스 서비스 적용 (나중에 실행될 test1)
curl -X POST "http://localhost:8080/api/bytebuddy/attach" \
  -H "Content-Type: application/json" \
  -d '{
    "pid": "<test1-pid>",
    "instrumentation": "com.test.service.test1.controller.Test1Controller:test1:spanAttribute:userId"
  }'

# 3️⃣ test2 요청 (먼저 실행 - 링크의 대상)
TEST_ID="ASYNC_LINK_$(date +%s)"
curl -H "userId: $TEST_ID" "http://localhost:8082/test2"
# → Span 생성, userId=ASYNC_LINK_... 저장

# 4️⃣ 10초 대기 (Jaeger 인덱싱 필수!)
sleep 10

# 5️⃣ test1 요청 (나중에 실행 - 링크의 출발점)
curl -H "userId: $TEST_ID" "http://localhost:8081/test1"
# → Span 생성, 같은 userId를 가진 test2 span 검색
# → FOLLOWS_FROM 링크 생성!

# 6️⃣ Jaeger에서 링크 확인
curl -s "http://localhost:16686/api/traces?service=unknown_service:java&tag=arthas.attribute.userId:$TEST_ID" | jq '.'
```
