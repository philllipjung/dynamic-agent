## 🔗 동적 Attach/Detach 시스템

### 1. 개요

ByteBuddy Agent는 **실행 중인 JVM에 동적으로 부착**되어 런타임에 코드 계측(instrumentation)을 수행합니다.

**핵심 기술**:
- `com.sun.tools.attach.VirtualMachine` - JVM Attach API
- `java.lang.instrument.Instrumentation` - Bytecode 조작 API
- HTTP API - 원격 제어

---

### 2. Attach 동작 과정

#### 2.1 전체 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│  1. HTTP 요청 수신                                              │
│     POST /api/bytebuddy/attach                                  │
│     { "pid": "12492", "instrumentation": "..." }                │
└─────────────────────────────────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 중복 체크                                                   │
│     if (attachedPids.contains("12492"))                         │
│       → "Already attached" 반환                                 │
└─────────────────────────────────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. Agent JAR 찾기                                              │
│     findAgentJar()                                              │
│     1. ./bytebuddy-agent-1.0.0.jar (없음)                      │
│     2. ./bytebuddy-agent/target/bytebuddy-agent-1.0.0.jar ✅   │
└─────────────────────────────────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. JVM Attach (VirtualMachine API)                            │
│     ┌──────────────────────────────────────────────────────┐   │
│     │ VirtualMachine vm = VirtualMachine.attach("12492")  │   │
│     │ vm.loadAgent(agentJar, "instrumentation=...")       │   │
│     │ vm.detach()  // VM 연결 해제 (Agent는 남음)          │   │
│     └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────────────┐
│  5. Agent 초기화 (target JVM 내부)                              │
│     ByteBuddyAgent.agentmain(agentArgs, instrumentation)        │
│     → Helper classes injection                                 │
│     → createSpan() 호출 (bytecode transform)                   │
│     → Advice 적용                                               │
└─────────────────────────────────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────────────┐
│  6. 추적 관리                                                    │
│     attachedPids.add("12492")                                   │
└─────────────────────────────────────────────────────────────────┘
```

#### 2.2 핵심 코드 분석

**ByteBuddyController.java**:
```java
@PostMapping("/attach")
public Map<String, Object> attach(@RequestBody AttachRequest request) {
    String pid = request.getPid();

    // 중복 체크
    if (attachedPids.contains(pid)) {
        return Map.of("success", true, "message", "Already attached to PID " + pid);
    }

    // Agent JAR 찾기
    String agentJar = findAgentJar();

    // JVM Attach
    VirtualMachine vm = VirtualMachine.attach(pid);
    String agentArgs = "instrumentation=" + request.getInstrumentation();
    vm.loadAgent(agentJar, agentArgs);
    vm.detach();  // VM 연결 해제 (Agent는 계속 실행됨)

    // 추적 목록에 추가
    attachedPids.add(pid);

    return Map.of("success", true, "message", "ByteBuddy Agent attached to PID " + pid);
}
```

**findAgentJar() - JAR 검색 순서**:
```java
private String findAgentJar() {
    // 1. 현재 디렉토리
    File agentJar = new File("bytebuddy-agent-1.0.0.jar");
    if (agentJar.exists()) return agentJar.getAbsolutePath();

    // 2. 빌드 디렉토리 (실제 사용됨) ✅
    agentJar = new File("bytebuddy-agent/target/bytebuddy-agent-1.0.0.jar");
    if (agentJar.exists()) return agentJar.getAbsolutePath();

    // 3. 상위 디렉토리 검색
    // ...
}
```

### 3. Detach 동작

#### 3.1 API Detach

```bash
curl -X DELETE "http://localhost:8080/api/bytebuddy/detach?pid=12492"
```

**동작**:
```java
@DeleteMapping("/detach")
public Map<String, Object> detach(@RequestParam String pid) {
    if (attachedPids.remove(pid)) {
        return Map.of(
            "success", true,
            "message", "Detached from PID " + pid,
            "note", "Agent will be fully detached when JVM terminates"
        );
    }
    return Map.of("success", false, "error", "PID " + pid + " is not attached");
}
```

**중요**:
- **API 호출은 추적 목록에서만 제거**
- **실제 Agent는 JVM 종료 시까지 메모리에 남음**
- Java Instrumentation API는 동적 Agent unload를 지원하지 않음

#### 3.2 자연스러운 Detach

```
JVM 프로세스 종료
    ↓
OS가 프로세스 강제 종료
    ↓
Agent가 포함된 모든 메모리 해제
    ↓
완전한 Detach 상태
```

## 🛠️ 연결 관리 방법

### 1. 신규 서비스 Attach

**시나리오**: 새로운 서비스를 시작하고 Agent를 부착

```bash
# 1. 서비스 시작
cd test-services/test3-service
mvn spring-boot:run

# 2. PID 확인
netstat -ano | grep ":8083"
# 출력: TCP 0.0.0.0:8083 ... PID 30000

# 3. Agent Attach
curl -X POST "http://localhost:8080/api/bytebuddy/attach" \
  -H "Content-Type: application/json" \
  -d '{
    "pid": "30000",
    "instrumentation": "com.test.service.test3.controller.Test3Controller:test3:spanAttribute:userId"
  }'

# 4. 확인
curl http://localhost:8080/api/bytebuddy/attached
```

---

### 2. 서비스 재시작 후 재부착

**시나리오**: 서비스를 재시작했을 때 새 PID로 재부착

```bash
# 1. 기존 서비스 중지
# (Ctrl+C 또는 taskkill //F //PID 12492)

# 2. (선택 사항) 이전 PID Detach

  -H "Content-Type: application/json" \
```

### 3. 종료된 프로세스 Cleanup

**문제**: JVM이 종료되어도 `attachedPids`에 PID가 남아있음

**해결 방법**:

#### 방법 1: 수동 Detach

```bash
# 종료된 PID들을 하나씩 detach
curl -X DELETE "http://localhost:8080/api/bytebuddy/detach?pid=5680"
curl -X DELETE "http://localhost:8080/api/bytebuddy/detach?pid=16100"

# 확인
curl http://localhost:8080/api/bytebuddy/attached
```

#### 방법 2: 서버 재시작

```bash
# agent-server 재시작
# attachedPids가 초기화됨
```

#### 방법 3: 자동 Cleanup (개선 제안)

```java
// ByteBuddyController.java에 추가
@Scheduled(fixedRate = 60000)  // 1분마다 체크
public void cleanupDeadProcesses() {
    Set<String> deadPids = new HashSet<>();
    for (String pid : attachedPids) {
        if (!isProcessAlive(pid)) {
            deadPids.add(pid);
            System.out.println("[ByteBuddyController] Cleaning up dead PID: " + pid);
        }
    }
    attachedPids.removeAll(deadPids);
}

private boolean isProcessAlive(String pid) {
    try {
        // Windows
        ProcessBuilder pb = new ProcessBuilder(
            "tasklist", "/FI", "PID eq " + pid, "/NH", "/FO", "TABLE"
        );
        Process process = pb.start();
        return process.waitFor() == 0;

        // Linux/Mac
        // ProcessBuilder pb = new ProcessBuilder("kill", "-0", pid);
    } catch (Exception e) {
        return false;
    }
}
```

**증상**:
```json
{
  "success": true,

}
```

**해결**:
1. 먼저 Detach: `curl -X DELETE "http://localhost:8080/api/bytebuddy/detach?pid=12492"`
2. 다시 Attach

### 문제 3: "com.sun.tools.attach.AttachNotSupportedException: xxx"

**증상**:
```
com.sun.tools.attach.AttachNotSupportedException: Unable to attach to PID 12492
```

**원인**:
1. PID가 존재하지 않음

**해결**:
```bash
# PID 확인
netstat -ano | grep ":8081"

# 같은 JDK 버전 사용 확인
java -version
```

## 📝 요약

### JAR 파일

| 항목 | 내용 |
|------|------|
| **실제 사용 Agent JAR** | `bytebuddy-agent/target/bytebuddy-agent-1.0.0.jar` |
| **불필요한 복사본** | `agent-server/`, `test-services/*/` 디렉토리의 JAR들 |
| **실행 가능 JAR** | 각 서비스의 `target/*.jar` |

### Attach/Detach

| 작업 | API | 비고 |
|------|-----|------|
| **Attach** | `POST /api/bytebuddy/attach` | PID 또는 className으로 부착 |
| **Detach** | `DELETE /api/bytebuddy/detach?pid=xxx` | 추적 목록에서만 제거 |
| **조회** | `GET /api/bytebuddy/attached` | 현재 부착된 PID 목록 |

### 주의 사항

1. **JVM 종료 시 Agent도 자동 종료됨**
2. **API Detach는 추적 목록에서만 제거**
3. **종료된 PID는 수동으로 Cleanup 필요**
4. **Agent JAR은 항상 최신 버전 유지**
