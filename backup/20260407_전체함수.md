## 1. 프로젝트 소개

### 1.1 이 프로젝트가 무엇인가요?

**Java Agent System**은 실행 중인 Java 애플리케이션의 코드를 **수정하지 않고도** 다음을 할 수 있게 해줍니다:

```
✅ 메서드 실행 추적 (어떤 메서드가 언제 호출되었는지)
✅ 파라미터 값 캡처 (메서드에 어떤 값이 전달되었는지)
✅ 분산 추적 (Span) 생성 (여러 서비스 간 호출 경로)
✅ HTTP 요청/응답 모니터링
✅ 스레드명에 추적 ID 표시 (리눅스 /proc 파일시스템 연동)
✅ Jaeger 링크 생성 (서로 다른 Span 간 연결)
```

### 1.2 왜 필요한가요?

**일반적인 상황:**
```
사용자: "왜 주문이 늦게 처리되나요?"
개발자: "로그를 봐야 하는데... 로그가 분산되어 있어서 찾기 어려워요..."
```

**이 프로젝트를 사용하면:**
```
개발자: "Jaeger에서 userId로 검색하니
       - API Gateway → 주문 서비스 → 재고 서비스 → 결제 서비스
       로의 호출 경로가 모두 보입니다!
       각 단계에서 파라미터 값도 확인할 수 있고요."
```

### 1.3 어떻게 동작하나요?

```
┌─────────────────────────────────────────────────────────────┐
│  실행 중인 Java 애플리케이션                                 │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  TargetApplication.class (원본 코드)                │  │
│  │                                                      │  │
│  │  public void processOrder(String userId,            │  │
│  │                          String orderId) {          │  │
│  │      // 원래 비즈니스 로직                           │  │
│  │  }                                                   │  │
│  └─────────────────────────────────────────────────────┘  │
│                          ↑                                  │
│                          │ ByteBuddy 계측                  │
│                          │                                  │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  런타임에 추가되는 코드 (Advice)                     │  │
│  │                                                      │  │
│  │  @Advice.OnMethodEnter                               │  │
│  │  public static void enter(...) {                    │  │
│  │      // 1. Span 생성                                 │  │
│  │      // 2. 파라미터 캡처                             │  │
│  │      // 3. 스레드명 변경                             │  │
│  │  }                                                   │  │
│  │                                                      │  │
│  │  @Advice.OnMethodExit                                │  │
│  │  public static void exit(...) {                     │  │
│  │      // 4. Span 완료                                 │  │
│  │      // 5. 스레드명 복원                             │  │
│  │  }                                                   │  │
│  └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 2. 기술 이해하기

### 2.1 핵심 기술 스택

| 기술 | 용도 | 설명 |
|------|------|------|
| **ByteBuddy** | 런타임 코드 조작 | 실행 중인 클래스의 바이트코드를 수정하여 코드를 주입 |
| **Arthas** | Java 진단 도구 | 알리바바 오픈소스. 클래스 검색, 메서드 추적, 파라미터 감시 |
| **OpenTelemetry** | 분산 추적 표준 | Span 생성, 추적 데이터 수집 |
| **Jaeger** | 추적 데이터 저장 | 수집된 Span을 저장하고 시각화 |
| **OpenSearch** | 데이터 검색 | Jaeger가 저장한 Span을 쿼리 |
| **Spring Boot** | 웹 프레임워크 | REST API 서버 제공 |

### 2.2 ByteBuddy란?

**ByteBuddy**는 Java의 런타임에 클래스를 수정할 수 있게 해주는 라이브러리입니다.

```java
// 원본 클래스
public class OrderService {
    public void processOrder(String userId) {
        // 비즈니스 로직
    }
}

// ByteBuddy로 계측 후 실제 실행되는 코드
public class OrderService {
    public void processOrder(String userId) {
        // >>> ByteBuddy가 주입한 코드 시작
        SpanHelper span = SpanHelper.createSpan("processOrder");
        span.setAttribute("userId", userId);
        // <<< ByteBuddy가 주입한 코드 끝

        // 원래 비즈니스 로직
        // ...

        // >>> ByteBuddy가 주입한 코드 시작
        span.complete();
        // <<< ByteBuddy가 주입한 코드 끝
    }
}
```

### 2.3 Advice란?

**Advice**는 ByteBuddy에서 메서드의 특정 지점에 실행할 코드를 추가하는 방법입니다.

```java
public class MyAdvice {
    // 메서드 시작 전에 실행
    @Advice.OnMethodEnter
    public static void onEnter(@Advice.Origin String method) {
        System.out.println("메서드 시작: " + method);
    }

    // 메서드 종료 후에 실행
    @Advice.OnMethodExit
    public static void onExit(@Advice.Origin String method) {
        System.out.println("메서드 종료: " + method);
    }
}
```

### 2.4 Arthas란?

**Arthas**는 알리바바가 개발한 Java 진단 도구로, 실행 중인 애플리케이션을 **재시작 없이** 분석할 수 있습니다.

```
[Arthas 기능]

🔍 sc com.example.*      → 클래스 검색
🔍 sm MyClass get*        → 메서드 검색
📊 trace MyClass myMethod  → 호출 스택 추적
👀 watch MyClass myMethod params  → 파라미터 값 확인
📋 stack MyClass myMethod  → 스택 트레이스
```

### 2.5 OpenTelemetry & Jaeger

**OpenTelemetry**는 분산 추적을 위한 표준이고, **Jaeger**는 추적 데이터를 저장하고 시각화하는 도구입니다.

```
[분산 추적 예시]

사용자 요청
    ↓
┌─────────────┐     TraceID: abc123...     ┌─────────────┐
│ API Gateway │ ──────────────────────────→│  Service A  │
│   Span #1   │                           │   Span #2   │
└─────────────┘                           └─────────────┘
                                                 ↓
                                          ┌─────────────┐
                                          │  Service B  │
                                          │   Span #3   │
                                          └─────────────┘

[Jaeger UI에서 보이는 형태]

Trace: abc123...
├─ Span #1: GET /api/orders (API Gateway)
│  └─ attributes: {userId: "user123", http.status_code: 200}
│
├─ Span #2: processOrder (Service A)
│  ├─ attributes: {userId: "user123", orderId: "ORD-001"}
│  ├─ duration: 150ms
│  └─ links: [→ Span #3]
│
└─ Span #3: checkInventory (Service B)
     ├─ attributes: {productId: "PROD-123", quantity: 1}
     └─ duration: 50ms
```

### 3.2 모듈 간 통신

```
1. 사용자 → agent-server
   - REST API로 명령 전달

2. agent-server → Arthas
   - HTTP API로 진단 명령 실행
   - 파라미터 값 조회

3. agent-server → 대상 JVM
   - ByteBuddy 에이전트 동적 연결
   - 계측 지시

4. 대상 JVM → Jaeger
   - OpenTelemetry OTLP로 Span 전송

5. agent-server → OpenSearch
   - Jaeger Span 데이터 쿼리

6. agent-server → Redis
   - 파라미터 매핑 정보 저장
```

### 4.3 bytebuddy-agent

**역할:** 런타임에 클래스를 계측하고 Span 생성하는 에이전트 핵심

```
bytebuddy-agent/
├── ByteBuddyAgent.java     # 에이전트 메인 클래스
└── AttachManager.java      # 동적 연결 관리자
```

**수명주기:**

```
1. JVM 시작 시 (premain)
   └─> 에이전트 초기화
       ├─ OpenTelemetry Tracer 생성
       ├─ AgentBuilder 설정
       └─ HTTP 서버 시작 (포트 9999-10010)

2. 동적 연결 시 (agentmain)
   └─> premain과 동일 + 자동 필터 계측

3. 계측 요청 시
   └─> Helper 클래스 주입
       └─> 대상 클래스 재변환
           └─> Advice 적용
```

**주요 메서드:**

#### (1) Span 생성
```java
String result = ByteBuddyAgent.createSpan(
    null,  // PID (호환성용, 미사용)
    "com.example.OrderService",
    "processOrder"
);
```

**내부 동작:**
```
1. 대상 클래스 로드
   └─> Class.forName("com.example.OrderService")

2. Helper 클래스를 대상 ClassLoader에 주입
   └─> injectHelper(targetClassLoader)
       ├─ SpanHelper
       ├─ SpanAttributeHelper
       ├─ ThreadNameHelper
       └─ EventHelper

3. SpanAdvice를 로드
   └─> targetClassLoader.loadClass("com.javaagent.bytebuddy.advices.SpanAdvice")

4. ByteBuddy로 바이트코드 생성
   └─> ByteBuddy.redefine(targetClass)
       .visit(Advice.to(SpanAdvice).on(named("processOrder")))
       .make()
       .getBytes()

5. 재변환 적용
   └─> instrumentation.addTransformer(transformer, true)
       instrumentation.retransformClasses(targetClass)
```

#### (2) Span 속성 추가
```java
Map<Integer, String> paramMapping = new LinkedHashMap<>();
paramMapping.put(0, "userId");
paramMapping.put(1, "orderId");

String result = ByteBuddyAgent.createSpanAttributeAdvice(
    null,
    "com.example.OrderService",
    "processOrder",
    paramMapping
);
```

#### (3) 이벤트 캡처 (HTTP 요청)
```java
String result = ByteBuddyAgent.createEventAdvice(
    "org.springframework.web.servlet.DispatcherServlet",
    "doService"
);
```

### 4.4 bytebuddy-advice

**역할:** ByteBuddy로 대상 메서드에 주입될 코드

```

├── advices/
│   ├── SpanAdvice.java      # Span 생성 Advice
│   └── EventAdvice.java     # HTTP 이벤트 캡처 Advice
└── helper/
    ├── SpanHelper.java              # Span 라이프사이클 관리
    ├── SpanAttributeHelper.java     # Span 속성 설정
    ├── ThreadNameHelper.java        # 스레드명 변경
    ├── EventHelper.java             # HTTP 요청 정보 출력

```

#### SpanAdvice 상세

**목적:** 메서드 진입/종료 시 자동으로 Span을 생성하고 완료

```java
public class SpanAdvice {
    // 파라미터 매핑 저장소
    public static final Map<String, Map<Integer, String>> parameterMappings
        = new ConcurrentHashMap<>();

    @Advice.OnMethodEnter(inline = false)
    public static SpanHelper onMethodEnter(
        @Advice.Origin String method,        // 전체 메서드 서명
        @Advice.This Object target,          // this 객체
        @Advice.AllArguments Object[] args   // 모든 파라미터
    ) {
        // 1. 원래 스레드명 저장
        String originalThreadName = Thread.currentThread().getName();

        // 2. 클래스명, 메서드명 추출
        String className = target.getClass().getName();
        String methodName = extractMethodName(method);

        // 3. userId 추출
        String userId = null;
        Map<Integer, String> mapping = parameterMappings.get(
            className + "." + methodName
        );
        if (mapping != null) {
            for (Map.Entry<Integer, String> entry : mapping.entrySet()) {
                if ("userId".equals(entry.getValue())) {
                    userId = args[entry.getKey()].toString();
                }
            }
        }

        // 4. Jaeger Link 검색
        List<SpanContext> linkedContexts = Collections.emptyList();
        if (userId != null) {
            linkedContexts = findLinkedContexts(
                className, methodName, userId
            );
        }

        // 5. Span 생성 (Links 포함)
        SpanHelper spanHelper = SpanHelper.createSpanWithLinks(
            method, linkedContexts
        );

        // 6. 스레드명 변경 (TraceId + SpanId)
        if (spanHelper != null && ThreadNameHelper.isTargetThread()) {
            ThreadNameHelper.renameThread(spanHelper.getSpan());
            spanHelper.setOriginalThreadName(originalThreadName);
        }

        // 7. 파라미터를 Span 속성으로 추가
        if (spanHelper != null && mapping != null) {
            SpanAttributeHelper attrHelper =
                new SpanAttributeHelper(spanHelper.getSpan());
            for (Map.Entry<Integer, String> entry : mapping.entrySet()) {
                int index = entry.getKey();
                String paramName = entry.getValue();
                attrHelper.setParameterAttribute(paramName, args[index]);
            }
        }

        return spanHelper;
    }

    @Advice.OnMethodExit(inline = false)
    public static void onMethodExit(@Advice.Enter SpanHelper spanHelper) {
        // 1. Span 완료
        if (spanHelper != null) {
            spanHelper.complete();

            // 2. 스레드명 복원
            String originalName = spanHelper.getOriginalThreadName();
            if (originalName != null) {
                Thread.currentThread().setName(originalName);
            }
        }
    }
}
```

#### SpanHelper 상세

**목적:** OpenTelemetry Span을 쉽게 사용할 수 있는 래퍼

```java
public class SpanHelper {
    private static Tracer tracer;  // OpenTelemetry Tracer
    private final Span span;       // OpenTelemetry Span
    private final Scope scope;     // Span을 현재 컨텍스트로 설정
    private String originalThreadName;  // 원래 스레드명

    // Span 생성 (Links 포함)
    public static SpanHelper createSpanWithLinks(
        String methodName,
        List<SpanContext> linkedContexts
    ) {
        // 1. SpanBuilder 생성
        SpanBuilder builder = tracer.spanBuilder(methodName);

        // 2. Links 추가
        if (linkedContexts != null) {
            for (SpanContext context : linkedContexts) {
                builder.addLink(context);
            }
        }

        // 3. Span 시작
        Span span = builder.startSpan();

        // 4. 현재 컨텍스트로 설정
        Scope scope = span.makeCurrent();

        return new SpanHelper(span, scope);
    }

    // 속성 추가
    public SpanHelper addAttribute(String key, String value) {
        span.setAttribute(key, value);
        return this;  // 체이닝
    }

    // 예외 기록
    public SpanHelper recordException(Throwable throwable) {
        span.recordException(throwable);
        span.setStatus(StatusCode.ERROR, throwable.getMessage());
        return this;
    }

    // Span 완료
    public void complete() {
        span.setStatus(StatusCode.OK);
        span.end();
        scope.close();  // 컨텍스트 정리
    }
}
```

#### ThreadNameHelper 상세

**목적:** 스레드명을 TraceId와 SpanId로 변경하여 /proc에서 추적 가능

```java
public class ThreadNameHelper {
    // 스레드명 형식: {TraceId 7자}{SpanId 6자}{이니셜}{숫자}
    // 예: 3a5b7c1d4567ABCTSRV1234

    public static void renameThread(Span span) {
        SpanContext context = span.getSpanContext();
        String traceId = context.getTraceId();
        String spanId = context.getSpanId();

        // 포맷: {TraceId 앞 7자}{SpanId 앞 6자}{이니셜}{0-9999}
        String newName = String.format("%s%s%s%04d",
            traceId.substring(0, 7),      // TraceId 앞 7자
            spanId.substring(0, 6),       // SpanId 앞 6자
            "SRV",                        // 이니셜
            ThreadLocalRandom.current().nextInt(10000)
        );

        Thread.currentThread().setName(newName);

        // 로그 출력
        System.out.println("[ThreadName] Renamed: " +
            Thread.currentThread().getName() +
            " (TraceId: " + traceId + ", SpanId: " + spanId + ")"
        );
    }
}
```

```java
public class OpenSearchManager {

              "size": 0,

              "aggs": {
                "tag_keys": {
                  "nested": {"path": "tags"},
                  "aggs": {
                    "keys": {
                      "terms": {"field": "tags.key", "size": 1000}
                    }
                  }
                }
              }
            }
            """.formatted(spanName);
        // → ["arthas.attribute.userId", "http.method", "http.status_code", ...]
```

### 5.1 Span 생성 흐름

```

   └─> SpanAdvice.onMethodEnter()

       │       └─> Span 시작

   └─> Span이 jaeger-span-YYYY-MM-DD 인덱스에 저장
```

```bash
4. 스레드명 형식 확인
   $ cat /proc/{pid}/task/{tid}/comm
   3a5b7c1d4567ABCTSRV1234  ← 16자 이상이면 잘릴 수 있음
   └─> Linux 커널 제한: 16자 (COMMLEN)
```

## 9. 요약

### 9.1 핵심 포인트

```
1. 실행 중인 애플리케이션을 수정하지 않고 계측 가능
   └─> ByteBuddy의 런타임 클래스 재변환

2. OpenTelemetry 표준을 사용하여 Jaeger와 통합
   └─> Jaeger UI에서 시각화

3. Arthas를 통해 실시간 파라미터 확인 가능
   └─> UI에서 파라미터 이름을 쉽게 매핑

4. /proc 파일시스템과 연동하여 리눅스에서 추적 ID 확인
   └─> top, htop에서 스레드명으로 TraceId 확인

5. OpenSearch를 통해 유연한 쿼리 가능
   └─> 사용자 정의 속성으로 검색 및 집계
```

### 9.2 퀵 레퍼런스

| 작업 | API | 설명 |
