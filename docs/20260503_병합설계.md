## 2. 아키텍처 비교 분석

### 2.1 Instrumentation 방식

| 기능 | Agent 프로젝트 | Spark 프로젝트 |
|------|----------------|----------------|
| Instrumentation 도구 | ByteBuddy | ByteBuddy (OTEL Extension) |
| Advice 방식 | SpanAdvice | SparkJobInstrumentation, OpenSearchBatchInstrumentation |
| OTEL SDK 사용 | Manual span creation | Custom OpenTelemetrySdk |
| Trace 전파 | JaegerLinkLookupService | traceparent system property |

### 2.2 Span 생성 방식

**Agent 프로젝트:**
```java
// SpanAdvice.java
Span span = tracer.spanBuilder(spanName)
    .setParent(Context.current())
    .startSpan();
```

**Spark 프로젝트:**
```java
// SparkJobInstrumentation.java
// 1. Custom SDK 생성
OtlpGrpcSpanExporter spanExporter = OtlpGrpcSpanExporter.builder()
    .setEndpoint(otelEndpoint)
    .build();

// 2. traceparent 파싱
String traceParent = System.getProperty("traceparent");
String[] parts = traceParent.split("-");
String traceId = parts[1];

// 3. Parent context 생성
SpanContext parentSpanContext = SpanContext.create(traceId, spanId, ...);
Span parentSpan = Span.wrap(parentSpanContext);
Context parentContext = Context.root().with(parentSpan);

// 4. Span 생성 (parent context로)
span = tracer.spanBuilder("spark.wordcount.job")
    .setParent(parentContext)  // ← 같은 Trace ID
    .startSpan();
```

### 2.3 Span Link 방식

**Agent 프로젝트:**
- Redis에 span link 정보 저장
- SpanLinkManager, SpanLinkService

**Spark 프로젝트:**
- OpenSearch에서 transaction 검색
- SpanContextInfo로 trace 정보 전달
- `span.addLink(parentSpanContext)`로 FOLLOWS_FROM 관계 생성

## 8. 이슈 및 해결 방안

### 8.1 클래스로더 충돌

**이슈:** Spark Job 실행 시 복잡한 클래스로더 구조로 인한 OTEL 클래스 충돌

**해결:**
1. Custom OpenTelemetrySdk 사용 (SparkJobInstrumentation 기존 방식 유지)
2. ByteBuddy shade로 패키지 relocation
3. OTEL Extension을 별도 agent로 유지 (Option A)

### 8.2 traceparent 전달

**이슈:** Microservice → Spark Job 간 traceparent 전달 방식

**해결:**
1. SparkSubmitInstrumentation을 SparkOtelAgent에 통합
2. `System.setProperty("traceparent", ...)`로 전달
3. SparkJobInstrumentation에서 `System.getProperty("traceparent")`로 읽기

### 8.3 OpenSearch 검색

**이슈:** 기존 JaegerLinkLookupService와 OpenSearchBatchProcessor 중복

**해결:**
1. JaegerLinkLookupService를 통합 검색 서비스로 확장
2. transaction 태그 검색 기능 추가
3. 두 프로젝트 모두에서 사용
