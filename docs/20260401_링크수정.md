## 🔥 트러블슈팅: 발생한 문제

### ❌ 문제: Instrumentation이 적용되지 않음

#### 발생 상황
```
[ByteBuddy] Auto-instrumenting: com.test.service.test1.controller.Test1Controller.test1 (type: spanAttribute)
[ByteBuddy] Parameter mapping registered for com.test.service.test1.controller.Test1Controller.test1

# 요청 전송 후...
# (아무런 로그 없음)
# ADVICE ENTER 메시지 없음
# SPAN CREATED 메시지 없음
```

#### 원인 분석

**코드 분석** (`ByteBuddyAgent.java:241-255`):

```java
if ("span".equals(type)) {
    // ✅ span 타입: instrumentation 적용
    String result = createSpan(null, className, methodName);
    System.out.println("[ByteBuddy] " + result);
} else if ("spanAttribute".equals(type) && parts.length >= 4) {
    // ❌ spanAttribute 타입: 매핑만 등록, instrumentation 안 함!
    String[] params = parts[3].split(",");
    java.util.Map<Integer, String> paramMapping = new java.util.LinkedHashMap<>();
    int index = 0;
    for (String param : params) {
        paramMapping.put(index++, param.trim());
    }
    SpanAdvice.setParameterMapping(className, methodName, paramMapping);
    System.out.println("[ByteBuddy] Parameter mapping registered for " + className + "." + methodName);
    // ⚠️ createSpan() 호출 없음!
}
```

**문제점**:
- `span` 타입: parameter mapping 없이 span만 생성
- `spanAttribute` 타입: parameter mapping만 등록하고 **instrumentation 안 함**
- 실제로는 `spanAttribute` 타입도 instrumentation이 적용되어야 함!

**설계 의도**:
| 타입 | 의도 | 실제 동작 |
|------|------|-----------|
| `span` | span만 생성 | ✅ 정상 작동 |
| `spanAttribute` | span + parameter 속성 | ❌ 매핑만 등록, 미작동 |

#### 해결책

**수정된 코드**:

## 🎨 수정된 코드

### ByteBuddyAgent.java (전체)

**위치**: `bytebuddy-agent/src/main/java/com/javaagent/bytebuddy/ByteBuddyAgent.java`

**수정된 메서드**: `processAgentArgs(String args)`

```java
private static void processAgentArgs(String args) {
    if (args != null && args.startsWith("instrumentation=")) {
        // Support multiple instrumentations separated by ;
        String[] instructions = args.substring("instrumentation=".length()).split(";");
        for (String instruction : instructions) {
            String[] parts = instruction.trim().split(":");
            if (parts.length >= 3) {
                String className = parts[0];
                String methodName = parts[1];
                String type = parts[2];

                System.out.println("[ByteBuddy] Auto-instrumenting: " + className + "." + methodName + " (type: " + type + ")");

                if ("span".equals(type)) {
                    String result = createSpan(null, className, methodName);
                    System.out.println("[ByteBuddy] " + result);
                } else if ("spanAttribute".equals(type) && parts.length >= 4) {
                    // Parse parameter names: userId,sessionId
                    String[] params = parts[3].split(",");
                    java.util.Map<Integer, String> paramMapping = new java.util.LinkedHashMap<>();
                    int index = 0;
                    for (String param : params) {
                        paramMapping.put(index++, param.trim());
                    }
                    // Register parameter mapping in SpanAdvice
                    SpanAdvice.setParameterMapping(className, methodName, paramMapping);
                    System.out.println("[ByteBuddy] Parameter mapping registered for " + className + "." + methodName);

                    // 🆕 CRITICAL FIX: Also apply instrumentation (createSpan) for spanAttribute type
                    String result = createSpan(null, className, methodName);
                    System.out.println("[ByteBuddy] " + result);
                }
            }
        }
    }
}
```

## 📚 기술적 배경

### Instrumentation 타입 설계

**원래 설계 의도**:

| 타입 | 용도 | Parameter Mapping |
|------|------|-------------------|
| `span` | 기본 span 생성 (파라미터 속성 없음) | 없음 |
| `spanAttribute` | Span + 파라미터 속성 추출 | 필요 |

**구현 로직**:

```
┌─────────────────────────────────────────────────────────────────┐
│ span 타입                                                        │
├─────────────────────────────────────────────────────────────────┤
│ 1. createSpan() 호출                                            │
│    → Bytecode transform                                        │
│    → Advice 적용                                                 │
│    → Span 생성 (속성 없음)                                       │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ spanAttribute 타입 (수정 전)                                    │
├─────────────────────────────────────────────────────────────────┤
│ 1. Parameter mapping 등록                                      │
│    → SpanAdvice.setParameterMapping(...)                         │
│ 2. ❌ createSpan() 호출 안 함                                   │
│    → Bytecode transform 안 일어남                               │
│    → Advice 미실행                                             │
│    → Span 생성 안 됨                                            │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ spanAttribute 타입 (수정 후) ✅                                 │
├─────────────────────────────────────────────────────────────────┤
│ 1. Parameter mapping 등록                                      │
│    → SpanAdvice.setParameterMapping(...)                         │
│ 2. ✅ createSpan() 호출 추가!                                   │
│    → Bytecode transform                                        │
│    → Advice 적용                                                 │
│    → Span 생성 + 속성 추출                                       │
└─────────────────────────────────────────────────────────────────┘
```

### ByteBuddy Instrumentation 프로세스

**createSpan() 메서드의 역할**:

1. **Target Class 로드**
   ```java
   Class<?> targetClass = Class.forName(className);
   ClassLoader targetClassLoader = targetClass.getClassLoader();
   ```

2. **Helper Injection**
   ```java
   injectHelper(targetClassLoader);
   // SpanAdvice를 target ClassLoader로 주입
   ```

3. **Advice 적용 (Bytecode Transform)**
   ```java
   byte[] transformedBytes = new ByteBuddy()
       .redefine(targetClass)
       .visit(Advice.to(adviceClass)
           .on(ElementMatchers.named(methodName)))
       .make()
       .getBytes();
   ```

4. **클래스 재정의**
   ```java
   instrumentation.redefineClasses(new ClassDefinition(targetClass, transformedBytes));
   ```

**중요**: `createSpan()`이 호출되지 않으면 이 전체 과정이 실행되지 않아 Advice가 적용되지 않음!

### 1. Instrumentation 타입별 처리 로직

**교훈**: 모든 instrumentation 타입이 공통으로 수행해야 할 작업이 있을 수 있음

```java
// 좋은 설계
if ("span".equals(type) || "spanAttribute".equals(type)) {
    // 공통: createSpan() 호출 (instrumentation 적용)
    createSpan(null, className, methodName);

    // 개별: 타입별 추가 처리
    if ("spanAttribute".equals(type)) {
        // parameter mapping 등록
    }
}
```

### 2. 디버깅 팁

**로그 추가의 중요성**:
```java
// Instrumentation 적용 여부 확인 로직 필요
System.out.println("[ByteBuddy] *** ADVICE APPLIED TO " + className + "." + methodName + " ***");
System.out.println("[ByteBuddy] Transformed bytecode created, size: " + transformedBytes.length);
```

**확인할 로그**:
1. `Auto-instrumenting:` - instrumentation 요청
2. `ADVICE APPLIED TO` - bytecode transform 완료
3. `Transformed bytecode created` - 클래스 재정의 완료
4. `>>> ADVICE ENTER:` - 실제 런타임 실행

### 3. 테스트 접근

**단계별 확인**:
1. Agent attach 성공 확인

## 🔮 향후 개선 사항

### 1. 코드 리팩토링

**현재**: 중복된 `createSpan()` 호출
```java

    createSpan(...);
} else if ("spanAttribute".equals(type)) {
    ...
    createSpan(...);  // 중복
}
```

**개선안**:
```java
// 공통 로직 먼저 실행
createSpan(null, className, methodName);

    SpanAdvice.setParameterMapping(...);
}
```

### 2. 타입 시스템 개편

**현재**: 문자열 비교 (`if ("span".equals(type))`)

```java
public enum InstrumentationType {

// 사용

}
```
