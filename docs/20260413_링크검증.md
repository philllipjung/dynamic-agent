## 🎯 검증 배경

### 검증 목적
2026-04-01에 수정된 `spanAttribute` 타입 instrumentation 버그가 실제로 정상 작동하는지 검증

### 수정 내용 요약 (20260401_링크수정.md)
**문제**: `spanAttribute` 타입 instrumentation 시 parameter mapping만 등록되고 실제 instrumentation이 적용되지 않음

**해결**: `ByteBuddyAgent.java` Line 270-272에 `createSpan()` 호출 추가

```java
// 수정 전
} else if ("spanAttribute".equals(type) && parts.length >= 4) {
    SpanAdvice.setParameterMapping(className, methodName, paramMapping);
    System.out.println("[ByteBuddy] Parameter mapping registered...");
    // ❌ createSpan() 호출 없음
}

// 수정 후
} else if ("spanAttribute".equals(type) && parts.length >= 4) {
    SpanAdvice.setParameterMapping(className, methodName, paramMapping);
    System.out.println("[ByteBuddy] Parameter mapping registered...");
    // ✅ CRITICAL FIX: Also apply instrumentation
    String result = createSpan(null, className, methodName);
    System.out.println("[ByteBuddy] " + result);
}
```

## 🔍 트러블슈팅: 발생한 이슈와 해결

### 이슈 1: OpenSearch 인덱스 조회 실패

**증상**:
```bash
$ curl -X POST "http://localhost:9200/jaeger-span-*/_search" ...
{"hits":{"total":{"value":0},"hits":[]}}
```

**원인**: 와일드카드 쿼리 사용 시 인덱스가 바로 반영되지 않음 (OpenSearch 인덱싱 지연)

**해결**:
1. Jaeger API를 직접 호출하여 실시간 데이터 확인
2. 특정 날짜 인덱스(`jaeger-span-2026-03-29`)로 쿼리 시도
3. 결국 Jaeger API(`http://localhost:16686/api/traces`) 사용으로 성공

**교훈**: OpenSearch는 실시간 인덱싱이 아니므로, 즉시 확인이 필요한 경우 Jaeger API를 사용

---

$ curl ... | jq '.hits.hits[] | ...'

### 1. Instrumentation 프로세스 완전 이해

**spanAttribute 타입의 올바른 처리 흐름**:
```
1. Agent 부착 요청 (api/bytebuddy/attach)
   ↓
2. processAgentArgs() 호출
   ↓
3. Parameter mapping 등록 (SpanAdvice.setParameterMapping)
   ↓
4. 🆕 createSpan() 호출 (Bytecode transform 시작)
   ↓
5. Helper class injection
   ↓
6. Advice 적용 (ByteBuddy.redefine())
   ↓
7. 클래스 재정의 (Instrumentation.redefineClasses)
   ↓
8. 런타임에 Advice 실행
   ↓
9. Span 생성 + Link 생성 + 속성 추출
```

**핵심 수정**: Step 4가 누락되어 있었음을 발견하고 추가

### 2. 로그 기반 디버깅의 중요성

**확인해야 할 로그 순서**:
1. `[ByteBuddy] Auto-instrumenting:` - Instrumentation 요청 수신
2. `[SpanAdvice] Parameter mapping registered:` - 매핑 등록
3. `[ByteBuddy] SUCCESS: Created span for:` - 🆕 **적용 완료 (수정된 코드)**
4. `>>> ADVICE ENTER:` - 런타임 실행 시작
5. `[JaegerLink] Found X matching spans:` - Link 조회 결과
6. `>>> Set attribute:` - 속성 추출
7. `<<< SPAN COMPLETED:` - Span 완료

### 3. Link 생성 메커니즘

**Jaeger API 조회 로직**:
```java
// SpanAdvice.java
String jaegerUrl = "http://localhost:16686/api/traces?"
    + "service=unknown_service:java&"
    + "tag=arthas.attribute.userId:" + userId + "&"
    + "limit=10";

// 응답 파싱
JSONObject response = httpClient.get(jaegerUrl);
JSONArray traces = response.getJSONArray("data");

// 각 trace의 spans 확인
for (int i = 0; i < traces.length(); i++) {
    JSONArray spans = traces.getJSONObject(i).getJSONArray("spans");
    // 이전 span 찾기
    if (matched) {
        createLink(traceID, spanID);  // FOLLOWS_FROM link 생성
    }
}
```
