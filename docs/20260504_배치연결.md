## 1. 개요

### 1. 목적

OpenSearch에 저장된 원본 Trace를 검색하고, 배치 처리 시 새로운 Span을 생성하며 **FOLLOWS_FROM 링크**로 원본 Trace와 연결

### 1. 핵심 기능

- **OpenSearch Trace 검색**: transaction 태그 또는 Trace ID로 원본 Span 검색
- **Span Link 생성**: FOLLOWS_FROM 관계로 배치 Span과 원본 Span 연결
- **자동 계측**: ByteBuddy Advice로 배치 처리 메서드에 자동으로 Span 생성

### 1. 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 원본 트랜잭션 완료                                            │
```

## 2. 핵심 컴포넌트

### 2.1 BatchInstrumentation

**위치**: `spark-integration/batch-processor/src/main/java/com/javaagent/spark/batch/BatchInstrumentation.java`

**역할**: 배치 처리 메서드에 진입하여 OpenSearch에서 원본 Trace를 검색하고 Span Link 생성

**핵심 로직**:
```java
@Advice.OnMethodEnter
public static void onMethodEnter(
        @Advice.Argument(0) String transactionNumber,
        @Advice.Argument(1) String batchType,
        @Advice.Argument(2) Object spanContextInfo,
        @Advice.Local("batchSpan") Span batchSpan) {

    getCustomOpenTelemetry();

    // 1. Reflection으로 Trace 정보 추출
    String traceId = extractTraceId(spanContextInfo);
    String spanId = extractSpanId(spanContextInfo);

    // 2. 원본 SpanContext 재구성
    SpanContext parentSpanContext = SpanContext.createFromRemoteParent(
        traceId, spanId,
        TraceFlags.getSampled(),
        TraceState.getDefault()
    );

    // 3. Batch Span 생성 + Link 추가
    SpanBuilder spanBuilder = customTracer.spanBuilder("batch.execute")
            .setSpanKind(SpanKind.INTERNAL);

    if (parentSpanContext.isValid()) {
        spanBuilder.addLink(parentSpanContext);  // FOLLOWS_FROM 관계
    }

    batchSpan = spanBuilder.startSpan();

    // 4. Link 태그 추가
    batchSpan.setAttribute("batch.linked.trace.id", traceId);
    batchSpan.setAttribute("batch.linked.span.id", spanId);
    batchSpan.setAttribute("transaction", transactionNumber);
    batchSpan.setAttribute("batch.type", batchType);
}
```

### 2.2 OpenSearchBatchProcessor

**위치**: `spark-integration/batch-processor/src/main/java/com/javaagent/spark/batch/OpenSearchBatchProcessor.java`

**역할**: 순수 비즈니스 로직 - 배치 큐를 읽고 OpenSearch 검색 후 배치 실행

**핵심 로직**:
```java
public void startProcessing() throws InterruptedException {
    while (true) {
        // 1. 배치 큐 파일 읽기
        List<Map<String, String>> batches = readBatchQueue();

        // 2. PENDING 상태인 배치 처리
        for (Map<String, String> batch : batches) {
            if ("PENDING".equals(batch.get("status"))) {
                String transactionNumber = batch.get("transactionNumber");
                String batchType = batch.get("batchType");

                // 3. OpenSearch에서 원본 Trace 검색
                SpanContextInfo spanContextInfo = searchTransactionInOpenSearch(transactionNumber);

                // 4. 배치 실행 (BatchInstrumentation에 의해 자동 계측됨)
                processBatch(transactionNumber, batchType, spanContextInfo);
            }
        }

        // 5. 5초마다 폴링
        Thread.sleep(5000);
    }
}
```

### 2.3 Span Link 개념

**FOLLOWS_FROM 관계**:
```
원본 Span (Trace A, Span A)
    ↓ FOLLOWS_FROM
배치 Span (Trace B, Span B)
```

**특징**:
- 비동기 작업에서 사용
- "A가 B를 따른다" (B follows A)
- 독립적인 Trace ID를 가짐
- Jaeger UI에서 점선선으로 연결 표시

## 3. 소스 구조

### 3.1 BatchInstrumentation 상세

```java
public class BatchInstrumentation {

    public static volatile OpenTelemetrySdk customOpenTelemetry;
    public static volatile Tracer customTracer;

    /**
     * 배치 처리 메서드에 Instrumentation 설치
     */
    public static void install(Instrumentation instrumentation) {
        log.info("[BatchInstrumentation] Installing batch processing auto-instrumentation...");

        new AgentBuilder.Default()
                .type(named("com.javaagent.spark.batch.OpenSearchBatchProcessor"))
                .transform(new AgentBuilder.Transformer() {
                    @Override
                    public DynamicType.Builder<?> transform(
                            DynamicType.Builder<?> builder,
                            TypeDescription typeDescription,
                            ClassLoader classLoader,
                            JavaModule module,
                            ProtectionDomain protectionDomain) {

                        return builder.visit(
                            Advice.to(BatchProcessAdvice.class)
                            .on(named("processBatch"))  // processBatch 메서드에 Advice 적용
                        );
                    }
                })
                .installOn(instrumentation);

        log.info("[BatchInstrumentation] Batch processing instrumentation installed successfully!");
    }

    /**
     * 배치 처리 Advice
     */
    public static class BatchProcessAdvice {

        @Advice.OnMethodEnter
        public static void onMethodEnter(
                @Advice.Argument(0) String transactionNumber,
                @Advice.Argument(1) String batchType,
                @Advice.Argument(2) Object spanContextInfo,
                @Advice.Local("batchSpan") Span batchSpan) {

            getCustomOpenTelemetry();

            // Reflection으로 Trace 정보 추출
            String traceId = extractTraceId(spanContextInfo);
            String spanId = extractSpanId(spanContextInfo);

            // 원본 SpanContext 재구성
            SpanContext parentSpanContext = SpanContext.createFromRemoteParent(
                traceId, spanId,
                TraceFlags.getSampled(),
                TraceState.getDefault()
            );

            // Batch Span 생성 + Link 추가
            SpanBuilder spanBuilder = customTracer.spanBuilder("batch.execute")
                    .setSpanKind(SpanKind.INTERNAL);

            if (parentSpanContext.isValid()) {
                spanBuilder.addLink(parentSpanContext);  // FOLLOWS_FROM
            }

            batchSpan = spanBuilder.startSpan();
            batchSpan.setAttribute("batch.linked.trace.id", traceId);
            batchSpan.setAttribute("batch.linked.span.id", spanId);
            batchSpan.setAttribute("transaction", transactionNumber);
            batchSpan.setAttribute("batch.type", batchType);
        }

        @Advice.OnMethodExit(onThrowable = Throwable.class)
        public static void onMethodExit(
                @Advice.Local("batchSpan") Span batchSpan,
                @Advice.Local("scope") io.opentelemetry.context.Scope scope,
                @Advice.Thrown Throwable throwable) {

            if (batchSpan == null) {
                return;
            }

            try {
                if (throwable != null) {
                    batchSpan.recordException(throwable);
                    batchSpan.setStatus(StatusCode.ERROR, throwable.getMessage());
                } else {
                    batchSpan.setStatus(StatusCode.OK);
                }
            } finally {
                batchSpan.end();
                if (scope != null) {
                    scope.close();
                }
            }
        }
    }

    /**
     * Reflection으로 Trace ID 추출
     */
    private static String extractTraceId(Object spanContextInfo) {
        try {
            Method getTraceIdMethod = spanContextInfo.getClass()
                .getMethod("getTraceId");
            return (String) getTraceIdMethod.invoke(spanContextInfo);
        } catch (Exception e) {
            return null;
        }
    }

    /**
     * Reflection으로 Span ID 추출
     */
    private static String extractSpanId(Object spanContextInfo) {
        try {
            Method getSpanIdMethod = spanContextInfo.getClass()
                .getMethod("getSpanId");
            return (String) getSpanIdMethod.invoke(spanContextInfo);
        } catch (Exception e) {
            return null;
        }
    }
}
```

### 3.2 OpenSearchBatchProcessor 상세

```java
public class OpenSearchBatchProcessor {

    public void startProcessing() throws InterruptedException {
        while (true) {
            try {
                processBatches();
            } catch (Exception e) {
                log.error("Failed to process batches", e);
            }

            // 5초마다 폴링
            Thread.sleep(5000);
        }
    }

    private void processBatches() throws IOException {
        List<Map<String, String>> batches = readBatchQueue();

        for (Map<String, String> batch : batches) {
            String status = batch.get("status");

            if ("PENDING".equals(status)) {
                String transactionNumber = batch.get("transactionNumber");
                String batchType = batch.get("batchType");

                log.info("[OpenSearchBatchProcessor] Processing batch: " + transactionNumber);

                // OpenSearch에서 원본 Trace 검색
                SpanContextInfo spanContextInfo = searchTransactionInOpenSearch(transactionNumber);

                if (spanContextInfo != null) {
                    log.info("[OpenSearchBatchProcessor] ✓ Trace found in OpenSearch!");
                    log.info("  Trace ID: " + spanContextInfo.getTraceId());
                    log.info("  Span ID: " + spanContextInfo.getSpanId());

                    // 배치 실행 (BatchInstrumentation에 의해 자동 계측됨)
                    processBatch(transactionNumber, batchType, spanContextInfo);
                } else {
                    log.warn("[OpenSearchBatchProcessor] ✗ Trace not found in OpenSearch");
                }
            }
        }
    }

    /**
     * OpenSearch에서 transaction 태그로 Trace 검색
     */
    private SpanContextInfo searchTransactionInOpenSearch(String transactionNumber) {
        try {
            String today = LocalDate.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd"));
            String index = "jaeger-span-" + today;

            String query = String.format(
                "{\"query\":{\"nested\":{\"path\":\"tags\",\"query\":{\"bool\":{\"must\":["
                + "{\"match\":{\"tags.key\":\"transaction\"}},"
                + "{\"match\":{\"tags.value\":\"%s\"}}"
                + "]}}}}}}",
                transactionNumber
            );

            // HTTP POST 요청으로 OpenSearch 검색
            HttpClient client = HttpClient.newHttpClient();
            HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create(opensearchUrl + "/" + index + "/_search"))
                .header("Content-Type", "application/json")
                .POST(HttpRequest.BodyPublishers.ofString(query))
                .build();

            HttpResponse<String> response = client.send(request,
                HttpResponse.BodyHandlers.ofString());

            // 응답 파싱
            ObjectMapper mapper = new ObjectMapper();
            JsonNode root = mapper.readTree(response.body());
            JsonNode hits = root.path("hits").path("hits");

            if (hits != null && hits.size() > 0) {
                JsonNode firstHit = hits.get(0);
                JsonNode source = firstHit.path("_source");

                String traceId = source.path("traceID").asText();
                String spanId = source.path("spanID").asText();

                return new SpanContextInfo(traceId, spanId);
            }

        } catch (Exception e) {
            log.error("Failed to search OpenSearch", e);
        }

        return null;
    }
}
```

## 7. 트러블슈팅

### 7.1 OpenSearch 검색 실패

**현상**:
```
WARN - No transaction found in OpenSearch - HTTP 200
```

**원인**: transaction 태그가 없는 Trace 검색

**해결 1**: transaction 태그 추가
```java
// OpenTelemetry Span에 attribute 설정
span.setAttribute("transaction", transactionNumber);
```

**해결 2**: Trace ID 직접 검색 사용
```java
if (transaction 태그 검색 실패) {
    return searchByTraceId(transactionNumber, today);
}
```

### 7.2 배치 처리 무한 루프

**현상**: 배치가 5초마다 계속 처리됨

**원인**: 배치 큐 파일에서 status 업데이트 안 됨

**임시 해결**: 배치 처리 후 파일에서 제거
```java
batches.removeIf(batch ->
    transactionNumber.equals(batch.get("transactionNumber"))
);
writeQueueFile(batches);  // 파일 업데이트
```

**추천 해결**: DB 또는 Redis 사용 (status 컬럼/필드 업데이트)

## 8. 개선이 필요한 사항

### 8.1 현재 한계

1. **수동 배치 큐 관리**: 파일을 직접 수정해야 함
2. **상태 업데이트 없음**: 처리 완료 후 status가 PENDING으로 남음
3. **폴링 방식**: 5초마다 파일 읽기 (비효율적)

### 8.2 추천 개선사항

#### 8.2.1 데이터베이스 활용

```sql
CREATE TABLE batch_queue (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    transaction_number VARCHAR(64) NOT NULL,
    batch_type VARCHAR(50) NOT NULL,
    status VARCHAR(20) NOT NULL,  -- PENDING, PROCESSING, COMPLETED, FAILED
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status (status),
    INDEX idx_transaction (transaction_number)
);
```

#### 8.2.2 REST API 제공

```java
@RestController
@RequestMapping("/api/batch")
public class BatchController {

    @PostMapping
    public ResponseEntity<?> addBatch(@RequestBody BatchRequest request) {
        // DB에 배치 추가
        batchService.addToQueue(request);
        return ResponseEntity.ok().body(Map.of("status", "QUEUED"));
    }

    @DeleteMapping("/{transactionNumber}")
    public ResponseEntity<?> removeBatch(@PathVariable String transactionNumber) {
        // DB에서 배치 제거
        batchService.removeFromQueue(transactionNumber);
        return ResponseEntity.ok().body(Map.of("status", "REMOVED"));
    }
}
```
