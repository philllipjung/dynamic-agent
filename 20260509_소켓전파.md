## 💻 핵심 컴포넌트 소스

### 1. SocketTraceAgent.java - Agent 진입점

**역할:** ByteBuddy Agent를 초기화하고 SocketHandler를 자동 계측

```java
public class SocketTraceAgent {
    public static void premain(String agentArgs, Instrumentation inst) {
        System.out.println("🤖 100% 자동 계측 Agent 시작");

        new AgentBuilder.Default()
            .type(ElementMatchers.named("com.socket.server.SocketHandler"))
            .transform((builder, typeDescription, classLoader, module, protectionDomain) -> {
                return builder.visit(
                    net.bytebuddy.asm.Advice.to(SocketTraceAdvice.class)
                        .on(ElementMatchers.named("handle")
                            .and(ElementMatchers.takesArguments(1))
                            .and(ElementMatchers.takesArgument(0, java.net.Socket.class)))
                );
            })
            .installOn(inst);
    }
}
```

**핵심 포인트:**
- `premain()`: JVM 시작 시 자동 호출
- `AgentBuilder`: ByteBuddy를 사용한 클래스 변환
- `ElementMatchers`: SocketHandler.handle(Socket) 메서드만 대상

### 2. SocketTraceAdvice.java - 100% 자동 계측 핵심

**역할:** 소켓에서 traceparent를 추출하고 OpenTelemetry Context를 자동 생성

```java
public class SocketTraceAdvice {

    @Advice.OnMethodEnter(inline = false, suppress = Throwable.class)
    public static Scope onMethodEnter(@Advice.Argument(0) Socket socket) {
        System.out.println("🤖 100% 자동 계측 시작");

        try {
            // 1. 소켓에서 traceparent 추출 (스트림 소비하지 않음)
            String traceParent = extractTraceParentFromSocket(socket);

            if (traceParent != null && !traceParent.isEmpty()) {
                // 2. W3C traceparent 파싱
                String[] parts = traceParent.split("-");
                String traceId = parts[1];
                String spanId = parts[2];
                byte flags = (byte) Integer.parseInt(parts[3], 16);

                // 3. SpanContext 생성
                SpanContext spanContext = ImmutableSpanContext.create(traceId, spanId, flags);

                // 4. Context 생성 및 자동 ThreadLocal 저장
                Context extractedContext = Context.root().with(Span.wrap(spanContext));
                Scope scope = extractedContext.makeCurrent();  // 🔥 핵심!

                System.out.println("✅ 100% 자동으로 Context 생성");
                System.out.println("🔍 Trace ID: " + Span.current().getSpanContext().getTraceId());

                return scope;  // 메서드 종료 시 자동 정리를 위해 반환
            }
        } catch (Exception e) {
            System.err.println("❌ Error: " + e.getMessage());
        }
        return null;
    }

    @Advice.OnMethodExit(inline = false, suppress = Throwable.class)
    public static void onMethodExit(@Advice.Enter Scope scope) {
        if (scope != null) {
            scope.close();  // 자동으로 Scope 정리
            System.out.println("✅ 자동으로 Scope 정리 완료");
        }
    }

    // 소켓에서 traceparent 추출 (BufferedInputStream mark/reset 사용)
    private static String extractTraceParentFromSocket(Socket socket) {
        java.io.BufferedInputStream bufferedInput =
            new java.io.BufferedInputStream(socket.getInputStream());
        bufferedInput.mark(1024);  // 스트림 위치 저장

        // 헤더 읽기
        BufferedReader lineReader = new BufferedReader(new InputStreamReader(bufferedInput));
        String line;
        while ((line = lineReader.readLine()) != null) {
            if (line.startsWith("traceparent:")) {
                bufferedInput.reset();  // 스트림 위치 복원
                return line.substring("traceparent:".length()).trim();
            }
            if (line.isEmpty()) break;
        }

        bufferedInput.reset();
        return null;
    }
}
```

**기술적 하이라이트:**
- `@Advice.OnMethodEnter`: 메서드 진입 전에 자동 실행
- `BufferedInputStream.mark/reset`: 스트림을 소비하지 않고 peeking
- `makeCurrent()`: OpenTelemetry Context를 ThreadLocal에 자동 저장
- `Scope 반환`: 메서드 종료 시 자동 정리 보장

### 3. SocketHandler.java - 🔥 서버 측 순수 비즈니스 로직!

**역할:** 소켓 요청을 처리하고 HTTP 서버를 호출
**특징:** 서버 측 코드에 추적 관련 코드가 전혀 없음 (100% 자동 계측)

```java
@Component
public class SocketHandler {

    @Autowired
    private RestTemplate restTemplate;

    /**
     * 🔥 100% 자동 계측 소켓 핸들러
     *
     * 추적 관련 코드가 전혀 없습니다!
     * ByteBuddy Agent가 모든 추적을 자동으로 처리합니다.
     */
    public void handle(Socket socket) {
        System.out.println("📦 SocketHandler.handle() 호출됨");
        System.out.println("🔥 추적 코드 없이 순수 비즈니스 로직만!");

        try {
            // 비즈니스 로직만!
            processClientMessage(socket);  // 메시지 처리
            callHttpServer();             // HTTP 서버 호출

        } finally {
            try {
                socket.close();
            } catch (Exception e) {
                // Ignore
            }
        }
    }

    private void processClientMessage(Socket socket) throws Exception {
        socket.setSoTimeout(1000);
        BufferedReader reader = new BufferedReader(
            new InputStreamReader(socket.getInputStream())
        );

        String line;
        StringBuilder message = new StringBuilder();
        try {
            while ((line = reader.readLine()) != null) {
                if (line.isEmpty()) break;
                message.append(line);
            }
        } catch (java.net.SocketTimeoutException e) {
            // 타임아웃 무시
        }

        if (message.length() > 0) {
            System.out.println("📨 받은 메시지: " + message.toString());
        }
    }

    private String callHttpServer() {
        System.out.println("=================================================");
        System.out.println("🌐 HTTP 서버 호출 시작");
        System.out.println("=================================================");

        try {
            String response = restTemplate.getForObject(
                "http://localhost:8080/hello",
                String.class
            );

            System.out.println("📥 HTTP 응답: " + response);
            System.out.println("=================================================");
            return response;

        } catch (Exception e) {
            System.err.println("❌ HTTP 호출 실패: " + e.getMessage());
            return "Error: " + e.getMessage();
        }
    }
}
```

**핵심 포인트:**
- ✅ 서버 측 추적 관련 코드가 **0줄**
- ✅ 순수 비즈니스 로직만 포함
- ✅ ByteBuddy Agent가 자동으로 Context 생성 및 전파

### 4. TracingSocketClient.java - 🔧 클라이언트 측 수동 계측

**역할:** traceparent를 생성하여 소켓 서버로 전송
**특징:** 클라이언트 측 수동 계측 - traceparent 직접 생성 및 전송

```java
public class TracingSocketClient {

    public void sendWithTrace(String message) {
        // traceparent 생성
        String traceId = generateTraceId();      // 32자 hex
        String spanId = generateSpanId();        // 16자 hex
        String traceParent = String.format("00-%s-%s-01", traceId, spanId);

        System.out.println("Trace ID: " + traceId);
        System.out.println("Traceparent: " + traceParent);

        try (Socket socket = new Socket(host, port);
             PrintWriter writer = new PrintWriter(socket.getOutputStream(), true)) {

            // traceparent 헤더 전송
            writer.println("traceparent: " + traceParent);
            writer.println();  // 빈 줄로 헤더 끝 표시
            writer.println(message);
            writer.flush();

            System.out.println("Message sent: " + message);
        }
    }

    private String generateTraceId() {
        return UUID.randomUUID().toString().replace("-", "");
    }

    private String generateSpanId() {
        return UUID.randomUUID().toString().replace("-", "").substring(0, 16);
    }
}
```

## 🔧 트러블슈팅 과정

### 문제 1: ClassNotFoundException
**에러:**
```
java.lang.ClassNotFoundException: com.socket.server.context.TraceContextHolder
```

**원인:** Reflection으로 Spring Boot 클래스로더에 접근하지 못함

**해결:** ContextClassLoader 사용
```java
// ❌ 실패
Class.forName("com.socket.server.context.TraceContextHolder")

// ✅ 성공
ClassLoader contextClassLoader = Thread.currentThread().getContextClassLoader();
Class.forName("com.socket.server.context.TraceContextHolder", true, contextClassLoader)
```

### 문제 2: 소켓 스트림 소비
**에러:** SocketTraceAdvice가 traceparent를 추출하면서 스트림을 소비하여 SocketHandler에서 메시지를 읽을 수 없음

**해결:** BufferedInputStream의 mark/reset 사용
```java
BufferedInputStream bufferedInput = new BufferedInputStream(inputStream);
bufferedInput.mark(1024);           // 현재 위치 저장
// ... 헤더 읽기 ...
bufferedInput.reset();            // 원래 위치로 복원
```

### 문제 3: Advice가 실행되지 않음
**증상:** "🤖 100% 자동 계측 시작" 메시지가 나타나지 않음

**원인:**
- SocketOutputAdvice를 참조하는데 클래스가 없음
- @Advice.Local 사용 방식이 잘못됨

**해결:**
1. SocketOutputAdvice 참조 제거
2. @Advice.OnMethodEnter가 Scope를 반환하도록 수정
3. @Advice.OnMethodExit에서 @Advice.Enter로 받기

### 문제 4: Trace ID가 모두 0으로 나옴
**증상:** Trace ID가 `00000000000000000000000000000000`

**원인:** OpenTelemetry API 버전 호환性问题

**해결:** SpanContext에서 TraceFlags 타입 수정
```java
// ❌ 실패
public byte getTraceFlags() { return traceFlags; }

// ✅ 성공
public TraceFlags getTraceFlags() { return traceFlags; }
```

### 문제 5: 동일한 클래스 이름 충돌
**증상:** socket 모듈과 spring-boot-server 모듈에 SocketHandler가 존재

**해결:** socket 모듈의 중복 클래스 삭제
- socket/src/main/java/com/socket/server/SocketHandler.java ❌ 제거
- spring-boot-server/src/main/java/com/socket/server/SocketHandler.java ✅ 유지

## 📚 기술적 하이라이트

### 1. ByteBuddy Advice

**Advice란?**
- 메서드의 시작/종료에 자동으로 코드를 주입하는 기술
- 사용자 코드를 변경하지 않고 동작을 수정

**우리가 사용한 방식:**
- `@Advice.OnMethodEnter`: 메서드 시작 전에 자동 실행
- `@Advice.OnMethodExit`: 메서드 종료 후에 자동 실행
- `@Advice.Enter`: Enter의 반환값을 Exit에서 받기

### 2. OpenTelemetry Context 전파

**ThreadLocal 저장:**
```java
Context context = Context.root().with(Span.wrap(spanContext));
Scope scope = context.makeCurrent();  // ThreadLocal에 저장
```

**Span.current()로 접근:**
```java
// 어디서든 현재 Trace ID에 접근 가능
String traceId = Span.current().getSpanContext().getTraceId();
```

### 3. W3C traceparent 포맷

```
00-traceid-spanid-flags

└─────────────────────────────── Version
```

### 4. 스트림 처리 기법

**BufferedInputStream mark/reset:**
```java
bufferedInput.mark(1024);     // 현재 위치 저장
// ... 스트림 읽기 ...
bufferedInput.reset();       // 원래 위치로 복원
```

**목적:** 스트림을 소비하지 않고 peeking (훔쳐보기)
