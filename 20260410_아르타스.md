## 1. 핵심 기능 개요

### 1.1 세 가지 핵심 명령 비교

| 명령 | 용도 | 출력 | 주요 사용처 |
|------|------|------|-------------|
| **Watch** | 메서드 호출 시점의 값 관찰 | 파라미터 값, 반환값 | 🔥 파라미터 추출 (가장 중요) |
| **Stack** | 호출 스택 추적 | 스택 프레임 목록 | 메서드 호출 경로 분석 |
| **Trace** | 메서드 실행 내부 추적 | 호출 그래프 | 성능 병목 지점 분석 |

### 1.2 왜 비동기 방식이 필요한가?

**문제:** 동기 방식은 메서드가 호출될 때까지 **무한 대기**

```bash
# 동기 방식 (문제 있음)
POST /api/arthas/watch
→ 60초~120초 동안 메서드 호출을 기다림
→ 타임아웃 발생
```

**해결:** 비동기 방식으로 **요청-대기-조회** 패턴 사용

```bash
# 비동기 방식 (권장)
POST /api/arthas/startWatch  → 즉시 jobId 반환
메서드 호출                    → 실제로 메서드 실행
GET /api/arthas/getWatchResult/{jobId} → 결과 조회
```

## 2. 주요 문제점 및 해결

### 2.1 JSON 파싱 오류 (Watch, getParameter)

**🔴 문제 현상:**
```json
{
  "success": false,
  "error": "Expected a ',' or '}' at 60 [character 61 line 1]"
}
```

**🔍 원인 분석:**

1. **RestTemplate 인코딩 문제**
   - Spring RestTemplate이 Arthas HTTP API 응답을 처리할 때 UTF-8 인코딩 오류 발생
   - 한글 및 특수 문자가 포함된 JSON 응답 파싱 실패

2. **이중 JSON 변환**
   - Arthas HTTP API가 JSON 문자열을 반환
   - RestTemplate이 다시 JSON으로 변환하려고 시도
   - 결과: 이스케이프 문자 충돌

**✅ 해결 방안:**

RestTemplate → `java.net.http.HttpClient`로 변경

**변경 전:**
```java
// ArthasController.java - watch() 메서드
String result = ArthasManager.watchWithParameters(
    request.getClassName(),
    request.getMethodName(),
    request.getLimit()
);
// ↑ RestTemplate 사용 중 → JSON 파싱 오류
```

**변경 후:**
```java
// ArthasController.java - watch() 메서드
@PostMapping("/watch")
public Map<String, Object> watch(@RequestBody WatchRequest request) {
    try {
        String url = "http://127.0.0.1:8563/api";
        org.json.JSONObject requestBody = new org.json.JSONObject();
        requestBody.put("action", "exec");
        requestBody.put("command", "watch " + request.getClassName() + " "
            + request.getMethodName() + " params -n " + request.getLimit() + " -x 1");
        requestBody.put("execTimeout", "5000");

        java.net.http.HttpClient client = java.net.http.HttpClient.newHttpClient();
        java.net.http.HttpRequest httpRequest = java.net.http.HttpRequest.newBuilder()
            .uri(java.net.URI.create(url))
            .header("Content-Type", "application/json")
            .POST(java.net.http.HttpRequest.BodyPublishers.ofString(requestBody.toString()))
            .build();

        java.net.http.HttpResponse<String> response = client.send(httpRequest,
            java.net.http.HttpResponse.BodyHandlers.ofString());

        return Map.of(
            "success", true,
            "data", response.body(),  // ← Arthas 응답을 그대로 반환 (파싱 없이)
            "command", "watch " + request.getClassName() + " " + request.getMethodName()
        );
    } catch (Exception e) {
        e.printStackTrace();
        return Map.of("success", false, "error", e.getMessage());
    }
}
```

**적용 대상:**
- `POST /api/arthas/watch`
- `GET /api/arthas/getParameter`

**결과:** ✅ JSON 파싱 오류 100% 해결

### 2.2 Stack/Trace 타임아웃 문제

**🔴 문제 현상:**
```json
{
  "success": true,
  "message": "{\"error\": \"API call failed: I/O error on POST request for \\\"http://127.0.0.1:8563/api\\\": Read timed out\"}",
  "command": "stack com.test.service.test1.controller.Test1Controller test1"
}
```

**🔍 원인 분석:**

1. **메서드 호출 대기**
   - stack/trace 명령은 메서드가 실제로 호출될 때까지 대기
   - 개발/테스트 환경에서는 사용자가 수동으로 메서드를 호출해야 함
   - HTTP 요청 타임아웃(60~120초) 발생

2. **동기식 API 설계 문제**
   - 하나의 HTTP 요청에서 모든 작업을 완료하려고 시도
   - 사용자가 메서드를 호출할 시간을 확보할 수 없음

**✅ 해결 방안:**

비동기 API 추가 구현 (Watch와 동일한 패턴)

**ArthasManager.java에 추가:**
```java
// Async stack support
private static final ExecutorService stackExecutor = Executors.newCachedThreadPool();
private static final Map<String, String> stackResults = new ConcurrentHashMap<>();
private static final Map<String, String> stackStatus = new ConcurrentHashMap<>();

public static String startStackAsync(String className, String methodName) {
    String jobId = "stack-" + System.currentTimeMillis();
    String pid = getFirstAvailablePid();

    stackStatus.put(jobId, "RUNNING");

    // 백그라운드 스레드에서 실행
    stackExecutor.submit(() -> {
        try {
            String stackCmd = String.format("stack %s %s -n 1", className, methodName);
            String result = executeTunnelApiCommand(pid, stackCmd);
            stackResults.put(jobId, result);
            stackStatus.put(jobId, "COMPLETED");
        } catch (Exception e) {
            stackResults.put(jobId, "ERROR: " + e.getMessage());
            stackStatus.put(jobId, "FAILED");
        }
    });

    return jobId;
}

public static JSONObject getStackResult(String jobId) {
    JSONObject result = new JSONObject();
    result.put("jobId", jobId);

    String status = stackStatus.getOrDefault(jobId, "UNKNOWN");
    result.put("status", status);

    if ("COMPLETED".equals(status) || "FAILED".equals(status)) {
        String data = stackResults.get(jobId);
        result.put("result", data);
        result.put("success", !data.startsWith("ERROR"));
    }

    return result;
}
```

**ArthasController.java에 추가:**
```java
/**
 * Start async stack job
 * POST /api/arthas/startStack
 */
@PostMapping("/startStack")
public Map<String, Object> startStack(@RequestBody MethodRequest request) {
    String jobId = ArthasManager.startStackAsync(
        request.getClassName(),
        request.getMethodName()
    );

    return Map.of(
        "success", true,
        "jobId", jobId,
        "message", "Stack job started. Call the method, then check result with /api/arthas/getStackResult/" + jobId,
        "status", "RUNNING"
    );
}

/**
 * Get stack result
 * GET /api/arthas/getStackResult/{jobId}
 */
@GetMapping("/getStackResult/{jobId}")
public Map<String, Object> getStackResult(@PathVariable("jobId") String jobId) {
    org.json.JSONObject result = ArthasManager.getStackResult(jobId);

    // Convert JSONObject to Map
    Map<String, Object> resultMap = new java.util.LinkedHashMap<>();
    for (String key : result.keySet()) {
        resultMap.put(key, result.get(key));
    }

    return resultMap;
}
```

**동일한 패턴으로 Trace도 구현:**
- `POST /api/arthas/startTrace`
- `GET /api/arthas/getTraceResult/{jobId}`

**결과:** ✅ 타임아웃 문제 완전 해결

## 3. Watch 상세 가이드

**목적:** 🔥 메서드 파라미터 값 추출 (가장 중요한 기능)

### 3.1 Watch 동작 원리

```
사용자 요청 → Watch 시작 → 메서드 호출 감시 → 파라미터 캡처 → 결과 반환
```

**Watch가 캡처하는 정보:**
- 메서드 파라미터 값 (params)
- 메서드 반환값 (returnObj)
- 메서드 실행 전/후 상태 (target, params, returnObj)

### 3.2 비동기 Watch 테스트 (권장)

**단계 1: Watch 시작**
```bash
curl -X POST http://localhost:8080/api/arthas/startWatch \
  -H "Content-Type: application/json" \
  -d '{
    "className": "com.test.service.test1.controller.Test1Controller",
    "methodName": "test1",
    "limit": 1
  }'
```

**응답:**
```json
{
  "success": true,
  "jobId": "watch-1774679668818",
  "message": "Watch job started. Call the method, then check result with /api/arthas/getWatchResult/watch-1774679668818",
  "status": "RUNNING"
}
```

**단계 2: 메서드 호출**
```bash
# 실제 애플리케이션의 메서드 호출
curl "http://localhost:8081/test1?userId=0004"
```

**응답:**
```
Test1 - userId: 0004, setUserId result: 0001
```

**단계 3: 결과 조회**
```bash
curl "http://localhost:8080/api/arthas/getWatchResult/watch-1774679668818"
```

**응답:**
```json
{
  "jobId": "watch-1774679668818",
  "success": true,
  "status": "COMPLETED",
  "className": "com.test.service.test1.controller.Test1Controller",
  "methodName": "test1",
  "parameterCount": 3,
  "parameters": "[{\"name\":\"param0\",\"index\":0,\"type\":\"Object\",\"value\":\"@Object\"},{\"name\":\"param1\",\"index\":1,\"type\":\"Object\",\"value\":\"@String\"},{\"name\":\"param2\",\"index\":2,\"type\":\"Object\",\"value\":\"0007\"}]"
}
```

### 3.3 추출된 파라미터 분석

**param0: HttpServletRequest**
- **타입:** Object
- **값:** @Object
- **설명:** Spring Framework가 자동으로 주입하는 HttpServletRequest
- **인덱스:** 0

**param1: @RequestParam 애너테이션 파라미터**
- **타입:** Object
- **값:** @String
- **설명:** @RequestParam 애너테이션이 붙은 String 파라미터
- **인덱스:** 1

**param2: 실제 사용자 입력 값**
- **타입:** Object
- **값:** "0007" 또는 "0004"
- **설명:** 사용자가 실제로 전달한 값
- **인덱스:** 2

## 4. Stack 상세 가이드

**목적:** 🔥 메서드 호출 경로(스택 프레임) 추적

### 4.1 Stack 동작 원리

```
메서드 호출 → 호출 스택 수집 → 스택 프레임 분석 → 호출 경로 반환
```

**Stack이 보여주는 정보:**
- 메서드 호출 순서 (top-down)
- 각 스택 프레임의 클래스명, 메서드명, 라인번호
- 스레드 정보

## 5. Trace 상세 가이드

**목적:** 🔥 메서드 내부 호출 그래프 추적 (성능 분석)

### 5.1 Trace 동작 원리

```
메서드 호출 → 내부 호출 수집 → 호출 트리 구성 → 실행 시간 측정
```

**Trace가 보여주는 정보:**
- 메서드 내부에서 호출한 다른 메서드들
- 각 호출의 실행 시간 (cost)
- 호출 횟수 (times)
- 호출 트리 구조

### 5.2 비동기 Trace 테스트

```bash

```

```json
  "success": true,
  "status": "RUNNING"
}
```

**단계 2: 메서드 호출**
```bash
curl "http://localhost:8081/test1?userId=traceTest"
```

**응답:**
```
Test1 - userId: 0011, setUserId result: 0001
```

**단계 3: 결과 조회**
```bash
curl "http://localhost:8080/api/arthas/getTraceResult/trace-1774681684644"
```

**응답 (요약):**
```json
{
  "jobId": "trace-1774681684644",
  "success": true,
  "status": "COMPLETED",
  "result": "{\"body\":{\"command\":\"trace com.test.service.test1.controller.Test1Controller test1 -n 1\",\"jobStatus\":\"TERMINATED\",\"results\":[{\"nodeCount\":3,\"root\":{\"children\":[{\"className\":\"com.test.service.shared.UserIdService\",\"methodName\":\"generateUserId\",\"cost\":69800,\"times\":1},{\"className\":\"com.test.service.shared.UserIdService\",\"methodName\":\"setUserId\",\"cost\":4900,\"times\":1}],\"className\":\"com.test.service.test1.controller.Test1Controller\",\"methodName\":\"test1\",\"cost\":334200,\"times\":1}}]},\"state\":\"SUCCEEDED\"}"
```

### 5.3 Trace 트리 분석

**호출 트리 구조:**
```
test1 (334.2ms) ← Test1Controller
├── generateUserId (69.8ms) ← UserIdService
│   └── 실행 시간: 69.8ms
│   └── 호출 횟수: 1회
└── setUserId (4.9ms) ← UserIdService
    └── 실행 시간: 4.9ms
    └── 호출 횟수: 1회
```

**성능 분석:**
- **총 실행 시간:** 334.2ms
- **generateUserId:** 69.8ms (20.9%)
- **setUserId:** 4.9ms (1.5%)

### 7.1 문제 해결 프로세스

1. **문제 발견:** JSON 파싱 오류, 타임아웃

4. **검증:** 11/11 테스트 통과 (100%)

### 7.2 핵심 교훈

1. **비동기 패턴의 중요성**
   - 사용자가 개입해야 하는 작업(메서드 호출)은 비동기로
   - 요청-대기-조회 패턴이 타임아웃 문제 해결

2. **UTF-8 인코딩 처리**
   - 한글 및 특수 문자가 포함된 JSON은 주의
   - java.net.http.HttpClient가 안정적

3. **일관된 API 설계**
   - Watch, Stack, Trace 모두 동일한 비동기 패턴
   - 사용자가 쉽게 이해하고 사용 가능
