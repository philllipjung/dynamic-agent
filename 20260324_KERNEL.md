## Overview

Kernel Advice & Helper는 OpenTelemetry 기반의 커널 수준 추적 시스템으로, Spring WebFlux 애플리케이션의 Reactor Scheduler (parallel, boundedElastic)에서 실행되는 비동기 작업을 자동으로 추적합니다.

### Key Capabilities

- ✅ 자동 OpenTelemetry Span 생성/관리
- ✅ TraceId/SpanId 자동 추출
- ✅ 스레드 이름 자동 커스터마이징 (TraceId/SpanId 포함)
- ✅ Reactor Scheduler 통합 (parallel, boundedElastic)
- ✅ Child Span 자동 생성 (병렬 작업)
- ✅ 메서드 실행 시간 측정
- ✅ 예외 자동 기록

### Use Cases

1. **분산 추적**: 마이크로서비스 간 요청 추적
2. **성능 분석**: 메서드 실행 시간 측정
3. **스레드 모니터링**: Reactor 스레드 풀 추적
4. **디버깅**: TraceId/SpanId로 로그 연결

## Service1Controller Analysis

### 원본 코드 (Before)

```java
@RestController
public class Service1Controller {
    private final Tracer tracer;

    @GetMapping("/")
    public Mono<String> index(@RequestParam String name) {
        return Mono.just(name)
            .flatMap(n -> {
                // 40줄의 추적 로직이 비즈니스 로직에 섞여있음
                Span span = tracer.spanBuilder("index-operation").startSpan();
                SpanContext spanContext = span.getSpanContext();
                String traceId = spanContext.getTraceId();
                String spanId = spanContext.getSpanId();

                // reactor-http-epoll 스레드 이름 변경
                String customThreadName = "reactor-http-epoll-" +
                    traceId + "-" + spanId + "-" + threadNumber;
                Thread.currentThread().setName(customThreadName);

                // 10개의 parallel 스레드 생성
                Mono<Void> parallelTasks = Flux.range(1, 10)
                    .flatMap(i -> Mono.fromRunnable(() -> {
                        // Child span 생성
                        Span parallelSpan = tracer.spanBuilder("parallel-task-" + i)
                            .setParent(io.opentelemetry.context.Context.current())
                            .startSpan();
                        // 스레드 이름 변경
                        // ... 20줄의 코드
                    }).subscribeOn(Schedulers.parallel()))
                    .then();

                // 10개의 boundedElastic 스레드 생성
                Mono<Void> boundedElasticTasks = Flux.range(1, 10)
                    .flatMap(i -> Mono.fromRunnable(() -> {
                        // Child span 생성
                        Span boundedSpan = tracer.spanBuilder("bounded-elastic-task-" + i)
                            .setParent(io.opentelemetry.context.Context.current())
                            .startSpan();
                        // 스레드 이름 변경
                        // ... 20줄의 코드
                    }).subscribeOn(Schedulers.boundedElastic()))
                    .then();

                return Mono.when(parallelTasks, boundedElasticTasks).then(Mono.just(n));
            })
            .flatMap(mono -> mono)
            .doFinally(signalType -> {
                span.end();
            });
    }
}
```

### 문제점

| 문제 | 설명 | 영향 |
|------|------|------|
| **코드 중복** | 40줄의 추적 로직 반복 | 유지보수 어려움 |
| **비즈니스 로직 오염** | 추적 코드가 섞여있음 | 가독성 저하 |
| **수동 관리** | Span 수동 생성/종료 | 실수 가능성 |
| **스레드 감지** | 수동으로 스레드 타입 판별 | 누락 위험 |
| **에러 핸들링** | Span 실패 시 미처리 | 리스크 증가 |

### 개선된 코드 (After)

```java
@RestController
public class Service1Controller {
    @GetMapping("/")
    public Mono<String> index(@RequestParam String name) {
        // KernelAdvice가 자동으로:
        // 1. Span 생성
        // 2. TraceId/SpanId 추출
        // 3. 스레드 이름 변경
        // 4. Child span 생성
        // 5. Span 완료

        return Mono.just(name)
            .flatMap(this::callService2);
    }

    private Mono<String> callService2(String name) {
        return webClient.get()
            .uri("/?name=" + name)
            .retrieve()
            .bodyToMono(String.class);
    }
}
```

## Components

### 1. KernelHelper

**Location:** `bytebuddy-advice/src/.../helper/KernelHelper.java`

**Role:** OpenTelemetry Span 생명주기 관리 및 추적 정보 추출

#### 핵심 기능

```java
// 1. Span 생성
KernelHelper helper = new KernelHelper(tracer, "operation-name").startSpan();

// 2. TraceId/SpanId 추출
String traceId = helper.getTraceId();  // "1234abcd..."
String spanId = helper.getSpanId();    // "5678efgh..."

// 3. 스레드 이름 커스터마이징
helper.setReactorHttpEpollThreadName();      // 1234abcd5678efgR1
helper.setParallelThreadName(1);              // 1234abcd5678efgP1
helper.setBoundedElasticThreadName(1);        // 1234abcd5678efgB1

// 4. Child Span 생성
KernelHelper childHelper = helper.createChildSpan("parallel-task-1");

// 5. 속성 설정
helper.setAttribute("custom.key", "value");

// 6. 예외 기록
helper.recordException(exception);

// 7. Span 완료
helper.complete();
```

#### 주요 API

| 메서드 | 설명 | 반환값 |
|--------|------|--------|
| `startSpan()` | 새로운 Span 시작 | `KernelHelper` |
| `getTraceId()` | TraceId 추출 | `String` |
| `getSpanId()` | SpanId 추출 | `String` |
| `setThreadName(prefix, suffix)` | 스레드 이름 설정 | `KernelHelper` |
| `createChildSpan(name)` | Child Span 생성 | `KernelHelper` |
| `setAttribute(key, value)` | Span 속성 설정 | `KernelHelper` |
| `recordException(throwable)` | 예외 기록 | `KernelHelper` |
| `complete()` | Span 완료 | `void` |
| `isValid()` | Span 유효성 확인 | `boolean` |

### 2. KernelAdvice

**Location:** `bytebuddy-advice/src/.../advices/KernelAdvice.java`

**Role:** 메서드 인터셉트를 통한 자동 추적

#### 작동 방식

```java
@Advice.OnMethodEnter
public static void onMethodEnter(
    @Advice.Origin Method method,
    @Advice.This Object target,
    @Advice.AllArguments Object[] allArguments
) {
    // 1. Tracer 획득
    Tracer tracer = getTracer();

    // 2. KernelHelper 생성 + Span 시작
    KernelHelper helper = new KernelHelper(tracer, "ClassName.methodName").startSpan();

    // 3. 스레드 타입 감지 + 이름 변경
    customizeThreadName(helper, className, methodName);

    // 4. ThreadLocal에 저장
    helperHolder.set(helper);
}

@Advice.OnMethodExit
public static void onMethodExit(@Advice.Thrown Throwable throwable) {
    // 1. 실행 시간 계산
    long durationMs = (System.nanoTime() - startTime) / 1_000_000;

    // 2. 속성 설정
    helper.setAttribute("method.duration.ms", durationMs);

    // 3. 예외 기록 (있는 경우)
    if (throwable != null) {
        helper.recordException(throwable);
    }

    // 4. Span 완료
    helper.complete();
}
```

#### 자동 스레드 타입 감지

| 스레드 이름 패턴 | 감지 방법 | 이니셜 | 설정되는 이름 |
|-----------------|-----------|--------|---------------|
| `reactor-http-epoll-*` | `contains("reactor-http-epoll")` | R | `{TraceId 7}{SpanId 6}R{threadNum}` |
| `parallel-*` | `contains("parallel")` | P | `{TraceId 7}{SpanId 6}P{taskNum}` |
| `boundedElastic-*` | `contains("boundedElastic")` | B | `{TraceId 7}{SpanId 6}B{taskNum}` |
| 그 외 | 기본값 | X | `{TraceId 7}{SpanId 6}X{num}` |

**스레드 이름 형식:** `{TraceId 7자}{SpanId 6자}{이니셜}{숫자}`

**예시:** `1234abcd5678efgR1`

## Features

### 1. OpenTelemetry Span 관리

#### Span 생명주기

```
[Method Enter]
    │
    ├─ tracer.spanBuilder(name).startSpan()
    │  └─ Span 생성 + Scope 만들기
    │
    ├─ span.makeCurrent()
    │  └─ Context.current()로 현재 Span 설정
    │
    ├─ [Business Logic]
    │  └─ 비즈니스 로직 실행
    │
    ├─ span.setAttribute(key, value)
    │  └─ 속성 추가
    │
[Method Exit]
    │
    ├─ scope.close()
    │  └─ Context 정리
    │
    └─ span.end()
       └─ Jaeger/Tempo로 전송
```

### 4. Reactor Scheduler 통합

#### Parallel Scheduler

```java
// CPU 작업을 위한 고정 크기 스레드 풀
Mono<Void> parallelTasks = Flux.range(1, 10)

        // 비즈니스 로직만 작성
        doWork(i);

    .then();
```

```java
// I/O 작업을 위한 탄력적 스레드 풀
```

### 5. Child Span 생성

#### Parent-Child 관계

```java
// Parent Span
KernelHelper parentHelper = new KernelHelper(tracer, "parent-operation").startSpan();

KernelHelper childHelper = parentHelper.createChildSpan("child-operation-1");

// Child Span 사용

// Child Span 완료
childHelper.complete();

// Parent Span 완료
parentHelper.complete();
```

## Troubleshooting

### Issue 1: Span not appearing in Jaeger/Tempo

**Check:**
1. OpenTelemetry가 구성되었는지 확인
2. Jaeger/Tempo가 실행 중인지 확인

```bash
curl http://localhost:8080/actuator/health

# Jaeger 상태 확인

grep "KernelHelper" application.log
```

### Issue 2: Thread name not changing

**Cause:**
KernelAdvice가 적용되지 않음

**Solution:**
```bash
# KernelAdvice 재적용
```

### Issue 3: Child spans not created

**Cause:**
스레드 이름 패턴이 일치하지 않음

**Solution:**
```java
// 커스텀 스레드 이름 사용
KernelHelper helper = KernelHelper.getCurrentHelper();
if (helper != null && helper.isValid()) {
    helper.setThreadName("my-custom-thread", "1");
}
```

---

### Issue 4: Memory leak

**Cause:**
Span이 완료되지 않음

**Check:**
```bash
# JVM 메모리 확인
jmap -histo:live <pid> | grep OpenTelemetry

# Span 수 확인
curl http://localhost:8080/actuator/metrics/jvm.memory.used
```

**Solution:**
```java
// try-finally로 Span 완료 보장
KernelHelper helper = new KernelHelper(tracer, "operation").startSpan();
try {
    // 비즈니스 로직
} finally {
    helper.complete();
}
```
