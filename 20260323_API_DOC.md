## 1. 인프라 설정

### 1.1 서비스 시작

```bash
# 1. Docker Compose (Jaeger, OpenSearch, Redis)
cd jaeger-main/docker-compose/monitor
docker-compose up -d

# 2. Agent Server
cd agent-server
mvn spring-boot:run

# 3. Test Services
# Terminal 1 - Test1 Service
cd test-services/test1-service
mvn spring-boot:run

# Terminal 2 - Test2 Service
cd test-services/test2-service
mvn spring-boot:run
```

### 1.2 서비스 URL

| 서비스 | URL | 포트 | 설명 |
|--------|-----|------|------|
| Agent Server | http://localhost:8080 | 8080 | REST API |
| Test1 Service | http://localhost:8081 | 8081 | 첫 번째 테스트 서비스 |
| Test2 Service | http://localhost:8082 | 8082 | 두 번째 테스트 서비스 |
| Jaeger UI | http://localhost:16686 | 16686 | Trace 조회 UI |
| OpenSearch | http://localhost:9200 | 9200 | Span 데이터 검색 |
| Redis | localhost:6379 | 6379 | 설정/매핑 저장 |

### 1.3 Test Service 구조

```
test-services/
├── test1-service/           # 포트 8081
│   └── Test1Controller.java
│       ├── test1(String userId)
│       ├── setUserIdEndpoint(String userId)
│       └── generateUserId()
│
├── test2-service/           # 포트 8082
│   └── Test2Controller.java
│       ├── test2(String userId)
│       ├── setUserIdEndpoint(String userId)
│       └── generateUserId()
│
└── shared/                  # 공유 모듈
    └── UserIdService.java
        ├── generateUserId()    → "0001", "0002", ...
        └── setUserId(String)   → "0001" (고정 반환)
```

## 2. API 엔드포인트

### 2.1 Arthas API

#### GET /api/arthas/processes

실행 중인 JVM 프로세스 목록을 조회합니다.

**요청**:
```bash
curl http://localhost:8080/api/arthas/processes
```

**응답**:
```json
{
  "success": true,
  "processes": [
    {
      "pid": "12345",
      "mainClass": "com.test.service.test1.Test1Application",
      "appName": "test1-service"
    },
    {
      "pid": "12346",
      "mainClass": "com.test.service.test2.Test2Application",
      "appName": "test2-service"
    }
  ]
}
```

---

#### POST /api/arthas/attach

JVM에 Arthas를 연결합니다.

**요청**:
```bash
curl -X POST http://localhost:8080/api/arthas/attach \
  -H "Content-Type: application/json" \
  -d '{
    "pid": "12345"
  }'
```

**응답**:
```json
{
  "success": true,
  "message": "Arthas attached to PID 12345",
  "sessionId": "session-abc123"
}
```

---

#### POST /api/arthas/attachByClassName

클래스 이름으로 JVM에 연결합니다.

**요청**:
```bash
curl -X POST http://localhost:8080/api/arthas/attachByClassName \
  -H "Content-Type: application/json" \
  -d '{
    "className": "test1-service"
  }'
```

**응답**:
```json
{
  "success": true,
  "message": "Attached to PID 12345",
  "pid": "12345"
}
```

---

#### POST /api/arthas/watch

메서드 파라미터를 관찰합니다.

**요청**:
```bash
curl -X POST http://localhost:8080/api/arthas/watch \
  -H "Content-Type: application/json" \
  -d '{
    "pid": "12345",
    "className": "com.test.service.shared.UserIdService",
    "methodName": "setUserId",
    "condition": "params[0] != null",
    "numberOfIterations": 1
  }'
```

**응답**:
```json
{
  "success": true,
  "output": "Affect(class count:1 , method count:1) \n\n---ts=2026-03-22 10:30:00; [cost=1.23ms] result=Object --------------------------------\nparams[0]=String(user-123)"
}
```

```bash
      "0": "userId"
```

**응답**:
```json

  "className": "com.test.service.test1.controller.Test1Controller",

    "0": "userId"
```

### 2.2 ByteBuddy API

#### POST /api/bytebuddy/createSpan

메서드에 OpenTelemetry Span을 생성합니다.

**요청**:
```bash
curl -X POST http://localhost:8080/api/bytebuddy/createSpan \
  -H "Content-Type: application/json" \
  -d '{
    "className": "com.test.service.shared.UserIdService",
    "methodName": "setUserId"
  }'
```

**응답**:
```json
{
  "success": true,
  "message": "SUCCESS: Span created for com.test.service.shared.UserIdService.setUserId",
  "spanName": "com.test.service.shared.UserIdService.setUserId"
}
```

---

#### POST /api/bytebuddy/createSpanAttribute

메서드 파라미터를 스팬 속성으로 캡처합니다.

**요청**:
```bash
curl -X POST http://localhost:8080/api/bytebuddy/createSpanAttribute \
  -H "Content-Type: application/json" \
  -d '{
    "className": "com.test.service.shared.UserIdService",
    "methodName": "setUserId",
    "parameterMapping": {
      "0": "userId"
    }
  }'
```

**응답**:
```json
{
  "success": true,
  "message": "SUCCESS: Span attribute advice created for com.test.service.shared.UserIdService.setUserId",
  "attributes": {
    "0": "userId"
  }
}
```

---

### 2.3 Span Link API

#### POST /api/spanlink/create

스팬 링크를 생성합니다.

**요청**:
```bash
curl -X POST http://localhost:8080/api/spanlink/create \
  -H "Content-Type: application/json" \
  -d '{
    "sourceSpanName": "com.test.service.test1.controller.Test1Controller.setUserIdEndpoint",
    "sourceAttributeKey": "arthas.attribute.userId",
    "targetSpanName": "com.test.service.test2.controller.Test2Controller.setUserIdEndpoint",
    "targetAttributeKey": "arthas.attribute.userId",
    "linkType": "PARENT_TO_CHILD"
  }'
```

**응답**:
```json
{
  "success": true,
  "linkId": "setuseridendpoint-setuseridendpoint-a1b2c3d4",
  "message": "Span link created",
  "bytebuddyResult": "SUCCESS: Span link advice created"
}
```

## 3. 샘플 데이터

### 3.1 Test Service 메서드 구조

#### Test1Controller (포트 8081)

```java
@RestController
@RequestMapping("/test1")
public class Test1Controller {

    // GET /test1?userId=xxx
    public String test1(@RequestHeader String userId);

    // GET /test1/setUserId?userId=xxx
    public String setUserIdEndpoint(@RequestParam String userId);

    // GET /test1/generateUserId
    public String generateUserId();
}
```

#### Test2Controller (포트 8082)

```java
@RestController
@RequestMapping("/test2")
public class Test2Controller {

    // GET /test2?userId=xxx
    public String test2(@RequestHeader String userId);

    // GET /test2/setUserId?userId=xxx
    public String setUserIdEndpoint(@RequestParam String userId);

    // GET /test2/generateUserId
    public String generateUserId();
}
```

#### UserIdService (공유)

```java
@Service
public class UserIdService {

    // "0001", "0002", "0003", ... 형식의 userId 생성
    public String generateUserId();

    // 항상 "0001" 반환
    public String setUserId(String userId);
}
```

### 3.2 샘플 테스트 요청

#### Test1 Service 요청

```bash
# 1. userId 생성 (Counter 1 증가)
curl http://localhost:8081/test1/generateUserId
# 응답: "0001"

# 2. setUserId 호출
curl "http://localhost:8081/test1/setUserId?userId=user-123"
# 응답: "0001"

# 3. test1 엔드포인트 (헤더에 userId)
curl -H "userId: user-456" http://localhost:8081/test1
# 응답: "Test1 - userId: user-456, setUserId result: 0001"
```

```bash

curl http://localhost:8082/test2/generateUserId

curl "http://localhost:8082/test2/setUserId?userId=user-123"

curl -H "userId: user-789" http://localhost:8082/test2
```

```bash
curl -X POST http://localhost:8080/api/arthas/attach \
  -H "Content-Type: application/json" \

# 3단계: 스팬 생성 (Test1)

curl -X POST http://localhost:8080/api/bytebuddy/createSpan \

# 4단계: 스팬 생성 (Test2)

  }'

# Watch 실행 전에 먼저 테스트 요청:
curl "http://localhost:8081/test1/setUserId?userId=test-user-001"

# 예상 Watch 출력:
# params[0]=String(test-user-001)

# ========================================
# 6단계: 속성 캡처 (Test1)
# ========================================

    "methodName": "setUserIdEndpoint",

# 7단계: 속성 캡처 (Test2)
# ========================================

  -H "Content-Type: application/json" \
  -d '{
    "className": "com.test.service.test2.controller.Test2Controller",
    "methodName": "setUserIdEndpoint",
    "parameterMapping": {
      "0": "userId"
    }
  }'

# ========================================
# 8단계: 스팬 링크 생성
# ========================================
curl -X POST http://localhost:8080/api/spanlink/create \
  -H "Content-Type: application/json" \
  -d '{
    "sourceSpanName": "com.test.service.test1.controller.Test1Controller.setUserIdEndpoint",
    "sourceAttributeKey": "arthas.attribute.userId",
    "targetSpanName": "com.test.service.test2.controller.Test2Controller.setUserIdEndpoint",
    "targetAttributeKey": "arthas.attribute.userId",
    "linkType": "PARENT_TO_CHILD"
  }'

# ========================================
# 9단계: 테스트 요청 (여러 번)
# ========================================
# 요청 1: user-Alice
curl "http://localhost:8081/test1/setUserId?userId=user-Alice"
curl "http://localhost:8082/test2/setUserId?userId=user-Alice"

# 요청 2: user-Bob
curl "http://localhost:8081/test1/setUserId?userId=user-Bob"
curl "http://localhost:8082/test2/setUserId?userId=user-Bob"

# 요청 3: user-Charlie
curl "http://localhost:8081/test1/setUserId?userId=user-Charlie"
curl "http://localhost:8082/test2/setUserId?userId=user-Charlie"

# ========================================
# 10단계: Jaeger UI에서 확인
# ========================================
# 브라우저: http://localhost:16686
#
# 검색 조건:
#   - Service: test1-service 또는 test2-service
#   - Operation: setUserIdEndpoint
#   - Tags: arthas.attribute.userId
#
# 예상 결과:
#   Trace 1: user-Alice
#     Span 1: Test1Controller.setUserIdEndpoint (userId=user-Alice)
#       └─ link.parent_to_child.* → Span 2
#     Span 2: Test2Controller.setUserIdEndpoint (userId=user-Alice)

# ========================================
# 11단계: OpenSearch 검색
# ========================================

  -H "Content-Type: application/json" \
  -d '{
    "query": {
      "bool": {
        "must": [
          {"term": {"tags.key": "arthas.attribute.userId"}},
          {"term": {"tags.value": "user-Alice"}}
        ]
      }
    }
  }'

# ========================================
# 12단계: 링크 확인
# ========================================
curl http://localhost:8080/api/spanlink/list
```

```bash
  -d '{
    "pid": "12345",
    "className": "com.test.service.test1.controller.Test1Controller",

  }'
```

## 7. 빠른 테스트 명령어 모음

### 한 번에 실행하는 스크립트

```bash
#!/bin/bash
# quick-test.sh

echo "1. Attaching to services..."
curl -X POST http://localhost:8080/api/arthas/attachByClassName \
  -H "Content-Type: application/json" \
  -d '{"className": "test1-service"}'

  -H "Content-Type: application/json" \
  -d '{"className": "test2-service"}'

echo "2. Creating spans..."

  -H "Content-Type: application/json" \
  -d '{"className": "com.test.service.test1.controller.Test1Controller", "methodName": "setUserIdEndpoint"}'

curl -X POST http://localhost:8080/api/bytebuddy/createSpan \
  -H "Content-Type: application/json" \
  -d '{"className": "com.test.service.test2.controller.Test2Controller", "methodName": "setUserIdEndpoint"}'

echo "3. Creating attributes..."
curl -X POST http://localhost:8080/api/bytebuddy/createSpanAttribute \
  -H "Content-Type: application/json" \
  -d '{"className": "com.test.service.test1.controller.Test1Controller", "methodName": "setUserIdEndpoint", "parameterMapping": {"0": "userId"}}'

curl -X POST http://localhost:8080/api/bytebuddy/createSpanAttribute \
  -H "Content-Type: application/json" \
  -d '{"className": "com.test.service.test2.controller.Test2Controller", "methodName": "setUserIdEndpoint", "parameterMapping": {"0": "userId"}}'

echo "4. Creating span link..."
curl -X POST http://localhost:8080/api/spanlink/create \
  -H "Content-Type: application/json" \
  -d '{
    "sourceSpanName": "com.test.service.test1.controller.Test1Controller.setUserIdEndpoint",

    "targetAttributeKey": "arthas.attribute.userId",

  }'

echo "5. Sending test requests..."
```
