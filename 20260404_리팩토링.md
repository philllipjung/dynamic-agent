## 📋 개요

본 문서는 Java Agent 시스템의 주요 기능을 테스트하고 소스 코드를 분석한 결과를 정리한 것입니다.

**테스트한 기능:**
1. Span Link 생성 (스팬 간 연결)
2. Thread Name Helper (스레드명을 TraceId/SpanId로 변경)
3. KernelAdvice (HTTP 요청 헤더/바디 JSON 출력)

```java
    private static boolean injectHelper(ClassLoader targetLoader)

    // Span 생성 (Bytecode Transform)
```

- bytebuddy-core (Bytecode 조작)
- agent-commons (공통 상수)

### 2. bytebuddy-advice

**역할**: ByteBuddy Advice 클래스 (런타임에 삽입될 코드)

**주요 클래스**:

#### SpanAdvice.java
```java
public class SpanAdvice {
    @Advice.OnMethodEnter(inline = false)
    public static SpanHelper onMethodEnter(...) {
        // 1. 스레드명 변경
        if (ThreadNameHelper.isTargetThread()) {
            ThreadNameHelper.renameThread(span);
        }

        // 2. Span 생성 (Link 포함)
        SpanHelper spanHelper = SpanHelper.createSpanWithLinks(...);

        // 3. 파라미터 속성 추출
        // parameterMapping에서 userId 등 추출

        return spanHelper;
    }

    @Advice.OnMethodExit(inline = false)
    public static void onMethodExit(@Advice.Enter SpanHelper spanHelper) {
        // 1. Span 완료
        spanHelper.complete();

        // 2. 스레드명 복원
        Thread.currentThread().setName(originalThreadName);
    }
}
```

#### ThreadNameHelper.java
```java
public class ThreadNameHelper {
    // 스레드명 형식: {TraceId 7자}{SpanId 6자}{이니셜}{숫자}
    // 예: 26cd7480d92c2DBG6176

    public static void renameThread(Span span) {
        SpanContext context = span.getSpanContext();
        String traceId = context.getTraceId();
        String spanId = context.getSpanId();

        String newName = formatThreadName(traceId, spanId);
        Thread.currentThread().setName(newName);
    }

    private static String formatThreadName(String traceId, String spanId) {
        String tracePrefix = traceId.substring(0, 7);
        String spanPrefix = spanId.substring(0, 6);
        String initial = INITIALS[Math.abs(traceId.hashCode()) % INITIALS.length];
        int number = ThreadLocalRandom.current().nextInt(10000);

        return String.format("%s%s%s%04d", tracePrefix, spanPrefix, initial, number);
    }
}
```

#### KernelAdvice.java
```java
public class KernelAdvice {
    @Advice.OnMethodEnter(inline = true)
    public static void onMethodEnter(
            @Advice.Origin String method,
            @Advice.Argument(0) Object request
    ) {
        if (request instanceof HttpServletRequest) {
            HttpServletRequest httpRequest = (HttpServletRequest) request;

            // JSON 형식으로 요청 정보 출력
            KernelHelper.printRequestInfo(httpRequest);
            KernelHelper.printRequestHeaders(httpRequest);
            KernelHelper.printRequestBody(httpRequest);
        }
    }
}
```

#### KernelHelper.java
```java

}
```

## 🔧 리팩토링 포인트

### 1. inline=false 사용 (개발용)

**변경 전**:
```java
@Advice.OnMethodEnter(inline = true)
```

**변경 후**:
```java
@Advice.OnMethodEnter(inline = false)
```

**이유**:
- `inline=true`: Advice 코드를 bytecode에 직접 복사 (소스 수정 반영 안 됨)
- `inline=false`: 메서드 호출 방식 (소스 수정 즉시 반영, 디버깅 용이)

### 2. Maven 빌드 순서

**올바른 순서**:
```bash
cd agent-commons && mvn clean install
cd bytebuddy-advice && mvn clean install
cd bytebuddy-agent && mvn clean package
cp target/bytebuddy-agent-1.0.0.jar ../agent-server/
```

**중요**: `mvn install` 사용 (로컬 리포지토리에 설치)

### 3. Spring Boot 3.x 호환

**변경**:
- `javax.servlet.*` → `jakarta.servlet.*`
- Spring Boot 3.x는 Jakarta EE 사용

### 4. DispatcherServlet 계측

**변경**: Filter 대신 `DispatcherServlet.doService` 계측

**이유**:
- Spring Boot 3.x에서 Filter 구조가 복잡해짐
- `DispatcherServlet`은 모든 HTTP 요청이 통과하는 진입점

## 🎯 주요 발견 사항

### 1. @RequestHeader vs @RequestParam

Controller 코드:
```java
@GetMapping
public String test1(@RequestHeader(value = "userId", required = false) String userId)
```

**중요**: userId는 Header로 전달해야 함
- ✅ `curl -H "userId: TEST_ID" http://localhost:8081/test1`
- ❌ `curl http://localhost:8081/test1?userId=TEST_ID`

### 2. Jaeger 인덱싱 대기 시간

Span Link 생성을 위해서는 첫 번째 요청 후 **최소 10초 대기** 필요

이유: Jaeger가 span을 인덱싱하는 데 시간이 소요됨

### 3. Agent HTTP Server 포트

- **agent-server**: port 8080 (Spring Boot)
- **ByteBuddyAgent HTTP Server**: port 10000 (내장)

KernelAdvice 적용은 port 10000 사용:
```bash
curl -X POST "http://localhost:10000/api/createKernelAdvice"
```

## 🔍 소스 코드 흐름 분석

### Span 생성 흐름

```
1. HTTP 요청 도착
   ↓
2. DispatcherServlet.doService() (KernelAdvice 실행)
   → JSON 출력
   ↓
3. Test1Controller.test1() (SpanAdvice 실행)
   ↓
4. @OnMethodEnter
   → ThreadNameHelper.renameThread()
   → SpanHelper.createSpanWithLinks()
   → JaegerLinkLookupService 조회
   → Span에 Link 추가
   ↓
5. 비즈니스 로직 실행
   ↓
6. @OnMethodExit
   → SpanHelper.complete()
   → Thread.currentThread().setName(original)
   ↓
7. OpenTelemetry Reporter가 Jaeger로 전송
```
