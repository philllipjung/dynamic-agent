## Overview

Event Advice & Helper는 Spring Filter의 HTTP 요청/응답을 자동으로 캡처하고 저장하는 바이트코드 인스트루먼테이션 시스템입니다. OpenTelemetry와 연동하여 분산 추적을 위한 이벤트 데이터를 수집합니다.

### Key Capabilities

- ✅ HTTP Request/Response 캡처
- ✅ 자동 JSON 파싱 (요청/응답)
- ✅ Redis 영구 저장소
- ✅ UI 표시용 데이터 포맷
- ✅ Thread-safe 설계
- ✅ 최대 1000개 이벤트 자동 관리

### Use Cases

1. **API 모니터링**: 모든 HTTP 요청/응답 추적
2. **디버깅**: 요청/응답 내용 확인
3. **분석**: JSON 페이로드 키-값 분석
4. **트레이싱**: OpenTelemetry 연동

## Components

### 1. EventHelper

**Location:** `bytebuddy-advice/src/.../helper/EventHelper.java`

**Role:** HTTP 요청/응답 데이터 캡처 및 포맷팅

#### Key Methods

```java
// 생성자
public EventHelper(HttpServletRequest request, HttpServletResponse response)

// 요청 캡처 (JSON 자동 파싱 포함)
public EventHelper captureRequest()

// 응답 캡처
public EventHelper captureResponse()

// Redis에 저장
public EventHelper save()

// 정적 메서드 - 데이터 조회
public static List<Map<String, Object>> getAllEvents()
public static Map<String, Object> getEvent(String eventId)
public static void clearAllEvents()
```

#### EventData Structure

```java
public static class EventData {
    // 기본 정보
    private String method;                    // HTTP 메서드 (GET, POST, ...)
    private String requestUri;                // 요청 URI
    private String queryString;               // 쿼리 스트링
    private long timestamp;                   // 타임스탬프

    // 요청 정보
    private Map<String, String> requestHeaders;    // 요청 헤더
    private String requestBody;                    // 요청 바디 (원본)
    private Map<String, Object> requestBodyJson;   // 요청 바디 (JSON 파싱)

    // 응답 정보
    private int responseStatus;                     // 응답 상태 (200, 404, ...)
    private Map<String, String> responseHeaders;    // 응답 헤더
    private String responseBody;                    // 응답 바디 (원본)
    private Map<String, Object> responseBodyJson;   // 응답 바디 (JSON 파싱)
}
```

### 2. EventAdvice

**Location:** `bytebuddy-advice/src/.../advices/EventAdvice.java`

**Role:** Spring Filter의 `doFilter` 메서드를 인터셉트하여 자동으로 이벤트 캡처

#### ByteBuddy Advice Annotations

```java
@Advice.OnMethodEnter
public static void onMethodEnter(
    @Advice.Origin Method method,
    @Advice.Argument(0) ServletRequest request,
    @Advice.Argument(1) ServletResponse response
) {
    // 1. HTTP 요청/응답 확인
    // 2. EventHelper 생성
    // 3. 요청 정보 캡처
    // 4. ThreadLocal에 저장
}

@Advice.OnMethodExit(onThrowable = Throwable.class)
public static void onMethodExit(
    @Advice.Argument(1) ServletResponse response,
    @Advice.Thrown Throwable throwable
) {
    // 1. 응답 정보 캡처
    // 2. Redis에 저장
    // 3. ThreadLocal 정리
}
```

#### Target Filters

EventAdvice는 다음 타입의 Filter에 적용 가능:

- `OncePerRequestFilter` (Spring 권장)
- `GenericFilterBean`
- `javax.servlet.Filter` 구현체

## Features


## Limitations & Solutions

### 1. Response Body Capture

**Problem:**
```java
// HttpServletResponse의 응답 바디는 이미 쓰여진 후 읽을 수 없음
private String extractResponseBody(HttpServletResponse response) {
    return "[Response body not available - use ContentCachingResponseWrapper]";
}
```

**Solution - ContentCachingResponseWrapper:**

```java
@Component
public class ExampleFilter extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain filterChain
    ) throws ServletException, IOException {

        // 응답 래핑
        ContentCachingResponseWrapper wrappedResponse =
            new ContentCachingResponseWrapper(response);

        // 필터 체인 실행
        filterChain.doFilter(request, wrappedResponse);

        // 응답 바디 캡처
        byte[] content = wrappedResponse.getContentAsByteArray();
        String responseBody = new String(content, response.getCharacterEncoding());

        // 원본 응답에 바디 복사
        wrappedResponse.copyBodyToResponse();
    }
}
```

---

### 2. Request Body Already Read

**Problem:**
요청 바디가 다른 필터나 인터셉터에서 이미 읽힌 경우

**Solution - ContentCachingRequestWrapper:**

```java
ContentCachingRequestWrapper wrappedRequest =
    new ContentCachingRequestWrapper(request);

filterChain.doFilter(wrappedRequest, response);

byte[] content = wrappedRequest.getContentAsByteArray();
String requestBody = new String(content, request.getCharacterEncoding());
```

---

### 3. Memory Overhead

**Problem:**
모든 요청/응답을 캡처하면 메모리 과다 사용

**Solutions:**

1. **필터링:**
```java
// 특정 URI만 캡처
if (request.getRequestURI().startsWith("/api/important")) {
    helper.captureRequest();
}
```

2. **크기 제한:**
```java
// 바디 크기 제한
private static final int MAX_BODY_SIZE = 10_000; // 10KB

if (body.length() > MAX_BODY_SIZE) {
    body = body.substring(0, MAX_BODY_SIZE) + "... (truncated)";
}
```

3. **TTL 설정:**
```java
// Redis에 TTL 추가
jedis.setex(eventKey, 3600, jsonValue); // 1시간 후 만료
```

## Best Practices

### 1. Filter Placement

EventAdvice를 가장 먼저 적용:
```java
@Component
@Order(Ordered.HIGHEST_PRECEDENCE)  // 가장 높은 우선순위
public class LoggingFilter extends OncePerRequestFilter {
    // ...
}
```

### 2. Selective Capture

```java
@Override
protected void doFilterInternal(
        HttpServletRequest request,
        HttpServletResponse response,
        FilterChain filterChain
) throws ServletException, IOException {

    // 중요한 API만 캡처
    if (shouldCapture(request)) {
        EventHelper helper = new EventHelper(request, response);
        helper.captureRequest();
        // ...
    }

    filterChain.doFilter(request, response);
}

private boolean shouldCapture(HttpServletRequest request) {
    String uri = request.getRequestURI();
    return uri.startsWith("/api/") &&
           !uri.contains("/health") &&
           !uri.contains("/metrics");
}
```

### 3. Error Handling

```java
@Advice.OnMethodExit(onThrowable = Throwable.class)
public static void onMethodExit(
    @Advice.Argument(1) ServletResponse response,
    @Advice.Thrown Throwable throwable
) {
    try {
        EventHelper helper = helperHolder.get();
        if (helper == null) return;

        helper.captureResponse();

        // 예외 정보 기록
        if (throwable != null) {
            helper.setAttribute("error", throwable.getMessage());
            helper.setAttribute("errorType", throwable.getClass().getSimpleName());
        }

        helper.save();
    } catch (Exception e) {
        // EventHelper 실패가 원본 요청에 영향을 주지 않도록
        System.err.println("[EventAdvice] Failed to save event: " + e.getMessage());
    } finally {
        cleanup();
    }
}
```

## Troubleshooting

### Issue: Events not being captured

**Check:**
1. EventAdvice가 제대로 적용되었는지 확인
2. Filter가 실제로 호출되는지 확인
3. Log 메시지 확인

```bash
# Log 확인
tail -f application.log | grep EventAdvice
```

### Issue: Redis connection failed

**Check:**
1. Redis가 실행 중인지 확인
2. Host/Port 설정 확인

```bash
# Redis 상태 확인
redis-cli ping

# Redis 로그 확인
tail -f /var/log/redis/redis.log
```

### Issue: JSON parsing fails

**Solution:**
JSON 파싱은 선택 사항이며, 실패해도 원본 바디는 저장됩니다.

```java
// EventHelper.java
try {
    Map<String, Object> jsonMap = mapper.readValue(body, Map.class);
    eventData.setRequestBodyJson(jsonMap);
} catch (Exception jsonException) {
    // JSON이 아니면 그냥 넘어감
    System.out.println("[EventHelper] Body is not valid JSON");
}
```
