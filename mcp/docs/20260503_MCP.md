## Critical Discovery: Parameter Naming Convention

### The Problem

Initial testing failures with error:
```
"unexpected additional properties ["serviceName"]"
```

### Root Cause

**Jaeger MCP uses `snake_case` parameter names (Go convention), NOT `camelCase` (JavaScript convention)**

| ❌ WRONG | ✅ CORRECT |
|---------|-----------|
| `serviceName` | `service_name` |
| `spanId` | `span_id` |
| `traceId` | `trace_id` |

### Why This Happened

- Jaeger is written in **Go**
- Go uses **snake_case** for JSON field names
- JavaScript/TypeScript uses **camelCase**
- User error: Assuming JavaScript convention for a Go-based tool

## Session Management

### MCP Session Flow

1. **Initialize** - Get session ID from HTTP headers
2. **Send Initialized Notification** - Required before calling tools
3. **Call Tools** - Use session ID in all subsequent requests

### Example Session Initialization

```bash
# Step 1: Initialize and capture session ID
SESSION_ID=$(curl -v -X POST http://localhost:16687/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{...}}' \
  2>&1 | grep "< Mcp-Session-Id:" | sed 's/.*: //' | tr -d '\r')

# Step 2: Send initialized notification (REQUIRED!)
curl -s -X POST http://localhost:16687/mcp \
  -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SESSION_ID" \
  -d '{"jsonrpc":"2.0","method":"notifications/initialized"}'

# Step 3: Now you can call tools
curl -s -X POST http://localhost:16687/mcp \
  -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: $SESSION_ID" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{...}}'
```

**Common Error**: Skipping step 2 causes:
```
"method 'tools/list' is invalid during session initialization"
```

## API Response Format

### Server-Sent Events (SSE)

All responses use SSE format:

```
event: message
data: {"jsonrpc":"2.0","id":2,"result":{...}}
```

### Parsing SSE Responses

**Method 1: sed**
```bash
curl -s ... | sed 's/.*data: //' | jq '.result'
```

**Method 2: Python**
```python
import json, sys
data = json.loads(sys.stdin.read().split('data: ')[1])
print(json.dumps(data, indent=2))
```

**Method 3: jq with split**
```bash
curl -s ... | jq 'split("data: ") | .[1] | fromjson'
```

## Key Learnings

1. **Always use snake_case** for parameter names in Jaeger MCP
2. **Session initialization is required** before calling any tools
3. **Send notifications/initialized** after initialize
4. **SSE format** requires parsing `data:` prefix
5. **Trace IDs expire** - use current traces from search
6. **span_ids is required array** for get_span_details tool
7. **MCP Inspector not compatible** with Jaeger's SSE transport
8. **Direct curl API works perfectly** for all 8 tools

## Troubleshooting Guide

### Error: "unexpected additional properties"

**Cause**: Using camelCase parameter names

**Solution**: Use snake_case (e.g., `service_name` not `serviceName`)

### Error: "required: missing properties: ['span_ids']"

**Cause**: Missing span_ids array parameter

**Solution**: Include both trace_id AND span_ids array:
```json
{
  "trace_id": "...",
  "span_ids": ["span1", "span2"]
}
```

### Error: "method 'tools/list' is invalid during session initialization"

**Cause**: Didn't send notifications/initialized message

**Solution**: Always send initialized notification after initialize

### Error: "session not found"

**Cause**: Session ID expired or invalid

**Solution**: Initialize new session and get fresh session ID

```bash
pkill -f "jaeger.*--config"
```

```bash
sleep 5

# Verify
if ps aux | grep -q "jaeger.*--config" | grep -v grep; then
    echo "✅ Jaeger started successfully"
    ps aux | grep "jaeger.*--config" | grep -v grep
else
    echo "❌ Jaeger failed to start"
    tail -20 /root/test/logs/jaeger-current.log
    exit 1
fi

# Check ports
echo ""
echo "Checking ports..."
for port in 16686 16685 16687 4317 4318; do
    if ss -tlnp | grep -q ":$port"; then
        echo "✅ Port $port is listening"
    else
        echo "❌ Port $port is NOT listening"
    fi
done

# Test API
echo ""
echo "Testing API..."
if curl -s "http://localhost:16686/api/services" > /dev/null; then
    echo "✅ Jaeger API is responding"
else
    echo "❌ Jaeger API is NOT responding"
fi

echo ""
echo "=== Restart complete ==="
```
