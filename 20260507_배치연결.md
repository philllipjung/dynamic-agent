## 1. 개요

### 1.1 목적

OpenSearch에 저장된 원본 Trace를 검색하고, 배치 처리 시 새로운 Span을 생성하며 **Span Link**로 원본 Trace와 연결

### 1.2 핵심 기능

- **OpenSearch Trace 검색**: transaction 태그 또는 Trace ID로 원본 Span 검색
- **Span Link 생성**: 배치 Span과 원본 Span 연결 (tags로 link 정보 저장)
- **Java Agent 자동 계측**: ByteBuddy Advice로 배치 처리 메서드에 자동으로 Span 생성

### 1.3 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 원본 트랜잭션 완료                                            │
```

## 2. 소스 수정 내용

### 2.1 BatchInstrumentation.java - Java Agent 진입점 추가

**위치**: `spark-integration/batch-processor/src/main/java/com/javaagent/spark/batch/BatchInstrumentation.java`

**수정 전**: premain/agentmain 메서드 없음

**수정 후**:
```java
public class BatchInstrumentation {

    static final Logger log = Logger.getLogger(BatchInstrumentation.class.getName());

    public static volatile OpenTelemetrySdk customOpenTelemetry;
    public static volatile Tracer customTracer;

    /**
     * Java Agent premain - called when JVM starts with -javaagent
     */
    public static void premain(String agentArgs, Instrumentation instrumentation) {
        log.info("[BatchInstrumentation] ========================================");
        log.info("[BatchInstrumentation] Java Agent loaded via premain()");
        log.info("[BatchInstrumentation] Agent args: " + agentArgs);
        log.info("[BatchInstrumentation] ========================================");
        install(instrumentation);
    }

    /**
     * Java Agent agentmain - called when agent is attached to running JVM
     */
    public static void agentmain(String agentArgs, Instrumentation instrumentation) {
        log.info("[BatchInstrumentation] ========================================");
        log.info("[BatchInstrumentation] Java Agent attached via agentmain()");
        log.info("[BatchInstrumentation] Agent args: " + agentArgs);
        log.info("[BatchInstrumentation] ========================================");
        install(instrumentation);
    }

    // ... 나머지 코드
}
```

**변경 사항**:
1. `premain(String agentArgs, Instrumentation instrumentation)` 메서드 추가
   - JVM 시작 시 `-javaagent:` 옵션으로 로드될 때 호출됨
2. `agentmain(String agentArgs, Instrumentation instrumentation)` 메서드 추가
   - 실행 중인 JVM에 attach할 때 호출됨

### 2.2 pom.xml - Java Agent Manifest 설정

**위치**: `spark-integration/batch-processor/pom.xml`

**수정 전**:
```xml
<transformer implementation="org.apache.maven.plugins.shade.resource.ManifestResourceTransformer">
    <mainClass>com.javaagent.spark.batch.OpenSearchBatchProcessor</mainClass>
</transformer>
```

**수정 후**:
```xml
<transformer implementation="org.apache.maven.plugins.shade.resource.ManifestResourceTransformer">
    <manifestEntries>
        <Premain-Class>com.javaagent.spark.batch.BatchInstrumentation</Premain-Class>
        <Agent-Class>com.javaagent.spark.batch.BatchInstrumentation</Agent-Class>
        <Main-Class>com.javaagent.spark.batch.OpenSearchBatchProcessor</Main-Class>
        <Can-Retransform-Classes>true</Can-Retransform-Classes>
        <Can-Redefine-Classes>true</Can-Redefine-Classes>
    </manifestEntries>
</transformer>
```

**변경 사항**:
1. `Premain-Class`: JVM 시작 시 로드할 Agent 클래스
2. `Agent-Class`: 실행 중인 JVM에 attach할 Agent 클래스
3. `Can-Retransform-Classes`: 클래스 재변환 허용
4. `Can-Redefine-Classes`: 클래스 재정의 허용

### 2.3 OtelConfig.java - OTEL 설정 클래스 추가

**위치**: `spark-integration/batch-processor/src/main/java/com/javaagent/commons/config/OtelConfig.java`

**내용**:
```java
package com.javaagent.commons.config;

public class OtelConfig {

    private static final String DEFAULT_ENDPOINT = "http://localhost:4317";
    private static final int DEFAULT_TIMEOUT_SECONDS = 30;

    public static String getOtelEndpoint() {
        // 환경변수 OTEL_EXPORTER_OTLP_ENDPOINT 우선
        String endpoint = System.getenv("OTEL_EXPORTER_OTLP_ENDPOINT");
        if (endpoint != null && !endpoint.isEmpty()) {
            return endpoint;
        }

        // 시스템 속성 otel.exporter.otlp.endpoint
        endpoint = System.getProperty("otel.exporter.otlp.endpoint");
        if (endpoint != null && !endpoint.isEmpty()) {
            return endpoint;
        }

        return DEFAULT_ENDPOINT;
    }

    public static int getExporterTimeoutSeconds() {
        // 환경변수 또는 시스템 속성에서 timeout 읽기
        // 기본값 30초
        return DEFAULT_TIMEOUT_SECONDS;
    }
}
```

```java
try {

    if (scope != null) {
        scope.close();
    }
}
```

## 7. 주요 성공 요인

### 7.1 Java Agent 실행 방식

```
# 잘못된 방법 (-jar로 실행)
java -jar batch-processor-1.0.0.jar
→ BatchInstrumentation이 로드되지 않음

# 올바른 방법 (-javaagent로 실행)
java -javaagent:batch-processor-1.0.0.jar \
     -cp batch-processor-1.0.0.jar \
     com.javaagent.spark.batch.OpenSearchBatchProcessor
→ premain()이 호출되어 ByteBuddy instrumentation 설치
```

### 7.2 ByteBuddy Advice 패턴

```java
@Advice.OnMethodEnter
public static void onMethodEnter(
        @Advice.Argument(0) String transactionNumber,
        @Advice.Argument(1) String batchType,
        @Advice.Argument(2) Object spanContextInfo,
        @Advice.Local("batchSpan") Span batchSpan) {

    // 1. Custom OpenTelemetry SDK 초기화
    getCustomOpenTelemetry();

    // 2. Reflection으로 Trace 정보 추출
    String traceId = extractTraceId(spanContextInfo);
    String spanId = extractSpanId(spanContextInfo);

    // 3. SpanContext 재구성
    SpanContext parentSpanContext = SpanContext.createFromRemoteParent(
        traceId, spanId,
        TraceFlags.getSampled(),
        TraceState.getDefault()
    );

    // 4. Batch Span 생성 + Link 추가
    SpanBuilder spanBuilder = customTracer.spanBuilder("batch.execute")
            .setSpanKind(SpanKind.INTERNAL);

    if (parentSpanContext.isValid()) {
        spanBuilder.addLink(parentSpanContext);  // Span Link!
    }

    batchSpan = spanBuilder.startSpan();

    // 5. Link 태그 추가
    batchSpan.setAttribute("batch.linked.trace.id", traceId);
    batchSpan.setAttribute("batch.linked.span.id", spanId);
    batchSpan.setAttribute("transaction", transactionNumber);
    batchSpan.setAttribute("batch.type", batchType);
}
```

### 7.3 Span Link 정보 저장

OpenTelemetry SDK의 `addLink()`는 references 배열에 저장하지만,
현재 환경에서는 tags에도 link 정보를 저장해서 검색 가능하게 함:

```java
// Link는 Span 간의 논리적 연결 (references 배열)
spanBuilder.addLink(parentSpanContext);

// Tag은 검색 가능한 속성 (tags 배열)
batchSpan.setAttribute("batch.linked.trace.id", traceId);
batchSpan.setAttribute("batch.linked.span.id", spanId);
```

## 8. 실행 흐름 상세

```
1. JVM 시작
   ↓
2. -javaagent:batch-processor.jar 로드
   ↓
3. MANIFEST.MF에서 Premain-Class 읽기
   ↓
4. BatchInstrumentation.premain() 호출
   ↓
5. install(instrumentation) 호출
   ↓
6. ByteBuddy AgentBuilder 설정
   - type: OpenSearchBatchProcessor
   - transform: on(named("executeBatch"))
   - advice: ExecuteBatchAdvice
   ↓
7. OpenSearchBatchProcessor.main() 실행
   ↓
8. startProcessing() 무한 루프 (5초 간격)
   ↓
9. readQueueFile()로 배치 큐 읽기
   ↓
10. searchTransactionInOpenSearch()
    - transaction 태그로 검색
    - Trace ID, Span ID 추출
    ↓
11. executeBatch(transactionNumber, batchType, spanContextInfo)
    ↓
12. [ByteBuddy Advice 자동 실행]
    ↓
13. ExecuteBatchAdvice.onMethodEnter()
    - OpenTelemetry SDK 초기화
    - Span 생성 + addLink()
    - tags 설정 (batch.linked.trace.id 등)
    ↓
14. 배치 로직 실행 (processWordCount 등)
    ↓
15. ExecuteBatchAdvice.onMethodExit()
    - batchSpan.end()
    - OpenTelemetry Exporter가 OpenSearch로 전송
    ↓
16. OpenSearch에 batch.execute Span 저장
```
