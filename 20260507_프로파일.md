## 1. 개요

### 1.1 목적

Java 애플리케이션의 CPU 프로파일링을 통해 성능 병목을 식별하고 최적화 기회를 발견합니다.

### 1.2 핵심 기능

- **CPU 프로파일링**: async-profiler로 샘플 기반 CPU 수집
- **스택 추적**: 전체 호출 스택 분석
- **핫 경로 식별**: 성능 병목 발견
- **JSON 변환**: collapsed 포맷 → 구조화된 JSON
- **v2 최적화**: 중복 제거, 크기 절감

## 3. 소스 수정 내역

### 3.1 async-profiler-enhanced.py - v2 최적화

**위치**: `server1/async-profiler-enhanced.py`

**변경 사항**:

#### 제거된 필드 (6개)

```python
# v1 (제거됨)
def extract_metadata(trace_data, service_name):
    return {
        'top_frame': trace_data['stack'][0],        # ← 제거 (stack[0] 중복)
        'bottom_frame': trace_data['stack'][-1],     # ← 제거 (stack[-1] 중복)
        'has_kernel_frames': kernel_count > 0,       # ← 제거 (계산 가능)
        'has_java_frames': java_count > 0,           # ← 제거 (계산 가능)
        'java_packages': list(packages),             # ← 제거 (stack에서 추출)
        'java_classes': list(classes)                # ← 제거 (stack에서 추출)
    }

# v2 (최적화)
def extract_metadata(trace_data, service_name):
    return {
        'service': service_name,                     # 메타데이터 우선
        'profiler': 'async-profiler',
        'profiler_version': '4.1',
        'timestamp': trace_data['timestamp'],
        'thread_id': trace_data['thread_id'],
        'thread_name': trace_data['thread_name'],
        'sample_count': trace_data['sample_count'],
        'stack_depth': trace_data['stack_depth'],
        'frame_types': trace_data['frame_types'],
        'stack': trace_data['stack']                 # 큰 데이터는 마지막에
    }
```

#### 필드 순서 최적화

| 순서 | v1 | v2 | 이유 |
|------|----|----|------|
| 1 | timestamp | service | 자주 접근하는 메타데이터 우선 |
| 2 | service | profiler | |
| 3 | profiler | profiler_version | |
| 4 | **stack** (큰 데이터) | timestamp | |
| 5 | stack_depth | thread_id | |
| 6 | sample_count | thread_name | |
| 7 | ... | sample_count | |
| 8 | ... | stack_depth | |
| n-1 | ... | frame_types | |
| n | thread_id | **stack** (큰 데이터) | 가장 큰 데이터를 마지막에 |

## 5. 구성 설정

### 5.1 async-profiler 설치

```bash
cd /root
wget https://github.com/jvm-profiling-tools/async-profiler/releases/download/v4.1/async-profiler-4.1-linux-x64.tar.gz
tar -xzf async-profiler-4.1-linux-x64.tar.gz
```

### 5.2 애플리케이션 빌드

```bash
cd server1
./gradlew clean build
```

### 5.3 프로파일링 환경 설정

**run-with-detailed-profiling.sh**:

```bash
#!/bin/bash

# async-profiler 경로
PROFILER_DIR="/root/async-profiler"
PROFILER="$PROFILER_DIR/build/libasyncProfiler.so"

# 출력 파일
OUTPUT_COLLAPSED="profiler-output-collapsed.txt"
OUTPUT_JFR="profiler-output.jfr"

# 프로파일링 옵션
INTERVAL_MS=10          # 샘플링 간격 (10ms)
DURATION_SEC=60         # 프로파일링 duration (60초)
FRAME_BUFFER=2097152    # 프레임 버퍼 (2MB)

# 프로파일링 시작
java -agentpath:$PROFILER=start,event=cpu,collapsed,file=$OUTPUT_COLLAPSED,interval=$INTERVAL_MS,framebuf=$FRAME_BUFFER \
     -jar build/libs/server1-0.0.1-SNAPSHOT.jar
```

**방법 1: 자동 PID 감지**

```bash

./run-with-detailed-profiling.sh
```

**방법 2: PID 직접 지정**

```bash
# PID 찾기
jps | grep server1

# 프로파일링 시작
./run-with-detailed-profiling.sh <PID>
```

**방법 3: 수동 프로파일링**

```bash
# 1. 애플리케이션 PID 찾기
PID=$(jps | grep server1 | awk '{print $1}')

# 2. 프로파일링 시작
./profiler.sh start $PID event=cpu collapsed=interval=10ms framebuf=2m

# 3. 부하 생성 (다른 터미널)
curl http://localhost:8081/api/test

# 4. 프로파일링 중지
./profiler.sh stop $PID -o profiler-output-collapsed.txt

# 5. JFR로 덤프
./profiler.sh dump $PID -o profiler-output.jfr
```

## 8. 트러블 슈팅

### 8.1 async-profiler attach 실패

**현상**:
```
Error: Could not start profiler: Permission denied
```

**원인**: 권한 문제 또는 PID 접근 불가

**해결**:
```bash
# root 권한 확인
sudo -i

# PID 확인
jps -l | grep server1

# 실행 권한 확인
chmod +x /root/async-profiler/profiler.sh
```

### 8.2 샘플이 수집되지 않음

**현상**:
```
Collected 0 samples
```

**원인**: 애플리케이션이 CPU를 사용하지 않음

**해결**:
```bash
# 1. 애플리케이션에 부하 생성
curl http://localhost:8081/api/test
ab -n 1000 -c 10 http://localhost:8081/api/test

# 2. 프로파일링 duration 증가
DURATION_SEC=120  # 60초 → 120초

# 3. 샘플링 간격 감소
INTERVAL_MS=5  # 10ms → 5ms
```

### 8.3 JSON 변환 실패

**현상**:
```
KeyError: 'top_frame'
```

**원인**: v1 코드로 v2 데이터를 읽음

**해결**:
```bash
# async-profiler-enhanced.py v2 사용 확인
grep "top_frame" async-profiler-enhanced.py
# 없어야 함 (v2)

# 스크립트 업데이트
git pull origin main
```

### 13.1 async-profiler-enhanced.py

**핵심 함수**:

```python
def convert_collapsed_to_jsonl(input_file, output_file, service_name):
    """
    collapsed 포맷을 JSONL v2로 변환

    Args:
        input_file: async-profiler collapsed 출력
        output_file: stack-traces.jsonl (v2)
        service_name: 서비스 이름
    """
    traces = parse_collapsed_file(input_file)

    with open(output_file, 'w') as f:
        for trace_data in traces:
            metadata = extract_metadata(trace_data, service_name)
            f.write(json.dumps(metadata) + '\n')
```

### 13.2 build-call-tree.py

**핵심 함수**:

```python
def build_call_tree_from_traces(traces):
    """
    스택 추적에서 호출 트리 빌드

    Returns:
        TreeNode: 계층적 트리 구조
    """
    tree = TreeNode("root")

    for trace in traces:
        node = tree
        for frame in trace['stack']:
            node = node.get_or_add_child(frame)
            node.sample_count += trace['sample_count']

    return tree

def extract_hot_paths(tree, top_n=50):
    """
    상위 N개 핫 경로 추출

    Args:
        tree: 호출 트리
        top_n: 추출할 경로 수

    Returns:
        List[HotPath]: 핫 경로 목록 (with duration_ns)
    """
    paths = []
    extract_paths_recursive(tree, [], paths)

    # duration_ns 추가 (v2)
    for path in paths:
        path['duration_ns'] = path['sample_count'] * 10_000_000

    return sorted(paths, key=lambda x: x['self_samples'], reverse=True)[:top_n]
```
