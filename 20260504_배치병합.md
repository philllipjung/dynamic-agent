### 2.2 핵심 컴포넌트

#### 2.2.1 SparkJobInstrumentation

**위치**: `spark-integration/spark-job-instrumentation/src/main/java/com/javaagent/spark/instrumentation/SparkJobInstrumentation.java`

**역할**: Spark Job의 main 메서드에 진입하여 traceparent를 파싱하고 같은 Trace ID로 Span 생성

**핵심 로직**:
```java
// 1. traceparent 파싱
String[] parts = traceParent.split("-");
String traceId = parts[1];
String spanId = parts[2];

// 2. SpanContext 생성
SpanContext parentSpanContext = SpanContext.create(
    traceId, spanId,
    TraceFlags.getSampled(),
    TraceState.getDefault()
);

// 3. Parent Context 생성 (중요!)
Span parentSpan = Span.wrap(parentSpanContext);
parentContext = Context.root().with(parentSpan);

// 4. Context를 current로 설정
io.opentelemetry.context.Scope parentScope = parentContext.makeCurrent();

try {
    // 5. Span 생성 - 같은 Trace ID로!
    span = tracer.spanBuilder("spark.job")
            .setSpanKind(SpanKind.INTERNAL)
            .startSpan();
} finally {
    parentScope.close();
}
```

**성공 비결**:
- `parentContext.makeCurrent()`를 호출하여 Context를 현재 스레드에 설정
- 이후 `tracer.spanBuilder()`가 `Context.current()`에서 Trace ID를 가져옴

#### 2.2.2 BatchInstrumentation

**위치**: `spark-integration/batch-processor/src/main/java/com/javaagent/spark/batch/BatchInstrumentation.java`

**역할**: 배치 처리 메서드에 진입하여 OpenSearch에서 원본 Trace를 검색하고 Span Link 생성

**핵심 로직**:
```java
// 1. OpenSearch에서 원본 Trace 검색
SpanContextInfo spanContextInfo = searchTransactionInOpenSearch(transactionNumber);

// 2. 원본 SpanContext 재구성
SpanContext parentSpanContext = SpanContext.createFromRemoteParent(
    spanContextInfo.getTraceId(),
    spanContextInfo.getSpanId(),
    TraceFlags.getSampled(),
    TraceState.getDefault()
);

// 3. Batch Span 생성 + Link 추가
SpanBuilder spanBuilder = tracer.spanBuilder("batch.execute")
        .setSpanKind(SpanKind.INTERNAL);

if (parentSpanContext.isValid()) {
    spanBuilder.addLink(parentSpanContext);  // FOLLOWS_FROM 관계
}

Span batchSpan = spanBuilder.startSpan();

// 4. Link 태그 추가
batchSpan.setAttribute("batch.linked.trace.id", traceId);
batchSpan.setAttribute("batch.linked.span.id", spanId);
batchSpan.setAttribute("transaction", transactionNumber);
```

## 4. 트러블슈팅

### 4.1 Trace ID 불일치 문제 (해결됨)

**현상**:
```
입력 traceparent: 00-abc123def45678901234567890123456-1234567890123456-01
기대 Trace ID:   abc123def45678901234567890123456
실제 Trace ID:   74812dc4b1107a1f19f5432447824b0f (새로운 ID!)
```

**원인**:
- Custom OpenTelemetry SDK를 사용할 때 `parentContext`를 `Context.current()`로 설정하지 않음
- `SpanBuilder`가 내부적으로 새로운 Trace ID를 생성

**해결 방법**:
```java
// ❌ 기존 코드
parentContext = Context.root().with(parentSpan);
span = tracer.spanBuilder("spark.job")
        .setParent(parentContext)
        .startSpan();

// ✅ 수정된 코드
parentContext = Context.root().with(parentSpan);
io.opentelemetry.context.Scope parentScope = parentContext.makeCurrent();
try {
    span = tracer.spanBuilder("spark.job")
            .setSpanKind(SpanKind.INTERNAL)
            .startSpan();  // Context.current()에서 Trace ID를 가져옴
} finally {
    parentScope.close();
}
```

**핵심**: `parentContext.makeCurrent()`를 호출하여 `Context.current()`가 parent context를 반환하도록 함

### 4.2 Spark 라이브러리 누락 (해결됨)

**현상**:
```
Exception in thread "main" java.lang.NoClassDefFoundError: org/apache/spark/sql/SparkSession
```

**원인**:
- pom.xml에서 Spark 의존성이 `<scope>provided</scope>`로 설정됨
- Spark 클러스터 환경에서는 정상이지만, 독립 실행 시에는 라이브러리가 필요

**해결 방법**:
```xml
<!-- ❌ 기존 설정 -->
<dependency>
    <groupId>org.apache.spark</groupId>
    <artifactId>spark-core_${scala.binary.version}</artifactId>
    <version>3.5.0</version>
    <scope>provided</scope>  <!-- 문제 원인 -->
</dependency>

<!-- ✅ 수정된 설정 -->
<dependency>
    <groupId>org.apache.spark</groupId>
    <artifactId>spark-core_${scala.binary.version}</artifactId>
    <version>3.5.0</version>
    <!-- scope 제거 -->
</dependency>
```

**결과**: ~225 MB fat JAR 생성 (Spark 라이브러리 포함)

```java
public class SparkJobInstrumentation {

    public static volatile OpenTelemetrySdk customOpenTelemetry;
    public static volatile Tracer customTracer;

    public static synchronized OpenTelemetrySdk getCustomOpenTelemetry() {
        if (customOpenTelemetry == null) {
            String otelEndpoint = OtelConfig.getOtelEndpoint();

            OtlpGrpcSpanExporter spanExporter = OtlpGrpcSpanExporter.builder()
                    .setEndpoint(otelEndpoint)
                    .setTimeout(OtelConfig.getExporterTimeoutSeconds(), TimeUnit.SECONDS)
                    .build();

            SdkTracerProvider tracerProvider = SdkTracerProvider.builder()
                    .addSpanProcessor(SimpleSpanProcessor.create(spanExporter))
                    .build();

            customOpenTelemetry = OpenTelemetrySdk.builder()
                    .setTracerProvider(tracerProvider)
                    .build();  // 주의: buildAndRegisterGlobal() 사용하지 않음

            customTracer = customOpenTelemetry.getTracer("spark-job-instrumentation", "1.0.0");
        }
        return customOpenTelemetry;
    }

    public static class SparkJobAdvice {
        @Advice.OnMethodEnter
        public static void onMethodEnter(
                @Advice.Local("span") Span span,
                @Advice.Local("parentContext") Context parentContext,
                @Advice.Local("parsedTraceId") String parsedTraceId) {

            String traceParent = System.getProperty("traceparent", "");
            parsedTraceId = null;
            parentContext = Context.root();

            if (traceParent != null && !traceParent.isEmpty()) {
                String[] parts = traceParent.split("-");
                if (parts.length == 4 && parts[0].equals("00")) {
                    String traceId = parts[1];
                    String spanId = parts[2];
                    parsedTraceId = traceId;

                    SpanContext parentSpanContext = SpanContext.create(
                        traceId, spanId,
                        TraceFlags.getSampled(),
                        TraceState.getDefault()
                    );

                    Span parentSpan = Span.wrap(parentSpanContext);
                    parentContext = Context.root().with(parentSpan);
                }
            }

            // 핵심: Context를 current로 설정
            io.opentelemetry.context.Scope parentScope = parentContext.makeCurrent();

            try {
                getCustomOpenTelemetry();
                span = customTracer.spanBuilder("spark.job")
                        .setSpanKind(SpanKind.INTERNAL)
                        .startSpan();

                // 검증
                String actualTraceId = span.getSpanContext().getTraceId();
                if (parsedTraceId != null && parsedTraceId.equalsIgnoreCase(actualTraceId)) {
                    Logger.getLogger("SparkJobInstrumentation")
                        .info("✓ Trace ID continuity VERIFIED!");
                }
            } finally {
                parentScope.close();
            }
        }
    }
}
```

public class BatchInstrumentation {

    public static volatile OpenTelemetrySdk customOpenTelemetry;
    public static volatile Tracer customTracer;

    public static class BatchProcessAdvice {
        @Advice.OnMethodEnter
        public static void onMethodEnter(
                @Advice.Argument(0) String transactionNumber,
                @Advice.Argument(1) String batchType,
                @Advice.Argument(2) Object spanContextInfo,
                @Advice.Local("batchSpan") Span batchSpan) {

            getCustomOpenTelemetry();

            // Reflection으로 Trace 정보 추출
            String traceId = extractTraceId(spanContextInfo);
            String spanId = extractSpanId(spanContextInfo);

            // 원본 SpanContext 재구성
            SpanContext parentSpanContext = SpanContext.createFromRemoteParent(
                traceId, spanId,
                TraceFlags.getSampled(),
                TraceState.getDefault()
            );

            // Batch Span 생성 + Link 추가
            SpanBuilder spanBuilder = customTracer.spanBuilder("batch.execute")
                    .setSpanKind(SpanKind.INTERNAL);

            if (parentSpanContext.isValid()) {
                spanBuilder.addLink(parentSpanContext);  // FOLLOWS_FROM
            }

            batchSpan = spanBuilder.startSpan();
            batchSpan.setAttribute("batch.linked.trace.id", traceId);
            batchSpan.setAttribute("batch.linked.span.id", spanId);
            batchSpan.setAttribute("transaction", transactionNumber);
            batchSpan.setAttribute("batch.type", batchType);
        }
    }
}

### 7.1 Spark와 Agent 통합 전후 비교

#### 7.1.1 기존 Spark Job (통합 전)

**아키텍처:**
```
Microservice → Spark Job (독립 실행)
     ↓              ↓
   Span A        Span B (Trace ID 불일치!)
```

**문제점:**
- Spark Job이 새로운 Trace ID 생성
- 원본 요청과 Trace 연결 끊김
- 배치 처리가 독립적인 Trace로 생성

#### 7.1.2 Spark & Agent 통합 후

**아키텍처:**
```
Microservice (Trace ID: abc123...)
     ↓
   traceparent 전달
     ↓
Spark Job (Trace ID: abc123... ← 일치!)
     ↓
   배치 큐 추가
     ↓
Batch Processor (FOLLOWS_FROM link)
```

**개선점:**
- Spark Job이 같은 Trace ID 유지
- 배치 처리가 FOLLOWS_FROM 링크로 연결
- 전체 Trace 흐름 추적 가능

### 7.2 배치 처리 트리거 방식

#### 7.2.1 현재 구현

**OpenSearchBatchProcessor (폴링 방식):**
```java
public void startProcessing() throws InterruptedException {
    while (true) {
        processBatches();           // 배치 큐 파일 읽기
        Thread.sleep(5000);         // 5초마다 폴링
    }
}
```

**작동流程:**
1. `OpenSearchBatchProcessor`가 백그라운드에서 실행

3. `status: "PENDING"`인 배치 처리
4. OpenSearch에서 원본 Trace 검색
5. FOLLOWS_FROM 링크로 Batch Span 생성

**원인**: 배치 큐 파일에서 status 업데이트 안 됨

**임시 해결**: 배치 처리 후 파일에서 제거
```java
// executeBatch() 호출 후
batches.removeIf(batch ->
    transactionNumber.equals(batch.get("transactionNumber"))
);
writeQueueFile(batches);  // 파일 업데이트
```

**추천 해결**: DB 또는 Redis 사용 (status 컬럼/필드 업데이트)
