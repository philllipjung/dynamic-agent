### 핵심 코드

#### ByteBuddyAgent.java (createSpan 메서드)

```java
public static String createSpan(String pid, String className, String methodName) {
    // 1. 타겟 클래스 로드
    Class<?> targetClass = Class.forName(className);
    ClassLoader targetClassLoader = targetClass.getClassLoader();

    // 2. Helper injection (Advice를 타겟 ClassLoader에 주입)
    injectHelper(targetClassLoader);

    // 3. Advice 클래스 로드 (타겟의 ClassLoader에서)
    Class<?> adviceClass = targetClassLoader.loadClass("com.javaagent.bytebuddy.advices.SpanAdvice");

    // 4. ByteBuddy로 변환된 bytecode 생성
    byte[] transformedBytes = new ByteBuddy()
        .redefine(targetClass)
        .visit(Advice.to(adviceClass)
            .on(ElementMatchers.named(methodName)))
        .make()
        .getBytes();

    // 5. Retransformation 적용
    ClassFileTransformer transformer = (loader, className2, classBeingRedefined,
                                         protectionDomain, classfileBuffer) -> {
        if (className2.replace('/', '.').equals(targetClass.getName())) {
            return transformedBytes;
        }
        return null;
    };

    instrumentation.addTransformer(transformer, true);
    instrumentation.retransformClasses(targetClass);
    instrumentation.removeTransformer(transformer);

    return "SUCCESS";
}
```

### Helper Class Injection

```java
private static boolean injectHelper(ClassLoader targetLoader) {
    for (String helperClassName : AgentConstants.HELPER_CLASSES) {
        // 1. System ClassLoader에서 bytecode 로드
        byte[] classBytes = loadHelperClassBytes(helperClassName);

        // 2. Reflection으로 타겟 ClassLoader에 주입
        java.lang.reflect.Method defineClass = ClassLoader.class.getDeclaredMethod(
            "defineClass", String.class, byte[].class, int.class, int.class
        );
        defineClass.setAccessible(true);
        defineClass.invoke(targetLoader, helperClassName, classBytes, 0, classBytes.length);
    }
    return true;
}
```

### SpanAdvice.java

```java
package com.javaagent.bytebuddy.advices;

import net.bytebuddy.asm.Advice;

public class SpanAdvice {
    @Advice.OnMethodEnter(inline = true)
    public static void onMethodEnter() {
        System.err.println(">>> ADVICE ENTER - Hello World!");
        System.err.flush();
        new Exception("ADVICE ENTER STACK TRACE").printStackTrace();
    }

    @Advice.OnMethodExit(inline = true)
    public static void onMethodExit() {
        System.err.println("<<< ADVICE EXIT - Hello World!");
        System.err.flush();
    }
}
```

## ❌ 작동하지 않는 방식

### 1. 정적 Agent 로드 (`-javaagent`)
```bash
# 이 방식은 작동하지 않음
mvn spring-boot:run -Dspring-boot.run.jvmArguments="-javaagent:./bytebuddy-agent-1.0.0.jar"
```

**원인:**
- `-javaagent`로 로드된 agent에서는 `ClassReloadingStrategy.fromInstalledAgent()`가 작동하지 않음
- "The Byte Buddy agent is not installed or not accessible" 에러 발생

### 2. 별도 Advice JAR 분리
```xml
<!-- 이 방식은 작동하지 않음 -->
<filter>
    <artifact>com.javaagent:bytebuddy-advice</artifact>
    <excludes>
        <exclude>**</exclude>
    </excludes>
</filter>
```

**원인:**
- Advice 클래스를 찾지 못함 ("Helper class file not found")
- `loadHelperClassBytes()`가 shaded JAR 안에서만 작동

### 3. Bootstrap Classpath
```bash
# 중복 정의 에러 발생
-Xbootclasspath/a:./bytebuddy-advice-1.0.0.jar
```

**원인:**
```
loader 'app' attempted duplicate class definition for com.javaagent.bytebuddy.advices.SpanAdvice
```

## 🔑 핵심 성공 요인

### 1. 동적 Attach 필수
- `VirtualMachine.attach(pid)`로 실행 중인 JVM에 attach
- `vm.loadAgent(jarPath, args)`로 agent 로드
- 정적 `-javaagent` 대신 동적 attach 사용

### 2. Advice는 Shaded JAR에 포함
- Advice 클래스가 bytebuddy-agent JAR 안에 포함되어야 함
- `loadHelperClassBytes()`가 System ClassLoader에서 bytecode를 읽을 수 있음

### 3. Helper Class Injection
- Advice를 타겟의 ClassLoader에 주입
- `ClassLoader.defineClass()`를 reflection으로 호출
- `--add-opens` 없이도 작동 (동적 attach에서는 필요 없음)

### 4. Retransformation Strategy
```java
instrumentation.addTransformer(transformer, true);  // canRetransform = true
instrumentation.retransformClasses(targetClass);
```

### 5. 같은 ClassLoader 사용
- Advice를 타겟의 ClassLoader에서 로드
- `targetClassLoader.loadClass("com.javaagent.bytebuddy.advices.SpanAdvice")`

## 📌 중요 제약 사항

1. **동적 Attach만 작동**
   - `-javaagent`로 시작된 application에서는 작동하지 않음
   - 실행 중인 JVM에 `VirtualMachine.attach()`해야 함

2. **Advice는 Shaded JAR에 포함**
   - 별도 JAR로 분리하면 "Helper class file not found" 에러

3. **ClassLoader 중요**
   - Advice를 타겟과 같은 ClassLoader에서 로드해야 함
   - Helper injection으로 타겟 ClassLoader에 Advice 주입

4. **Retransformation 필수**
   - 이미 로드된 클래스를 수정하려면 `retransformClasses()` 사용
   - `addTransformer(transformer, true)`에서 `true` 필수
