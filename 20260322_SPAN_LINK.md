# Span Link Feature (스팬 링크 기능)

## 개요

스팬 링크 기능은 서로 다른 Trace를 연결하여 분산 시스템에서의 요청 흐름을 추적할 수 있게 해줍니다. 속성(attribute) 값 기반으로 매칭되는 Trace를 찾아 연결합니다.

**버전**: 1.1.0
**생성일**: 2026-03-22

## 아키텍처

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           Span Link Flow                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 사용자가 링크 설정 (REST API)                                            │
│     ┌──────────────┐                                                        │
│     │ UI → Server  │ POST /api/spanlink/create                              │
│     └──────────────┘                                                        │
│            ↓                                                                │
│     ┌──────────────┐                                                        │
│     │ Redis 저장   │ SpanLinkConfig                                         │
│     └──────────────┘                                                        │
│            ↓                                                                │
│     ┌──────────────┐                                                        │
│     │ ByteBuddy    │ createSpanLinkAdvice()                                 │
│     │ Advice 적용  │                                                        │
│     └──────────────┘                                                        │
│                                                                             │
│  2. 런타임에 링크 처리                                                      │
│     ┌──────────────┐                                                        │
│     │ Source Span  │ 실행                                                    │
│     └──────────────┘                                                        │
│            ↓                                                                │
│     ┌──────────────────────────┐                                           │
│     │ SpanLinkAdvice.onEnter() │                                           │
│     └──────────────────────────┘                                           │
│            ↓                                                                │
│     ┌──────────────────────────────────────┐                               │
│     │ 1. Redis에서 링크 설정 조회           │                               │
│     │ 2. Source 속성 값 추출                │                               │
│     │ 3. OpenSearch에서 매칭 Trace 검색     │                               │
│     │ 4. Span Link 생성                     │                               │
│     └──────────────────────────────────────┘                               │
│            ↓                                                                │
│     ┌──────────────┐                                                        │
│     │ Target Trace │ 연결됨                                                 │
│     └──────────────┘                                                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 사용 예시

### 시나리오: userId로 Trace 연결

```java
// Source Span: /FindDriverIDs
// Attribute: arthas.attribute.userId = "12345"

// Target Span: /GetDriverProfile
// Attribute: arthas.attribute.userId = "12345"

// 링크 생성
POST /api/spanlink/create
{
  "sourceSpanName": "/FindDriverIDs",
  "sourceAttributeKey": "arthas.attribute.userId",
  "targetSpanName": "/GetDriverProfile",
  "targetAttributeKey": "arthas.attribute.userId",
  "linkType": "PARENT_TO_CHILD"
}

// 런타임에 /FindDriverIDs 실행 시
// 1. userId = "12345" 추출
// 2. OpenSearch에서 userId="12345"인 /GetDriverProfile Trace 검색
// 3. 발견된 Trace를 Span Link로 연결
```

## 구성 요소

### 1. SpanLinkConfig (데이터 모델)

**위치**: `agent-commons/src/main/java/com/javaagent/commons/SpanLinkConfig.java`

```java
public class SpanLinkConfig implements Serializable {
    private String linkId;              // 링크 ID
    private String sourceSpanName;      // 소스 스팬명
    private String sourceAttributeKey;  // 소스 속성 키
    private String targetSpanName;      // 타겟 스팬명
    private String targetAttributeKey;  // 타겟 속성 키
    private LinkType linkType;          // 링크 타입
    private boolean enabled;            // 활성화 여부
    private Instant createdAt;          // 생성 시간

    public enum LinkType {
        PARENT_TO_CHILD,   // 부모→자식 관계
        CHILD_TO_PARENT,   // 자식→부모 관계
        RELATED_SPAN       // 관련 스팬
    }
}
```

### 4. SpanLinkHelper (런타임 헬퍼)

**위치**: `bytebuddy-advice/src/main/java/com/javaagent/bytebuddy/helper/SpanLinkHelper.java`

```java
public void addSpanLink(String traceId, String spanId, String linkType) {
    // Span Link를 속성으로 저장
    // (OpenTelemetry SpanLink API는 SpanBuilder에서만 사용 가능)
    String linkKey = "link." + linkType.toLowerCase() + "." + traceId;
    setAttribute(linkKey, spanId);
}
```

**속성 저장 예시**:
```
link.parent_to_child.abc123def456 = span789
```

### 5. SpanLinkAdvice (ByteBuddy Advice)

**위치**: `bytebuddy-advice/src/main/java/com/javaagent/bytebuddy/advices/SpanLinkAdvice.java`

```java
@Advice.OnMethodEnter
public static void onMethodEnter(
    @Advice.Origin Method method,
    @Advice.This Object target
) {
    String className = target.getClass().getName();
    String methodName = method.getName();
    String sourceSpanName = className + "." + methodName;

    SpanLinkHelper helper = new SpanLinkHelper();
    processSpanLinks(helper, sourceSpanName, className, methodName);
}

private static void processSpanLinks(...) {
    // 1. Redis에서 링크 설정 조회 (Reflection)
    List<SpanLinkConfig> linkConfigs = getLinkConfigsFromRedis(sourceSpanName);

    for (SpanLinkConfig config : linkConfigs) {
        // 2. Source 속성 값 추출
        String sourceValue = helper.getAttribute(config.getSourceAttributeKey());

        // 3. OpenSearch에서 매칭 Trace 검색 (Reflection)
        List<TraceLink> targetTraces = findTargetTraces(config, sourceValue);

        // 4. Span Link 생성
        for (TraceLink trace : targetTraces) {
            helper.addSpanLink(
                trace.getTraceID(),
                trace.getSpanID(),
                config.getLinkType().toString()
            );
        }
    }
}
```

```bash

  "targetSpanName": "/GetDriverProfile",
  "targetAttributeKey": "arthas.attribute.userId",
  "linkType": "PARENT_TO_CHILD"
}
```

**응답**:

## 워크플로우

### 1. 링크 생성 워크플로우

```
1. 사용자가 UI에서 소스 스팬 선택
   ↓
2. 소스 스팬의 속성 목록 표시 (OpenSearch 쿼리)
   ↓
3. 연결할 속성 선택 (예: arthas.attribute.userId)
   ↓
4. 타겟 스팬 선택
   ↓
5. 타겟 스팬의 속성 선택
   ↓
6. 링크 타입 선택 (PARENT_TO_CHILD, CHILD_TO_PARENT, RELATED_SPAN)
   ↓
7. 링크 생성 API 호출
   ↓
8. Redis에 링크 설정 저장
   ↓
9. ByteBuddy Advice 적용
   ↓
10. 런타임에 자동 링크 처리 시작
```

### 2. 런타임 링크 처리 워크플로우

```
Source Span 실행
    ↓
SpanLinkAdvice.onMethodEnter() 호출
    ↓
Redis에서 링크 설정 조회 (Reflection)
    ↓
소스 스팬 속성 값 추출 (예: userId = "12345")
    ↓
OpenSearch에서 매칭 Trace 검색 (Reflection)
    ↓
    [Query: operationName=/GetDriverProfile AND
           tags.key=arthas.attribute.userId AND
           tags.value=12345]
    ↓
매칭된 Trace 발견
    ↓
Span Link 생성 (속성으로 저장)
    ↓
    link.parent_to_child.abc123def456 = span789
    ↓
Jaeger UI에서 링크 확인 가능
```

## 주의사항

### 1. OpenTelemetry SpanLink API 제한

OpenTelemetry의 `Span.addLink()` API는 `SpanBuilder` 단계에서만 사용 가능합니다. 이미 시작된 Span에서는 링크를 추가할 수 없습니다.

**해결책**: 링크를 속성으로 저장
```java
// 대신 속성 사용
setAttribute("link.parent_to_child." + traceId, spanId);
```

### 2. ClassLoader 격리

`bytebuddy-advice` 모듈은 `bytebuddy-agent`나 `agent-server` 모듈을 직접 import할 수 없습니다.

**해결책**: Reflection 사용
```java
Class<?> redisService = Class.forName("com.javaagent.bytebuddy.redis.ParameterMappingService");
Method getMethod = redisService.getMethod("getMapping", String.class, String.class);
```

### 3. 성능 고려사항

- **OpenSearch 쿼리**: 런타임에 매번 실행되므로 인덱스 최적화 필요
- **캐싱**: 자주 조회되는 링크 설정은 메모리에 캐시 고려
- **비동기 처리**: OpenSearch 쿼리는 비동기로 처리하여 성능 영향 최소화

### 4. 속성 네이밍 규칙

모든 속성은 `arthas.attribute.` 접두사를 사용해야 OpenSearch에서 인덱싱됩니다.

```java
// 올바른 예
setAttribute("arthas.attribute.userId", "12345");

// 잘못된 예 (OpenSearch에서 검색 불가)
setAttribute("userId", "12345");
```

## 예시 시나리오

### 시나리오 1: 마이크로서비스 간 요청 추적

```
Service A: /CreateOrder
  → attribute: arthas.attribute.orderId = "ORD-12345"

Service B: /ProcessPayment
  → attribute: arthas.attribute.orderId = "ORD-12345"

Service C: /UpdateInventory
  → attribute: arthas.attribute.orderId = "ORD-12345"

링크 설정:
1. /CreateOrder → /ProcessPayment (orderId 기반)
2. /CreateOrder → /UpdateInventory (orderId 기반)
```

### 시나리오 2: 사용자 Journey 추적

```
Frontend: /GenerateUserId
  → attribute: arthas.attribute.userId = "user-123"

Backend: /GetUserProfile
  → attribute: arthas.attribute.userId = "user-123"

Backend: /GetUserPreferences
  → attribute: arthas.attribute.userId = "user-123"

링크 설정:
1. /GenerateUserId → /GetUserProfile (userId 기반)
2. /GenerateUserId → /GetUserPreferences (userId 기반)
```
