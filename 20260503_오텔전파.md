# OTel Java Agent + ByteBuddy Agent 통합 완성 보고서

**날짜**: 2026-05-03
**버전**: 1.0.0
**상태**: ✅ 완성 - "한 트레이스에서 전파와 링크 동시에" 구현 성공

---

## 📋 개요

### 목표
한 트레이스 내에서 **전파(CHILD_OF)**와 **링크(FOLLOWS_FROM)**가 **동시에 존재**하는 구조 구현

### 성공 결과
- ✅ OTel Java Agent → ByteBuddy Agent 간 Context Propagation (CHILD_OF)
- ✅ 서로 다른 요청 간 Span Link (FOLLOWS_FROM)
- ✅ 한 ByteBuddy Span에서 두 관계가 동시에 존재

### 검증된 트레이스
- **Trace A** (test2): `8c6ae8fa83b6a7ae19b93ccd395acef6`
- **Trace B** (test1): `a6d889b73ace969c802c266680b6165d` ⭐

---

## 🎯 구현 목표

### 요구사항 정의

```
┌─────────────────────────────────────────────────────────────────┐
│                    한 요청 내에서                               │
│                                                                  │
│  [OTel Agent] HTTP Request Span (ROOT)                          │
│     │                                                            │
│     ├─ CHILD_OF                                                 │
│     │                                                          │
│     ▼                                                          │
│  [ByteBuddy Agent] Method Span                                 │
│     │                                                            │
│     ├─ CHILD_OF (OTel → ByteBuddy 전파) ✅                     │
│     │                                                            │
│     └─ FOLLOWS_FROM (이전 요청으로의 링크) ✅                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 기대 효과
1. **자동 계측 + 수동 계측 혼합**: OTel Agent가 HTTP 자동 계측, ByteBuddy Agent가 메서드 수동 계측
2. **연속性 추적**: 하나의 TraceID로 전체 요청 추적
3. **관계성 추적**: FOLLOWS_FROM으로 논리적 연결 표현

## ❌ 초기 문제점

### 문제 1: ClassNotFoundException
**증상**:
```
java.lang.ClassNotFoundException: com.test.service.test2.controller.Test2Controller
    at java.base/jdk.internal.loader.BuiltinClassLoader.loadClass
    at com.javaagent.bytebuddy.ByteBuddyAgent.createSpan(ByteBuddyAgent.java:325)
```

**원인**:
- `java -jar`로 Spring Boot JAR 실행 시 클래스가 `BOOT-INF/classes`에 있음
- `Class.forName()`이 AppClassLoader에서만 찾아서 클래스 발견 실패
- Spring Boot의 `LaunchedURLClassLoader` 구조 문제

### 문제 2: Span.current() == false
**증상**:
```
[ByteBuddy] Span created and made current: test2Controller.test2
[ByteBuddy] Span.current() == span: false  ❌
```

**원인**:
- OpenTelemetry Context가 제대로 전파되지 않음
- OTel Java Agent와 ByteBuddy Agent 간 Context 충돌

### 문제 3: 트레이스 미검출
**증상**:
- Span 생성 로그는 있으나 Jaeger UI에 트레이스 없음
- `curl -s "http://localhost:16686/api/traces?..."` 결과 0개

**원인**:
1. ByteBuddy Agent JAR가 최신 코드로 빌드되지 않음
2. shade plugin이 의존성을 포함하지 않음
3. OTel Exporter로 전송되지 않음

### 문제 4: 서비스 이름 검색 실패
**증상**:
- SpanAdvice 코드에서 `test1-service`, `test2-service`로 수정했음
- 하지만 여전히 `unknown_service:java`로 검색

**원인**:
- bytebuddy-advice 모듈 재빌드 미실시
- JAR 파일 업데이트 누락

### ⭐ 문제 5: JaegerLinkLookupService 파싱 버그
**증상**:
```
[JaegerLink] Matched span: traceID=..., spanID=bf6bb4190a9d0929, operation=Test2Controller.test2
[JaegerLink] Matched span: traceID=..., spanID=d6b94929d6f194, operation=Test2Controller.test2
[JaegerLink] Matched span: traceID=..., spanID=7f09a4f9174c4a7c, operation=Test2Controller.test2
```
3개의 span이 모두 같은 operationName으로 매칭됨

**원인**:
- JaegerLinkLookupService의 `parseJaegerResponse()` 메서드 파싱 로직 버그
- spanID와 operationName을 각각 별도로 추출한 후 순서대로 인덱스 매칭
- 실제 JSON에서는 span 객체 단위로 필드가 쌍을 이루어야 함

**실제 Jaeger 데이터**:
- `bf6bb4190a9d0929`: `"GET"` (HTTP Client)
- `d6b94929d6d3f194`: `"GET /test2"` (HTTP ROOT)
- `7f09a4f9174c4a7c`: `"Test2Controller.test2"` (ByteBuddy) ✅

**버그 로직**:
```java
// 틀린 버전
List<String> spanIds = new ArrayList<>();  // [id1, id2, id3]
List<String> opNames = new ArrayList<>(); // [op1, op2, op3]
// 순서대로 매칭 → spanIds[0]와 opNames[0]이 쌍이라고 가정 ❌
```

**결과**: ByteBuddy span 1개만 링크되어야 하는데 3개가 모두 링크됨

### 해결 5: JaegerLinkLookupService 파싱 로직 수정 (문제 5)

**문제**: Jaeger JSON 응답에서 spanID와 operationName을 따로 추출해서 인덱스로 매칭하다 보니, 첫 번째 operationName이 모든 spanID에 적용됨

**해결**: 각 span 객체를 단위로 추출해서 spanID와 operationName을 쌍으로 매칭

**수정 전 (버그 코드)**:
```java
// JaegerLinkLookupService.java - parseJaegerResponse()
private static List<SpanRef> parseJaegerResponse(String jsonResponse, String targetOperationName, Set<String> seenSpanKeys) {
    List<SpanRef> spans = new ArrayList<>();

    // ❌ 모든 spanID 추출
    Pattern spanIdPattern = Pattern.compile("\"spanID\"\\s*:\\s*\"([0-9a-fA-F]+)\"");
    Matcher spanIdMatcher = spanIdPattern.matcher(jsonResponse);
    List<String> spanIds = new ArrayList<>();
    while (spanIdMatcher.find()) {
        spanIds.add(spanIdMatcher.group(1));
    }

    // ❌ 모든 operationName 추출
    Pattern opNamePattern = Pattern.compile("\"operationName\"\\s*:\\s*\"([^\"]+)\"");
    Matcher opNameMatcher = opNamePattern.matcher(jsonResponse);
    List<String> opNames = new ArrayList<>();
    while (opNameMatcher.find()) {
        opNames.add(opNameMatcher.group(1));
    }

    // ❌ 인덱스로 매칭 (잘못됨!)
    for (int i = 0; i < Math.min(spanIds.size(), opNames.size()); i++) {
        String operationName = opNames.get(i);
        if (targetOperationName.equals(operationName)) {
            String spanId = spanIds.get(i);
            // ... 매칭된 것으로 처리
        }
    }
}
```

**수정 후 (정상 코드)**:
```java

private static List<SpanRef> parseJaegerResponse(String jsonResponse, String targetOperationName, Set<String> seenSpanKeys) {
    List<SpanRef> spans = new ArrayList<>();

    // ✅ 각 span 객체를 단위로 추출
    Pattern spanObjectPattern = Pattern.compile("\\{[^}]*\"spanID\"[^}]*\"operationName\"[^}]*\\}");
    Pattern spanIdPattern = Pattern.compile("\"spanID\"\\s*:\\s*\"([0-9a-fA-F]+)\"");
    Pattern opNamePattern = Pattern.compile("\"operationName\"\\s*:\\s*\"([^\"]+)\"");

    Matcher spanObjectMatcher = spanObjectPattern.matcher(jsonResponse);

    while (spanObjectMatcher.find()) {
        String spanObject = spanObjectMatcher.group();  // ✅ 단일 span 객체

        // ✅ 같은 객체 내에서 spanID와 operationName 추출
        Matcher spanIdMatcher = spanIdPattern.matcher(spanObject);
        Matcher opNameMatcher = opNamePattern.matcher(spanObject);

        if (spanIdMatcher.find() && opNameMatcher.find()) {
            String spanId = spanIdMatcher.group(1);
            String operationName = opNameMatcher.group(1);  // ✅ 올바른 쌍

            // ✅ 정확한 operationName 매칭
            if (targetOperationName.equals(operationName)) {
                // traceID 추출 및 SpanRef 생성
                // ...
            }
        }
    }
}
```

**결과**: 3개의 span이 모두 매칭되던 문제가 해결되어, 정확히 1개의 ByteBuddy span만 링크됨

    // Check if OpenTelemetry Java Agent is already initialized
    try {

        if (openTelemetry != null && openTelemetry != io.opentelemetry.api.OpenTelemetry.noop()) {

### 수정 3: SpanHelper.java

**Links 추가 로직**:
```java
// bytebuddy-advice/src/main/java/com/javaagent/bytebuddy/helper/SpanHelper.java
public static SpanHelper createSpanWithLinks(String methodName, List<SpanContext> linkedContexts) {
    if (tracer == null) {
        System.err.println("[SpanHelper] Tracer not initialized!");
        return null;
    }

    // SpanBuilder 생성
    io.opentelemetry.api.trace.SpanBuilder builder = tracer.spanBuilder(methodName);

    // Links 추가
    if (linkedContexts != null && !linkedContexts.isEmpty()) {
        for (SpanContext context : linkedContexts) {
            builder.addLink(context);
            System.out.println("[SpanHelper] Added link to span: "
                + context.getTraceId() + "/" + context.getSpanId());
        }
    }

    // Span 시작 (자동으로 현재 Context의 Span을 부모로 참조 - CHILD_OF)
    Span span = builder.startSpan();

    // CRITICAL: Make this span the current span in context
    Scope scope = span.makeCurrent();

    return new SpanHelper(span, scope);
}
```

### 수정 4: SpanAdvice.java

**서비스 이름 설정**:
```java
// bytebuddy-advice/src/main/java/com/javaagent/bytebuddy/advices/SpanAdvice.java
public static List<SpanContext> findLinkedContexts(
    String className,
    String methodName,
    String userId
) {
    String targetService = null;
    String targetOperation = null;

    // test1 → test2
    if (className.contains("Test1Controller")) {
        targetService = "test2-service";  // test1은 test2를 링크
        targetOperation = "public java.lang.String com.test.service.test2.controller.Test2Controller.test2(java.lang.String)";
    }
    // test2 → test1
    else if (className.contains("Test2Controller")) {
        targetService = "test1-service";  // test2는 test1을 링크
        targetOperation = "public java.lang.String com.test.service.test1.controller.Test1Controller.test1(java.lang.String)";
    }

    if (targetService == null || targetOperation == null) {
        return Collections.emptyList();
    }

    return JaegerLinkLookupService.findTargetSpanContexts(
        targetService,
        targetOperation,
        userId
    );
}
```

## 🎓 교훈 (Lessons Learned)

### 1. Spring Boot JAR 구조 이해
- `java -jar`는 `BOOT-INF/classes` 구조로 인해 `Class.forName()` 실패
- `mvn spring-boot:run`은 classpath를 직접 사용하여 성공
- 개발 중에는 `mvn spring-boot:run` 사용 권장

### 2. Shade Plugin 필수성
- 의존성이 포함된 fat JAR 생성 필요
- `maven-shade-plugin`으로 bytebuddy-agent 빌드
- JAR 크기로 확인: 14M 이상이어야 함

### 3. OTel Context Propagation
- `GlobalOpenTelemetry.get()`으로 기존 인스턴스 재사용
- `Span.makeCurrent()`로 Context 설정
- 자동으로 CHILD_OF 관계 형성

### 4. 테스트 절차의 중요성
- 첫 요청 → 대기 → 두 번째 요청 순서 준수
- OpenSearch 인덱싱 대기 시간 확보 (12-18초)
- userId로 트레이스 검색

### 5. 디버깅 팁
- 서비스 로그에서 `SUCCESS: Created span` 확인
- `SPAN CREATED: true` 확인
- Jaeger API로 먼저 검색 후 UI에서 확인
